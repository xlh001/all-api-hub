Web AI API スニッフィングと検証

ウェブページ上の Base URL と API Key をスニッフィングし、即座に検証します。右クリックメニューによるクイックチェックと、特定のサイトの自動検出をサポートしています。

背景

AI コミュニティ（Linux.do など）、GitHub Issues、または技術ドキュメントを閲覧しているとき、共有されたテストキーに遭遇することがよくあります。従来の方法は、Base URL をコピーし、Key をコピーし、拡張機能を開き、設定を入力して、検証をクリックすることでした。

「Web AI API スニッフィング」機能は、このプロセスを次のように簡素化します：

  • テキストを選択 → 右クリックで「AI API をチェック」 → 直接テスト。
  • または：ホワイトリストに登録されたサイトでは、拡張機能がページ上の設定を自動的に識別し、プロンプトを表示します。

コア機能

1. 右クリックメニューチェック (Context Menu)

任意のウェブページで Base URL または API Key を含むテキストを選択し、右クリックして「AI API をチェック」を選択します。 拡張機能は、選択したテキスト（またはクリップボード）から有効なインターフェース情報の抽出を試み、現在のページ内にテストオーバーレイを表示します。

2. 自動スニッフィング (Auto Detect)

特定のホワイトリストサイト（例：linux.do）で、多数のキーを含む投稿を開くと、拡張機能はページコンテンツを自動的にスキャンします。

  • 有効と思われる Base URL + Key の組み合わせが抽出されると、ページの右上に軽量なプロンプトが表示されます。
  • プロンプトをクリックすると、テストダイアログが開きます。

3. ページ内テストダイアログ

現在のウェブページを離れることなく、以下の操作を実行できます：

  • 自動抽出：ソーステキストから Base URL、シークレットキー、API タイプ を自動的に分離します。
  • 即時テスト： 「テスト開始」をクリックして、キーの接続性と利用可能なモデルを確認します。
  • モデルリストの取得：アップストリームサイトでサポートされているモデル ID の完全なリストをワンクリックで取得します。
  • API 認証情報庫に保存：検証されたら、ワンクリックで拡張機能の API 認証情報庫 に保存し、後でコピー、検証、または CherryStudio などのツールへのエクスポートに使えます。

設定とカスタマイズ

「設定 → 基本設定 → Web AI API チェック」に移動して、以下を構成します：

1. 基本スイッチ

  • 右クリックメニュー：ブラウザの右クリックメニューに「AI API をチェック」を表示するかどうか。
  • 自動検出：一致するサイトでバックグラウンドの自動スキャンを有効にするかどうか。

2. サイトホワイトリスト (URL Whitelist)

拡張機能が自動的にスニッフィングすることを許可するサイトをカスタマイズできます。

  • 正規表現 (RegExp) マッチング（大文字と小文字を区別しない）をサポートしています。
  • デフォルト設定：通常、一般的な AI フォーラムやコミュニティ向けの組み込みマッチングルールが含まれています。
  • 例：linux\.do と入力すると、そのフォーラムのすべてのページに一致します。

プライバシーとセキュリティ

  • ローカル抽出：スニッフィングと抽出プロセスは、ブラウザ内で完全にローカルに実行されます。
  • プライバシー保護：拡張機能は、検出されたテキストをサーバーに自動的にアップロードしません。API リクエストは、「テスト開始」をクリックしたときに入力した Base URL に従ってのみ送信されます。
  • サイレントスキャン：自動検出プロセスは非常に軽量であり、ウェブページのスクロールや読み込みパフォーマンスに影響を与えません。

関連ドキュメント

  • API 認証情報庫：保存されたスニッフィング結果を管理します。
  • インターフェース検証とヘルスステータス：拡張機能がインターフェースを検証する方法について説明します。
