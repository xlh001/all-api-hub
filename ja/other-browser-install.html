その他のブラウザへのインストールガイド

この記事では、QQブラウザ、360セーフブラウザ、360エクストリームブラウザ、チーターブラウザ、Brave、Vivaldi、OperaなどのブラウザにAll API Hub拡張機能をインストールする方法を説明します。

これらのブラウザの多くは Chromium ベースですが、利用できるインストール方法は同じではありません。ブラウザストアまたは Chrome ウェブストアからインストールできる場合は、まずストア版を使ってください。ストア経路が使えない場合、組織ポリシーで制限されている場合、または一時的に GitHub 版を確認したい場合だけ、GitHub Releases の Chrome パッケージを手動読み込みします。

まず違いを確認

  • Chrome / Edge / Firefoxユーザー：各ストア版を優先して使用してください。更新がより簡単です。
  • Brave / Vivaldi ユーザー：まず Chrome ウェブストアからのインストールを試してください。これらのブラウザは公式に Chrome ウェブストア拡張機能をサポートしています。
  • Opera ユーザー：Opera には Opera Add-ons がありますが、All API Hub は現在 Opera Add-ons には単独掲載されていません。まず Chrome ウェブストアを試し、Opera のバージョンやポリシーで利用できない場合に GitHub 手動パッケージを使ってください。
  • QQブラウザ、360系ブラウザ、チーターブラウザなどのユーザー：ブラウザ内蔵の拡張機能センターで All API Hub が見つからない場合は、GitHub の Chrome パッケージを使い、「展開された拡張機能を読み込む」でインストールします。
  • Safariユーザー：インストール方法が異なります。XcodeまたはSafari専用パッケージで処理する必要があります。Safari拡張機能インストールガイドを確認してください。
  • モバイルブラウザ：ブラウザによって拡張機能のサポート状況が大きく異なります。モバイル版については、FAQのモバイルブラウザサポートを確認してください。

ヒント

手動で読み込んだ拡張機能は通常自動更新されません。後続のアップグレードには、新しいバージョンのChrome圧縮ファイルを再度ダウンロードし、拡張機能管理ページで「再読み込み」または再インストールをクリックする必要があります。

ストアの有無と All API Hub の掲載状況は別です

ブラウザに独自の拡張機能ストアがあっても、All API Hub がそのストアに掲載されているとは限りません。たとえば Opera には Opera Add-ons がありますが、現在は Chrome ウェブストアまたは GitHub 手動パッケージを使ってください。

GitHub 手動インストールパッケージの準備

Chrome ウェブストア、Edge Add-ons、Firefox Add-ons から直接インストールできる場合、この手順は不要です。以下は手動読み込みが必要なブラウザ向けです。

  1. 最新リリースバージョンを開きます。
  2. 添付ファイルからChrome版の圧縮ファイルをダウンロードします：
all-api-hub-<version>-chrome.zip

例：

all-api-hub-3.32.0-chrome.zip
  1. 圧縮ファイルを、D:\Extensions\all-api-hub\ のような固定ディレクトリに解凍します。
  2. 選択した拡張機能ディレクトリに manifest.json ファイルが直接見えることを確認します。

ディレクトリの選択

拡張機能を読み込む際は、manifest.json を含むディレクトリを選択してください。「マニフェストファイルが見つかりません」または「manifest.json が存在しません」というエラーが表示される場合は、通常、外側の親ディレクトリを選択しすぎている可能性があります。

その他のデスクトップブラウザでの手動読み込み手順

ストア版を使えないものの、Chrome 拡張機能ディレクトリの読み込みには対応しているデスクトップブラウザ向けです。

  1. ブラウザの拡張機能管理ページを開きます。
    • アドレスバーに chrome://extensions/ と入力して試してください。
    • 開けない場合は、ブラウザメニューから「拡張機能」、「拡張機能」、「プラグイン管理」、または「アプリ管理」を選択してください。
  2. 開発者モード をオンにします。
  3. 「展開された拡張機能を読み込む」をクリックします。
  4. 先ほど解凍し、manifest.json を含むディレクトリを選択します。
  5. インストール後、拡張機能リストで All API Hub が有効になっていることを確認します。
  6. ツールバーにアイコンが表示されない場合は、拡張機能ボタンからAll API Hubをツールバーに固定します。
  7. インストール前に転送ステーションページを開いていた場合は、これらのページをリロードしてから自動認識を使用してください。

QQブラウザのインストール

QQブラウザのデスクトップ版では、通常Chrome版の圧縮パッケージを使用して手動で読み込むことができます。

1. 拡張機能管理ページへのアクセス

いずれかの方法を選択してください：

  • アドレスバーに qqbrowser://extensions と入力
  • 上記のアドレスが利用できない場合は、chrome://extensions/ を試す
  • 右上メニューから「拡張機能」、「拡張機能」、「アプリセンター」などを選択し、「拡張機能の管理」を開く

2. 開発者モードの有効化

拡張機能管理ページで 開発者モード のスイッチを見つけてオンにします。バージョンによって、このオプションはページの右上、ページ下部、または「高度な管理」セクションにある場合があります。

3. 拡張機能ディレクトリの読み込み

  1. 「展開された拡張機能を読み込む」をクリックします。
  2. 解凍した all-api-hub-<version>-chrome ディレクトリを選択します。
  3. インストール後、拡張機能が有効になっていることを確認します。

4. よくある問題への対応

  • ページに 開発者モード がない場合：現在のQQブラウザのバージョンでは外部拡張機能のインストールが制限されている可能性があります。最新のデスクトップフルバージョンにアップグレードするか、Chrome / Edgeの使用を検討してください。
  • 「QQブラウザストアからのものではありません」というメッセージが表示される場合：「展開された拡張機能を読み込む」を優先して使用してください。圧縮ファイルを直接ドラッグアンドドロップしないでください。
  • 拡張機能がインストールされているのにページが認識しない場合：転送ステーションのページをリロードし、ページがブラウザの互換モードや特殊なセキュリティモードになっていないことを確認してください。

360系ブラウザのインストール

360セーフブラウザと360エクストリームブラウザはどちらもChromium拡張機能をサポートしている可能性がありますが、アクセス名やセキュリティ通知はバージョンによって異なります。

1. エクストリームモードの使用を確認

360セーフブラウザには、「エクストリームモード/互換モード」などの異なるレンダリングエンジンモードがあります。All API HubはChromium拡張機能であるため、認識が必要なサイトページではエクストリームモードを使用することをお勧めします。

アドレスバーの近くに 稲妻 / e などのレンダリングエンジン切り替えアイコンが表示されている場合は、エクストリームモード に切り替えてから拡張機能を使用してください。

2. 拡張機能管理ページへのアクセス

いずれかの方法を選択してください：

  • アドレスバーに chrome://extensions/ と入力
  • 右上メニューから「拡張機能」、「拡張機能管理」、「プラグイン管理」、または「アプリ管理」を選択
  • 高度な管理 エントリがある場合は、まず高度な管理に入り、デベロッパー関連のオプションを有効にする

3. 開発者モードの有効化と読み込み

  1. 開発者モード をオンにします。
  2. 「展開された拡張機能を読み込む」をクリックします。
  3. 解凍した all-api-hub-<version>-chrome ディレクトリを選択します。
  4. インストール後、All API Hubを有効にし、必要に応じてツールバーに固定します。

4. よくある問題への対応

  • ブラウザのセキュリティポリシーが外部拡張機能をブロックする場合：展開して読み込む方法を維持してください。.zip または .crx ファイルを直接ドラッグアンドドロップしないでください。
  • ブラウザを再起動すると拡張機能が無効になる場合：拡張機能管理ページで手動で有効にしてください。それでも無効になる場合は、Chrome / Edgeへの切り替え、またはブラウザのセキュリティポリシーを確認することをお勧めします。
  • 一部のウェブページで拡張機能が機能しない場合：そのページが互換モードに切り替わっていないことを確認し、インストール前に開いていたページをリロードしてください。

チーターブラウザのインストール

チーターブラウザでは、組み込みの拡張機能管理ページからChrome版の圧縮パッケージを手動で読み込むことができます。

1. 拡張機能管理ページへのアクセス

アドレスバーに以下を入力します：

liebao://extensions/

このアドレスが利用できない場合は、ブラウザメニューから「拡張機能」、「拡張機能」、または「プラグイン管理」を選択してください。

2. 開発者モードの有効化

拡張機能管理ページで 開発者モード をオンにします。

3. 拡張機能ディレクトリの読み込み

  1. 「展開された拡張機能を読み込む」をクリックします。
  2. 解凍した all-api-hub-<version>-chrome ディレクトリを選択します。
  3. インストール後、拡張機能が有効になっていることを確認します。

4. よくある問題への対応

  • ブラウザに「このウェブサイトの設定を変更しますか？」のようなプロンプトが表示された場合：まず「設定を保持」を選択し、ブラウザがホームページやタブの設定を変更しないようにしてください。
  • 拡張機能がインストールされても機能しない場合：インストール前に開いていた対象サイトのページをリロードしてください。
  • 現在のバージョンで開発者モードまたは読み込みエントリが見つからない場合：チーターブラウザのデスクトップ版にアップグレードするか、Chrome / Edgeの使用を検討してください。

Brave / Vivaldi / Opera のインストール

これらのブラウザは通常 Chrome ウェブストアを利用できます。まず対象ブラウザで All API Hub の Chrome ウェブストアページ を開き、インストールを試してください。

  • Brave は公式に Chrome ウェブストアからの拡張機能インストールをサポートしています。
  • Vivaldi は公式に Chrome ウェブストア拡張機能をサポートしています。インストールボタンが表示されない場合は、Vivaldi の Google Extensions / Web Store 関連設定を確認してください。
  • Opera には Opera Add-ons があり、Chrome 拡張機能にも対応しています。All API Hub は現在 Opera Add-ons には単独掲載されていないため、まず Chrome ウェブストアを使ってください。

ストアインストールに失敗した場合は、以下の拡張機能管理ページから GitHub Chrome パッケージを手動読み込みします：

ブラウザ拡張機能管理ページ
Bravebrave://extensions/
Vivaldivivaldi://extensions/
Operaopera://extensions/
その他のChromiumブラウザchrome://extensions/ を優先して試してください

拡張機能管理ページにアクセスしたら、手動読み込み手順に従って開発者モードを有効にし、解凍したディレクトリを読み込んでください。

拡張機能の更新

手動で読み込んだバージョンは自動更新されません。ストア版、GitHub Stable、Nightly の違いについては、先に インストール方法と更新について を確認してください。手動で読み込んだバージョンを更新する場合は、以下の手順に従ってください：

  1. 最新リリースバージョンを開きます。
  2. 新しい all-api-hub-<version>-chrome.zip をダウンロードします。
  3. 元の固定ディレクトリに解凍するか、新しいバージョンディレクトリに解凍します。
  4. 拡張機能管理ページを開きます。
  5. 元のディレクトリを引き続き使用する場合は、All API Hubのカードにある「再読み込み」をクリックします。
  6. 新しいディレクトリに変更した場合は、まず古いバージョンを削除し、次に新しいディレクトリを読み込みます。

注意

拡張機能ディレクトリをシステムのテンポラリディレクトリ、ダウンローダーのキャッシュディレクトリ、または削除される可能性のあるソフトウェアのディレクトリに置かないでください。ディレクトリが削除または移動されると、ブラウザはその拡張機能を読み込めなくなります。

アンインストール

  1. ブラウザの拡張機能管理ページを開きます。
  2. All API Hub を見つけます。
  3. 「削除」、「アンインストール」、または有効化スイッチをオフにします。
  4. ローカルの解凍ディレクトリを削除します。

よくある質問

ChromeパッケージとFirefox / Safariパッケージのどちらをダウンロードすべきですか？

手動読み込みが必要な Chromium ブラウザでは、以下をダウンロードしてください：

all-api-hub-<version>-chrome.zip

Firefoxの.xpi / Firefoxパッケージや、SafariのXcodeバンドルはダウンロードしないでください。

.zip ファイルを直接インストールできますか？

通常は推奨しません。まず解凍し、「展開された拡張機能を読み込む」でmanifest.jsonを含むディレクトリを選択してください。

インストールに成功した後も自動認識が失敗するのはなぜですか？

一般的な原因は以下の通りです：

  • インストール前に目的のサイトページを開いていた場合、ページをリロードする必要があります。
  • 目的のサイトのログイン状態が無効になっている場合、まず同じブラウザで再ログインする必要があります。
  • ブラウザが互換モードまたはセキュリティモードに切り替わっており、Chromium拡張機能がページに注入されていない。
  • 現在のブラウザが拡張機能APIを完全にサポートしていない場合、Chrome / Edgeで再現を確認してください。

サイドバーのエントリが利用できない場合はどうすればよいですか？

一部のブラウザは、Chrome / Edgeのサイドバー機能をサポートしていません。このような場合は、拡張機能のポップアップまたは設定ページを使用してアカウント管理を行ってください。コア機能はサイドバーに依存しません。
