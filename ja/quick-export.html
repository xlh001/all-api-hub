サイト設定の高速エクスポート

登録済みの集約型中継ステーションアカウントを、CherryStudio、CC Switch、New API、Claude Code Hub などのダウンストリームシステムにワンクリックで同期し、Base URL、API キー、モデルリストの重複入力を回避します。

サポート対象

対象方法備考
CherryStudioローカルプロトコルを介してクライアントを起動し、API 情報を自動入力CherryStudio デスクトップ版を起動し、認証が必要です
CC SwitchJSON / クリップボード形式で出力し、専用のフィールドマッピングを内蔵CC Switch 内でインポート機能を使って内容を貼り付ける必要があります
セルフホスト型サイト（New API / DoneHub / Veloera / Octopus / AxonHub / Claude Code Hub）対象バックエンドの管理インターフェースを呼び出し、Channel / Provider を自動作成または更新拡張機能で対応するバックエンド設定を事前に完了する必要があります

事前設定

  1. サイト同期：まずプラグインでアカウント認識を完了し、キーリストにエクスポート可能な API が存在することを確認します。
  2. 対象資格情報：
    • CherryStudio / CC Switch：追加設定は不要ですが、アプリケーションを起動したままにしてください。
    • セルフホスト型サイト：基本設定 → セルフホスト型サイト管理 で対象タイプを選択し、設定を完了します。
  3. モデルリスト：ホワイトリストエクスポートが必要な場合は、「New API モデル同期」で事前にモデルを絞り込めます。

操作手順

  1. プラグインを開き、キー管理 で任意のサイトカードにある 「エクスポート」 ボタンをクリックします。
  2. 対象プラットフォームを選択します：CherryStudio / CC Switch / 現在のセルフホスト型サイト
  3. 表示に従って認証を完了します。
    • CherryStudio：ブラウザがデスクトップクライアントを開くか確認するので、承認すると自動で完了します。
    • CC Switch：JSON を生成してクリップボードにコピーするので、CC Switch に切り替えて貼り付けます。
    • セルフホスト型サイト：対応する管理インターフェースをバックグラウンドで呼び出し、現在のサイト設定を Provider / Channel としてインポートします。
  4. 対象システムにチャネル / プロバイダーが作成されたことを確認し、実際に呼び出してテストします。

エクスポート内容

フィールド説明
サイト名サイト / アカウントのメモから自動取得され、エクスポート前に変更できます
Base URLアカウントの base_url を使用します。プロトコルのプレフィックスが含まれていることを確認してください
API キーキーリストから取得されます。複数キーに対応するサイトでは、それぞれ個別に列挙されます
モデルリストサイト機能の検出結果、または New API モデル同期の結果から取得されます
チャージ比率CherryStudio / CC Switch での換算表示に使用されます
グループ / 優先度セルフホスト型サイト向け。エクスポートパネルで対象バックエンドの機能に応じて調整できます

よくある質問

問題対処方法
セルフホスト型サイトで 401/403 エラーが表示されるバックエンドの資格情報が期限切れでないこと、プラグイン側で設定を再保存済みであることを確認してください。必要に応じて Cloudflare シールド回避アシスタント を参照してください。
CherryStudio が応答しないデスクトップ版がインストールされており、ブラウザが cherrystudio:// プロトコルの起動を許可しているか確認してください。
CC Switch のインポートが失敗する生成された JSON を公式のインポートダイアログに貼り付けてください。フィールド不足が表示される場合は、CC Switch を最新バージョンに更新してください。
モデルリストが空サイトがまだモデルデータを返していません。まずプラグイン内でモデルリストを更新するか、New API モデル同期を実行してください。
