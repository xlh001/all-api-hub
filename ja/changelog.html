更新ログ

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録します。完全な履歴バージョンとより詳細な技術的変更については、GitHub Releases を参照してください。

初めての方へ

  • 現在のバージョンを確認する方法：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
  • このページが自動的に開かないようにするには：「設定 → 一般 → 更新ログ」で、「更新後に更新ログを自動的に開く」を制御できます。
  • 異常のトラブルシューティング：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて Issues に報告してください。

3.30.0

  • 新機能:
    • チャンネル管理：チャンネル移行 を追加しました。移行プレビューを確認してから、現在のフィルタリング結果または選択したチャンネルを他のホスティングサイトに移行できます。
    • ホスティングサイト：
      • 「設定 → 自社サイト管理」で、現在のホスティングサイトタイプを直接切り替えられます。
      • Done Hub / Veloera が実際のチャンネルキーを読み取れるようになりました。
    • CC Switch エクスポート：OpenCode / OpenClaw タイプを追加しました。
    • 多言語対応：日本語と繁体字中国語（zh-TW）のインターフェース言語を追加しました。
  • 体験最適化:
    • 初回使用とインターフェース：
      • ウェルカム / 権限ガイドのポップアップで言語を直接選択できるようになりました。
      • 設定ページの上部にテーマと言語のクイック切り替えを追加しました。
    • API 検証：保存済みの API 認証情報 とモデル検証で、直近の検出結果と時間が保持されるようになりました。
    • リソース使用量：一部の設定ページ、拡張機能ポップアップ、ビューでオンデマンド読み込みを採用し、不要な初期化とリソース使用量を削減しました。
  • 問題修正:
    • WebDAV：Nutstore が 409 AncestorsNotFound を返すシナリオに対応しました。この場合、「リモートバックアップが存在しない」として処理され、初回同期時や空のディレクトリでの誤った失敗報告が減少します。

場所のヒント:

  • ホスティングサイトタイプの切り替え：「設定 → 自社サイト管理」。
  • チャンネル移行：「設定 → チャンネル管理」ページのトップにある チャンネル移行。
  • CC Switch エクスポート：「設定 → キー管理」または「設定 → API 認証情報」のエクスポート入口。
  • テーマ / 言語のクイックコントロール：設定ページ右上の部分。初回言語選択は、最初に拡張機能を開いた際のウェルカム / 権限ガイドポップアップに表示されます。
  • 直近の API 検証結果：関連する検証ダイアログ、およびサポートされている API 認証情報 / モデル検証インターフェースで確認できます。

    • ブラウザ言語：繁体字中国語などのブラウザ言語環境との互換性を補完し、デフォルト言語のフォールバックロジックを調整しました。初回起動時の言語認識がより安定しました。
    • ドキュメントリンク：多言語ドキュメントのジャンプルールを最適化しました。サポートされていない言語環境では、自動的に英語ドキュメントにフォールバックし、誤った言語ページへのジャンプを減らします。
    • 多言語対応：一部の翻訳値の取得方法を統一し、不要なテキストをクリーンアップして、多言語体験を最適化しました。
  • 問題修正:
    • サイドバー：Chrome/Edge（MV3）でツールバークリック後にサイドバーが偶発的に開かない問題を修正しました。
    • キー管理：マスクされたキーを表示する際に、新しいロード状態を追加し、完全なキーの解析表示をサポートしました。再度展開しても、繰り返しリクエストされません。
    • ブラウザバックグラウンド：拡張機能が一時停止した際に、一時ページをより迅速にクリーンアップし、一時ページの残留問題を減らします。

場所のヒント:

  • 手動サインインページの一括オープン：「設定 → 自動サインイン」ページの失敗したアカウント関連操作。
  • コミュニティ入口：拡張機能ポップアップ右上角の フィードバック メニュー、および「設定 → バージョン情報」の フィードバックとサポート セクション。
  • キーの表示：「設定 → キー管理」のキーリストで表示/非表示ボタンをクリック。

  • 新機能:
    • API 認証情報：CLI 互換性検証 操作を追加しました。検証時にモデル ID を自動取得または手動入力でき、現在一時的な API タイプ オーバーライドを使用しているかどうかを明確に表示し、使い捨てテスト結果を保存済み設定と誤認するのを防ぎます。

    • キー管理：キー管理 にホスティングサイトのチャンネル状態、マッチングシグナル、ジャンプ可能な入口が表示されるようになりました。また、キーを API 認証情報 に保存する際に、より明確な名前が生成され、後続の検索と再利用が容易になります。
    • New API ホスティングサイト：ホスティングサイト設定にログイン補助情報（ユーザー名、パスワード、オプションの TOTP）とセッション検証を追加しました。状態を検証したり、実際のチャンネルキーを読み取る必要がある場合、拡張機能内で直接検証を完了できます。
    • ホスティングサイトマッチング：チャンネル識別を URL、キー、モデルなどのシグナルを総合してソートするように変更しました。バックエンドがマスクされたトークンのみを返すシナリオでも、チャンネル状態の判断、コピー、統合操作を継続して完了できます。
    • 初回使用：ウェルカム / 権限ガイドポップアップに言語セレクターを追加しました。初回起動時にインターフェース言語を切り替え、後続の使用設定を記憶します。
  • 体験最適化:
    • Veloera：現在 Base URL に基づくチャンネルの特定と状態検出をサポートしていないシナリオに対して、関連する入口は自動的に非表示または無効になり、説明が表示されるため、誤ってクリックして結果がない場合の混乱を減らします。
  • 問題修正:
    • 言語：起動時にブラウザ検出言語に安定して従わない問題を修正し、インターフェーステキストと日付/時刻のローカライズを同期して補正しました。
    • 権限ガイド：権限説明ポップアップのボタンレイアウトを最適化しました。小ウィンドウやボタンの折り返し時により整然とし、クリックしやすくなりました。

場所のヒント:

  • API 認証情報：「設定 → API 認証情報」で、CLI 互換性検証、モデル管理で開く などの操作を使用できます。
  • モデルリストデータソース：「設定 → モデルリスト」のトップにあるデータソース選択エリアで、API 認証情報 に切り替えることができます。
  • ホスティングサイトチャンネル状態：「設定 → キー管理」で各キーのホスティングサイト状態とマッチングヒントを確認します。
  • New API ホスティングサイトログイン補助：「設定 → 自社サイト管理」の New API 統合設定 エリア。
  • 初回言語選択：最初に拡張機能を開いた際に表示されるウェルカム / 権限ガイドポップアップ。

3.26.0

    • API 認証情報：API 認証情報 ページで、設定ナビゲーションと拡張機能ポップアップから直接アクセスできるようになりました。設定をエクスポートする際にも token のメモを保持し、複数のツール間での移行を容易にします。
    • WebDAV：同期データ選択を追加しました。必要に応じて アカウント、ブックマーク、API 認証情報 などの共有データのみを同期でき、複数デバイス間での不要な上書きを減らします。
    • Sub2API：Sub2API アカウントのキー管理をサポートし、キーの表示、作成、編集、削除を直接行えるようになりました。
    • CLIProxyAPI：インポート時に Provider タイプ選択を追加し、一般的な endpoint アドレスを自動的に正規化することで、手動での URL 変更を減らします。
  • 体験最適化:
    • 交換アシスタント：交換成功後にアカウント残高が自動的に更新され、手動更新で結果を確認する手順を減らしました。
  • 問題修正:
    • 自動サインイン：固定時間サインインに、より安定した補完実行メカニズムを追加し、拡張機能の更新などによる実行ウィンドウのスキップによるサインイン漏れを減らします。
    • 自動認識：アカウントの再認識後にカスタムサインイン設定が失われる問題を修正し、設定の意図しない消失を防ぎます。
    • 自動サインイン：Turnstile アシストまたは手動サインインヒントが一部のアカウントで誤って 外部サインインアドレス を使用する問題を修正しました。代わりに、常にサイトのデフォルトサインインページを開くようにし、誤ったページへのジャンプやサインイン完了不可を減らします。
    • ホスティングサイト：ホスティングサイトへのデータインポートまたは同期時に、ターゲットサイトのデフォルトグループを優先的に使用し、グループの不一致による異常を減らします。

注意

  • WebDAV の 同期データ選択 とアカウントの自動更新などのローカルデバイス設定は、WebDAV を介してデバイス間で上書きされなくなりました。

場所のヒント:

  • 重複アカウントリマインダー：「設定 → 基本設定 → アカウント管理」の 重複アカウント追加前のリマインダー。

  • チャンネル特定：「設定 → アカウント管理」の個々のアカウントの操作メニュー。

重要：自動リフレッシュ設定は強制的に調整されます

フィードバックによると、短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける可能性があります。

v3.15.0 では、自動リフレッシュ設定を強制的に変更しました：

  • 自動リフレッシュとプラグインを開いた際の自動リフレッシュ機能をオフにしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
  • リフレッシュ間隔 の最低値を 60 秒、最小リフレッシュ間隔保護 の最低値を 30 秒にしました。アップグレード前に上記閾値を下回る設定をしていた場合、アップグレード後に最低値に引き上げられます。以前の設定値が新しい合法範囲内であれば、影響はありません。

場所のヒント:

  • ブックマーク管理：「設定 → ブックマーク管理」；ポップアップ上部で「アカウント / ブックマーク」を切り替えます。
  • 自動リフレッシュ：「設定 → 基本設定 → 自動リフレッシュ」。
