更新ログ

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録します。完全な履歴バージョンと詳細な技術変更については、GitHub Releases を参照してください。

新規ユーザー向けガイド

  • 現在のバージョンを確認する方法：拡張機能のポップアップを開き、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
  • このページが自動的に開かないようにする方法：「設定 → 一般 → 更新ログ」で、「更新後に更新ログを自動的に開く」を制御できます。
  • 異常のトラブルシューティング：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて Issues までお送りください。

3.23.0

  • 新機能:
    • 自動サインイン：アカウント操作メニューに クイックサインイン を追加しました。これにより、単一のアカウントに対して即座にサインインを実行し、完了後にステータスを更新できます。
    • キー管理：すべてのアカウント ビューを追加しました。アカウントごとにキーを集約表示し、アカウントを横断した検索とコピーを容易にします。
    • モデルリダイレクト：モデルリダイレクトマッピングをクリア の一括操作を追加しました。チャネルを選択し、再確認後に model_mapping を迅速にリセットできます（取り消し不可）。
  • 体験最適化:
    • New API チャネル管理：チャネルリストの URL がクリック可能になり、検索体験を最適化しました。
  • 問題修正:
    • チャネル管理：チャネルダイアログの 優先度 のヒント文言の不正確さを修正しました。
    • モデルリダイレクト：自動生成マッピングに「バージョンガード」を追加し、クロスバージョンでの誤ったマッチングを防ぎます。
    • サイドバー：実行環境がサイドバーをサポートしていない場合、ポップアップ/設定ページを開くように自動フォールバックし、クリックしても応答がない問題を回避します。

3.22.0

  • 新機能:
    • モデルリスト：モデル対応キー ツール（鍵アイコン）を追加しました。現在利用可能なキーがモデルに存在するかどうかを確認できます。利用可能なキーがない場合、モデルの利用可能なグループに基づいてデフォルトキーをワンクリックで作成するか、カスタム作成フローに進むことができます。キーのワンクリックコピーもサポートしています。
    • スナップショット共有：概要スナップショット / アカウントスナップショット をワンクリックで共有できるようになりました。優先的に画像をクリップボードにコピーし、サポートされていない場合は PNG を自動ダウンロードします。スナップショットには共有可能な情報のみが含まれ（API Key などの機密フィールドは含まれません）、タイトル文言のワンクリックコピーも可能です。
  • 体験最適化:
    • アカウント無効化：残高履歴 / 使用量分析 / 使用量同期 などの更新および定期タスクで、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーを削減します。
  • 問題修正:
    • ボタン：ボタンが「読み込み中」で左側アイコンも表示されている場合にスピナーが見えなくなる問題を修正しました。

場所のヒント:

重要：自動リフレッシュ設定は強制的に調整されます

フィードバックによると、短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかけるため、

v3.15.0 では自動リフレッシュ設定を強制的に変更しました：

  • 自動リフレッシュおよびプラグインを開いたときの自動リフレッシュ機能をオフにしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
  • リフレッシュ間隔 は最低 60 秒、最小リフレッシュ間隔保護 は最低 30 秒です。アップグレード前に設定した値が上記閾値を下回っていた場合、アップグレード後に自動的に最低値に引き上げられます。以前設定した値が新しい合法範囲内であれば、影響はありません。

場所のヒント:

  • ブックマーク管理：「設定 → ブックマーク管理」；ポップアップ上部で「アカウント / ブックマーク」を切り替えます。
  • 自動リフレッシュ：「設定 → 基本設定 → 自動リフレッシュ」。
