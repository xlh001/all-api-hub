更新履歴

3.2.0

  • 新機能:
    • 「モデルリスト」ページに「インターフェース可用性検出」（Beta）を追加。これにより、指定されたモデルで現在のキーが利用可能かどうか（例：テキスト生成、ツール/関数呼び出し、構造化出力（JSON構造での返却）、インターネット検索（Grounding）などの検出項目）を迅速に確認できます。
    • 「モデルリスト」ページに「CLIツール互換性検出」（Beta）を追加。Claude Code / Codex CLI / Gemini CLI のツール呼び出しプロセスをシミュレートし、これらのツールにおけるインターフェースの互換性を評価します。
    • 「概要」ページに「評価とダウンロード」を追加：現在のストアソース（Chrome / Edge / Firefox）を自動的に識別し、ワンクリックでの評価と他のストアからのダウンロードエントリを提供します。
  • 問題の修正:
    • サイトの更新時にHTTPエラーが発生した場合、健全性ステータスにステータスコードとエラー原因が表示され、問題の特定が容易になります。
    • サイドバーモードでは、「サイドバーで開く」ボタンが表示されなくなり、重複して開くことを防ぎます。

3.1.1

  • 問題の修正:
    • サイトアクセス権限を拡大し（host_permissions: <all_urls>）、権限不足によるCookie取得失敗を減らしました。

    • CLIProxyAPIのエクスポート時に、プロキシ、モデル、およびモデルエイリアスリストを設定できるようになりました。
    • サイトチェックインとカスタムチェックインのロジックを分離し、互いに影響を与えないようにしました。
  • 問題の修正:
    • New-API サイトの手動チェックイン時のウェブページへのジャンプパスエラーを修正しました。
