更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の最適化 / 不具合修正）を記録しています。完全な履歴バージョンとより詳細な技術的な変更については、GitHub Releases をご覧ください。

初めての方へ

  • 現在のバージョンの確認方法：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。または、設定ページでも確認できます。
  • 更新後に本ページを自動で開く設定：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動で開く」かどうかを制御できます。
  • 異常のトラブルシューティング：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて Issues に報告してください。

3.16.0

  • 新機能:
    • Sub2API（JWTサイト）：新しいSub2APIサイトタイプを追加しました。残高/クォータの照会をサポートします。「自動識別」によるコンソールログイン状態の読み取りをサポートします。また、「プラグインホスト型セッション（マルチアカウント、推奨）」モードをサポートし、アカウントごとに認証を個別に更新できるため、同一サイトでのマルチアカウント体験が向上します。
    • 表示設定：「本日の収支表示」スイッチ（デフォルトでオン）を追加しました。これにより、「本日の消費/収入」などの統計情報を非表示にし、取得を停止することで、更新時のログ取得リクエストを削減します。
  • 注意:
    • Sub2APIは、現時点ではサイトのチェックイン/本日の利用量/収入などの関連機能をサポートしておらず、基本的な残高/クォータの照会のみを提供します。関連機能は、サイトの能力に応じて順次改善される予定です。
    • 「プラグインホスト型セッション（マルチアカウント）」では、refresh_token がアカウントのプライベートな認証情報として保存され、エクスポート/WebDAVバックアップに含まれます。バックアップファイルとWebDAV認証情報は適切に管理してください。

位置のヒント：

  • Sub2APIの追加/モードの説明：「設定 → アカウント管理」でアカウントを新規追加/編集し、サイトタイプで Sub2API を選択します。詳細な手順については、FAQ（「Sub2API」で検索）を参照してください。
  • 本日の収支スイッチ：「設定 → 基本設定 → 表示設定」にあります。

3.15.1

  • 不具合修正:
    • Chrome/Edge（MV3）：バックグラウンドのService Workerの安定性を向上させ、非同期の定期タスク（WebDAV自動同期 / 利用量同期 / モデル同期 / 自動チェックインなど）がバックグラウンドの早期終了により実行漏れする問題を軽減しました。また、ブラウザ再起動後に自動的に関連する定期タスクを復元します。
    • インストール/更新/起動プロセス：初期化段階でのエラー処理とログを強化し、予期せぬクラッシュを防ぎます。

3.15.0

  • 新機能:
    • ブックマーク管理：「ブックマーク管理」を追加しました。サイトのコンソール/ドキュメント/管理ページなどのショートカットリンクを保存するために使用でき、完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータは、バックアップ/復元およびWebDAV自動同期に含まれます。
  • 不具合修正:
    • アカウント更新：重複する「本日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部のサイトでは、更新インターフェースで既に today_income が返されています）。
    • 自動更新：更新間隔の最低値を60秒、最小更新間隔保護の最低値を30秒に設定しました。更新後、古い設定は自動的に合法的な範囲に修正され、関連するヒントの文言とドキュメントが改善されました。

重要：自動更新設定が強制的に調整されます

短すぎる自動更新間隔は、サイトのレート制限をトリガーしやすく、サイトに過度な負荷をかけるというフィードバックに基づき、

v3.15.0 では、自動更新設定に強制的な変更を加えました：

  • 自動更新機能とプラグインを開いたときの更新機能がオフになりました。引き続き有効にする必要がある場合は、手動で再有効化する必要があります。
  • 更新間隔 は最低60秒、最小更新間隔保護 は最低30秒です。アップグレード前にこれらのしきい値を下回る設定をしていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい合法的な範囲内にある場合は、影響はありません。

位置のヒント：

  • ブックマーク管理：「設定 → ブックマーク管理」にあります。ポップアップの上部で「アカウント / ブックマーク」を切り替えられます。
  • 自動更新：「設定 → 基本設定 → 自動更新」にあります。

3.14.0

  • 新機能:
    • Web AI API 機能の可用性テスト（ベータ版）：右クリックメニューに「AI API の機能可用性をクイックテスト」を追加しました。現在のウェブページで直接テストパネルを開くことができます。Base URL と API Key の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Google などのインターフェースの基本能力検出を実行します（OpenAI互換では、モデルリストの一括取得も可能です）。
    • （オプション）自動検出：「設定 → AI API テスト」で有効にし、URLホワイトリストを設定できます。ホワイトリスト内のページで利用可能な Base URL と API Key が検出された場合、確認プロンプトが表示され、確認後にのみテストパネルが開きます（デフォルトではオフであり、Keyは保存されません）。
    • 自動チェックイン：実行結果リストに、より多くのトラブルシューティングのヒントを追加しました。これには、「一時的なCloudflare回避タブが手動で閉じられた」「Access Tokenが無効」などの一般的な異常に対する処理の提案とドキュメントへの入り口が含まれます。
  • 不具合修正:
    • WebDAV：自動同期をタイマーからブラウザのAlarms APIに移行し、バックグラウンドのスリープ/省電力ポリシーによる同期漏れの可能性を低減しました。

位置のヒント：

  • AI API テストパネル：任意のウェブページで右クリックメニューから「AI API の機能可用性をクイックテスト」を選択します。自動検出に関する設定は「設定 → AI API テスト」にあります。
  • 自動チェックインのヒント：「設定 → 自動チェックイン」の実行結果リストで確認できます。

3.13.0

  • 新機能:
    • アカウント管理：「チェックインステータスの期限切れ」のヒントを追加しました。「本日チェックイン済み/未チェックイン」のステータスが本日検出されたものでない場合、オレンジ色の警告アイコンが表示されます。クリックすると、そのアカウントのデータをワンクリックで更新でき、古いステータスによる誤解を防ぎます。
    • インターフェース：複数選択コントロールをよりコンパクトなセレクターにアップグレードしました（スペースを節約し、検索とより明確な選択済み表示をサポートします）。
  • 不具合修正:
    • Veloera：アカウントデータ更新とチェックインロジックを修正し、可用性を向上させました。
    • Cookie認証：Cookieキャッシュメカニズムを削除し、Cookie更新後に古い値が読み取られることによる異常を軽減しました。

位置のヒント：

  • チェックインステータスの期限切れのヒント：「設定 → アカウント管理」のアカウントリストで、サイト情報の右側にあるチェックインアイコンの場所にあります。

3.12.0

  • 新機能:

  • 不具合修正:
    • アカウント管理：サイト名などのテキストが長すぎるためにレイアウトがオーバーフローする問題を修正し、自動的に切り詰めて表示するように変更しました。
    • ドロップダウンセレクター：空の状態のヒントを最適化し、オプションテキストが長すぎる場合のオーバーフローの問題を修正しました。

位置のヒント：

  • Kilo Codeへのエクスポート：「設定 → 秘密鍵管理」の秘密鍵リストで、特定の秘密鍵の右上にあるKilo Codeアイコンをクリックします。

3.11.0

  • 不具合修正:
    • アカウント管理：サイト名などのテキストが長すぎるためにレイアウトがオーバーフローする問題を修正し、自動的に切り詰めて表示するように変更しました。

3.10.0

  • 新機能:
    • アカウント管理：シークレットモードでサイトリンクをクリックすると、現在のシークレットウィンドウ内で開くようになりました（シークレットログイン状態を維持しやすくなります）。
    • アカウント管理：無効化されたアカウントでもサイトリンクをクリックできるようになり、ブックマークとして使用しやすくなりました。
    • 利用状況分析：アカウントが1つしかない場合、グラフとリストはユーザー名ではなくサイト名を優先的に表示し、情報がより直感的になりました。
    • Cloudflare回避アシスタント：一時的なCloudflare回避ウィンドウにCAP（cap.js）Pow検証のサポートを追加し、通過率を向上させました。
  • 不具合修正:
    • 兌換アシスタント：クリップボードから兌換コードを優先的に読み取り、トリガーの正確性を向上させました。
    • ヒントオーバーレイ：Toasterのレイヤー順序の問題を修正し、ヒントがウェブページに隠れなくなりました。

位置のヒント：

  • サイトリンク：「設定 → アカウント管理」のアカウントリストでサイト名をクリックします。
  • Cloudflare回避アシスタント：Cloudflare回避アシスタント を参照してください。

3.9.0

  • 新機能:
    • アカウント管理：アカウントに「手動残高（USD）」フィールドを追加しました。サイトが残高/クォータを自動的に取得できない場合、表示と統計のために手動で入力できます。
    • アカウント管理：アカウントに「合計残高から除外」スイッチを追加しました。特定のカウントを「合計残高」統計から除外するために使用されます（更新/チェックインなどの機能には影響しません）。
    • 設定：「更新後に更新履歴を自動で開く」スイッチを追加しました（更新後に本ページが自動で開く動作をオフにできます）。
    • 設定：ログ設定を追加しました。コンソールログを出力するかどうか、および最小ログレベルを制御でき、問題のトラブルシューティングに役立ちます。
    • 自動チェックイン：実行完了後、関連データを更新し、インターフェースを同期的に更新します。

位置のヒント：

  • アカウントの新規追加/編集：「設定 → アカウント管理」でアカウントの新規追加/編集を開きます。
  • 更新履歴スイッチ、ログ設定：「設定 → 一般」で設定します。

3.8.0

  • 新機能:
    • 利用状況分析：「利用状況分析」ページを追加しました。複数のサイト、複数のアカウントの利用傾向をグラフ化し、「どこで多く使っているか / 多く費用がかかっているか / 遅くなっているか」を一目で把握できるようにし、コスト管理、照合、問題のトラブルシューティングに役立ちます。
    • ダッシュボードの内容：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、使用時間ヒートマップ、および遅延傾向/ヒストグラムなどを表示できます。
    • 利用履歴同期：「利用履歴同期」機能を追加しました。「集計された利用量データ」を取得して保存するために使用されます（元のログは保存されません）。保持日数、自動同期方法、最小同期間隔を設定でき、「同期ステータス」で各アカウントの同期結果とエラーヒントを確認できます。
  • 使用方法:
    • まず「設定 → アカウント利用量」で「利用履歴同期」を有効にし、必要に応じて設定した後、「今すぐ同期」をクリックします。
    • 次に、左側のメニュー「利用状況分析」でグラフを確認します。保持または照合が必要な場合は、「エクスポート」をクリックできます。

3.7.0

  • 新機能:
    • 並べ替え：アカウントリストが「収入」による並べ替えをサポートしました。並べ替えの優先順位に「無効なアカウントを最下部に」を追加し、停止中/無効なアカウントが日常の使用を妨げないようにしました。
    • 自動チェックイン：「インターフェースを開いたときに本日のチェックインを事前にトリガー」を追加しました。時間枠内にポップアップ/サイドバー/設定ページを開くと、計画された時間を待たずに、本日のチェックインを一度事前に実行しようとします。
  • 不具合修正:
    • 自動チェックイン：同一アカウントは1日1回のみ実行されます。リトライは失敗したアカウントに対してのみ行われ、無意味なリクエストと重複した通知を減らします。
  • 位置のヒント:
    • 並べ替えの優先順位：「設定 → アカウント管理」で調整します。
    • 自動チェックインの事前トリガー/リトライ：左側のメニュー「自動チェックイン」で設定します。

3.4.0

  • 不具合修正:

3.3.0

3.2.0

  • 新機能:
    • 「モデルリスト」ページに「インターフェース可用性検出」（ベータ版）を追加しました。特定のモデルで現在の秘密鍵が利用可能かどうかを迅速に確認するために使用されます（例：テキスト生成、ツール/関数呼び出し、構造化出力（JSON構造で返す）、ネットワーキング検索（Grounding）などの検出項目）。

    • 「概要」ページに「評価とダウンロード」を追加しました。現在のストアソース（Chrome / Edge / Firefox）を自動的に識別し、ワンクリック評価と他のストアからのダウンロードエントリを提供します。
  • 不具合修正:
    • サイトの更新がHTTPエラーに遭遇した場合、健全性ステータスにステータスコードとエラー原因が表示され、問題の特定に役立ちます。
