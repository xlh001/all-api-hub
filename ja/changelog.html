更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の最適化 / 問題の修正）を記録しています。完全な履歴バージョンとより詳細な技術的変更については、GitHub Releases をご覧ください。

新規ユーザーの方へ

  • 現在のバージョンを確認する方法：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。または設定ページでも確認できます。
  • このページを自動で開かないようにする方法：「設定 → 一般 → 更新履歴」で「更新後に更新履歴を自動で開く」を制御できます。
  • エラーのトラブルシューティング：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を Issues に添付してください。

3.20.0

  • 体験の最適化:
    • キー管理：新規キー追加時のグループドロップダウンオプションに、グループ ID と説明が同時に表示されるようになり、複数のグループ/ルート間で迅速に区別し、選択しやすくなりました。
  • 問題の修正:
    • アカウント管理：アカウント新規追加時の「アカウント追加後にデフォルトキーを自動作成する」はデフォルトでオフに変更されました。アカウント追加後にデフォルトキーを自動生成したい場合は、設定で手動で有効にしてください。

場所のヒント：

  • グループ ID の表示：「設定 → キー管理」で「新規キー追加」をクリックし、グループドロップダウンオプションで確認。
  • デフォルトキー自動作成の切り替え：「設定 → 基本設定 → アカウント管理 → API キー」。

3.19.0

  • 新機能:
    • セルフホストサイト管理：Octopus ホストサイトのサポートを追加しました。Octopus のバックエンドに接続し、「チャネル管理」でアカウント API キーをチャネルとしてインポートできます。また、利用可能なモデルリストの取得もサポートしています。
    • キー管理：「アカウント追加後にデフォルトキーを自動作成する」（デフォルトで有効）を追加し、「少なくとも1つのキーを確保」機能を提供しました。これにより、キーが不足しているアカウントに対してデフォルトキーを自動で一括補完できます。
    • AI API テスト：インターフェース検証の「モデルリスト検出」が、OpenAI/OpenAI-compatible、Anthropic、Google/Gemini などのインターフェースタイプをサポートし、利用可能なモデル ID の提案を行うため、手動でモデルを推測する手間が減ります。
  • 体験の最適化:
    • アカウント管理：サイト/アカウントの識別ロジックを強化し、同一サイト複数アカウントのシナリオでの安定性を改善しました。
    • 利用状況/ログ取得：ログ系インターフェースにレート制限保護を追加し、頻繁な更新によるエラーやサイトのレート制限トリガーを減らしました。
    • チャネル管理：チャネル作成時により正確な重複検出を行い、確認のプロンプトを表示することで、重複したルートの誤作成を防ぎます。
  • 問題の修正:
    • アカウントの無効化：キー管理などの関連ドロップダウン/リストで、無効化されたアカウントを自動的にフィルタリングし、無効な操作を回避します。
    • 言語：拡張機能の言語設定がウェブページ自体の言語値に影響を与える可能性がある問題を修正しました。

場所のヒント：

  • Octopus の設定：「設定 → 基本設定 → セルフホストサイト管理」で Octopus を選択し、ベース URL / ユーザー名 / パスワードを入力。
  • チャネル管理の入り口：「設定 → チャネル管理」。
  • デフォルトキー自動作成の切り替え：「設定 → 基本設定 → アカウント管理 → API キー」。
  • デフォルトキーの一括補完：「設定 → キー管理」の右上にある「少なくとも1つのキーを確保」。
  • AI API テストの入り口：ウェブページの右クリックメニュー「AI API の機能可用性をクイックテスト」。

    • タブページ：「設定」グループタグと「モデルリスト」ベンダータグに左右スクロールボタンを追加し、狭いウィンドウでの切り替えがより便利になりました。
  • 問題の修正:
    • アカウント管理：サイトタイプ「自動識別」の精度が向上し、最近のバージョンで頻繁に発生していた不明なサイトタイプの問題を修正しました。

場所のヒント：

  • 残高履歴の通貨単位：「設定 → 残高履歴」ページのフィルタリングエリア「通貨単位」。
  • アカウントの換算レート（チャージ金額レート）：「設定 → アカウント管理」の新規/編集アカウントフォーム「チャージ金額レート」。

  • 体験の最適化:
    • チャネル管理：「チャネル管理」ツールバーの、小さな画面/狭いウィンドウでのレスポンシブレイアウトと使いやすさを最適化しました。
  • 問題の修正:
    • インポート/エクスポート：エクスポートエリアが一部の画面サイズでレスポンシブ表示されない問題を修正しました。

場所のヒント：

  • 残高履歴グラフの入り口：「設定 → 残高履歴」。

3.16.0

  • 新機能:
    • Sub2API（JWT サイト）：Sub2API サイトタイプを追加し、残高/クォータの照会をサポートします。「自動識別」によるコンソールログイン状態の読み取りをサポートします。また、「プラグインホストセッション（複数アカウント、推奨）」モードをサポートし、アカウントごとに認証を個別に更新できるため、同一サイト複数アカウントのエクスペリエンスが向上します。
    • 表示設定：「今日の収支を表示」スイッチ（デフォルトで有効）を追加しました。これにより、「今日の消費/収入」などの統計を非表示にし、取得を停止することで、更新時のログ取得リクエストを減らすことができます。
  • 注意:
    • Sub2API は、サイトのチェックイン/今日の利用状況/収入などの関連機能を一時的にサポートしていません。基本的な残高/クォータの照会のみを提供します。今後はサイトの機能に応じて関連機能を順次充実させていく予定です。
    • 「プラグインホストセッション（複数アカウント）」では、refresh_token がアカウントのプライベートな認証情報として保存され、エクスポート/WebDAV バックアップにも含まれます。バックアップファイルと WebDAV 認証情報は適切に保管してください。

場所のヒント：

  • Sub2API の追加/モードの説明：「設定 → アカウント管理」で新規/編集アカウントを開き、サイトタイプで Sub2API を選択します。より詳細な手順は FAQ（「Sub2API」で検索）を参照してください。
  • 今日の収支スイッチ：「設定 → 基本設定 → 表示設定」。

3.15.1

  • 問題の修正:
    • Chrome/Edge（MV3）：バックグラウンドの Service Worker の安定性を向上させ、非同期タイマー処理（WebDAV 自動同期 / 利用状況同期 / モデル同期 / 自動チェックインなど）がバックグラウンドの早期終了により実行漏れする問題を軽減しました。また、ブラウザ再起動後に自動的に関連するタイマー処理を回復します。
    • インストール/更新/起動プロセス：初期化段階でのエラー処理とログを強化し、予期せぬクラッシュを防ぎます。

3.15.0

  • 新機能:
    • ブックマーク管理：サイトのコンソール/ドキュメント/管理ページなどのショートカットリンクを保存するための「ブックマーク管理」を追加しました。完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータは、バックアップ/復元および WebDAV 自動同期に含まれます。
  • 問題の修正:
    • アカウント更新：重複する「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを減らしました（一部のサイトでは、更新インターフェースで既に today_income が返されています）。
    • 自動更新：更新間隔の最低値を 60 秒、最小更新間隔保護の最低値を 30 秒としました。更新後、古い設定は自動的に有効な範囲に修正され、関連するヒントの文言とドキュメントが改善されました。

重要：自動更新設定が強制的に調整されます

自動更新間隔が短すぎると、サイトのレート制限をトリガーしやすく、サイトに過大な負荷をかけるというフィードバックがあったため

v3.15.0 では、自動更新設定を強制的に変更しました：

  • 自動更新とプラグインを開いたときの更新機能がオフになりました。引き続き有効にする必要がある場合は、手動で再度有効にする必要があります。
  • 更新間隔 は最低 60 秒、最小更新間隔保護 は最低 30 秒です。アップグレード前にこれらのしきい値を下回る設定をしていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい有効な範囲内にある場合は影響ありません。

場所のヒント：

  • ブックマーク管理：「設定 → ブックマーク管理」。ポップアップの上部で「アカウント / ブックマーク」を切り替えられます。
  • 自動更新：「設定 → 基本設定 → 自動更新」。

3.14.0

  • 新機能:
    • ウェブページ AI API 機能可用性テスト（ベータ）：右クリックメニューに「AI API の機能可用性をクイックテスト」を追加しました。現在のウェブページで直接テストパネルを開くことができます。Base URL と API Key の入力/貼り付けをサポートし、OpenAI 互換 / OpenAI / Anthropic / Google などのインターフェースの基本機能検出（OpenAI 互換ではモデルリストの一括取得も可能）を行います。
    • （オプション）自動検出：「設定 → AI API テスト」で有効にし、URL ホワイトリストを設定できます。ホワイトリストのページで利用可能な Base URL と API Key が検出された場合、まず確認プロンプトが表示され、確認後にテストパネルが開きます（デフォルトではオフで、Key は保存されません）。
    • 自動チェックイン：実行結果リストに、より多くのトラブルシューティングのヒントを追加しました。「一時的なシールド解除タブが手動で閉じられた」「Access Token が無効」などの一般的な異常に対する処理の提案とドキュメントの入り口が含まれます。
  • 問題の修正:
    • WebDAV：自動同期をタイマーからブラウザの Alarms API に移行し、バックグラウンドの休止/省エネポリシーによる同期漏れの確率を減らしました。

場所のヒント：

  • AI API テストパネル：任意のウェブページで右クリックメニューから「AI API の機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI API テスト」にあります。
  • 自動チェックインのヒント：「設定 → 自動チェックイン」の実行結果リストで確認。

3.8.0

  • 新機能:
    • 利用状況分析：「利用状況分析」ページを追加しました。複数のサイト、複数のアカウントの利用傾向をグラフ化し、「どこが多く使われているか / 費用がかかっているか / 遅くなっているか」を一目で把握でき、コスト管理、照合、問題のトラブルシューティングに役立ちます。
    • ダッシュボード内容：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、使用時間ヒートマップ、および遅延傾向/ヒストグラムなどを確認できます。
