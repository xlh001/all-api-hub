更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の最適化 / 不具合修正）を記録しています。完全な履歴バージョンとより詳細な技術的な変更については、GitHub Releases をご覧ください。

初めての方へ

  • 現在のバージョンの確認方法：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
  • 更新後に自動で本ページを開く：「設定 → 一般 → 更新履歴」で「更新後に自動で更新履歴を開く」を制御できます。
  • 異常の調査：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて Issues に報告してください。

3.13.0

  • 新機能:
    • アカウント管理: 「サインイン状態の有効期限切れ」プロンプトを追加。—「本日サインイン / 未サインイン」の状態が本日確認されていない場合、オレンジ色の警告アイコンが表示されます。クリックするとワンクリックでアカウントデータを更新でき、古い状態表示による誤解を防ぎます。
    • インターフェース: 複数選択コントロールを、よりコンパクトなセレクターにアップグレード（省スペース、検索対応、選択項目の表示がより明確）。
  • バグ修正:
    • Veloera: アカウントデータの更新およびサインインロジックを修正し、使いやすさを向上。
    • Cookie認証: Cookie更新後も古い値を読み続けてしまうことによる例外を減らすため、Cookieキャッシュ機構を削除。

場所のヒント:

  • サインイン状態の有効期限切れプロンプト: 「設定 → アカウント管理」のアカウント一覧内、サイト情報のサインインアイコンの横。

3.12.0

  • 新機能:
    • キー管理: 「Kilo Code にエクスポート」を追加。— Kilo Code / Roo Code 用の providerProfiles 設定を生成し、apiConfigs スニペットのコピーや、インポート用 settings JSON のダウンロードをサポート（インポートは追加方式で、既存のプロバイダーは削除されません）。
  • バグ修正:
    • アカウント管理: サイト名などの過度に長いテキストによって発生していたレイアウトのはみ出し問題を修正。表示は自動的に省略されます。
    • ドロップダウンセレクター: 空状態のプロンプトを最適化し、オプションテキストが長すぎる場合のはみ出し問題を修正。

場所のヒント:

  • Kilo Code へのエクスポート: 「設定 → キー管理」のキー一覧で、キー右上にある Kilo Code アイコンをクリック。

3.11.0

  • 新機能:
    • New API 渠道管理：新規「重複チャネル」アラートを追加 — 既存の同じ/類似のチャネルが検出された場合、警告ダイアログがポップアップ表示され、作成を続行するかキャンセルするかを選択できます（エラーToastで作成を阻止することはなくなりました）。
  • 問題修复:
    • アカウント管理：サイト名などのテキストが長すぎるためにレイアウトがオーバーフローする問題を修正し、自動的に切り詰めて表示するように変更しました。

3.10.0

  • 新機能:
    • アカウント管理：シークレットモードでサイトリンクをクリックすると、現在のシークレットウィンドウ内で開きます（シークレットログイン状態を維持しやすくなります）。
    • アカウント管理：無効化されたアカウントもサイトリンクのクリックをサポートし、ブックマークとして使用しやすくなります。
    • 用量分析：単一アカウントのみの場合、グラフとリストでサイト名（ユーザー名ではなく）を優先的に表示し、情報がより直感的になります。
    • 過盾アシスタント：一時的な過盾ウィンドウに CAP（cap.js）Pow認証のサポートが追加され、通過率が向上しました。
  • 問題修复:
    • 兌換助手：クリップボードから交換コードを優先的に読み取り、トリガーの精度を向上させました。
    • 提示浮层：Toasterの階層問題を修正し、アラートがウェブページに隠れなくなりました。

位置のヒント：

  • サイトリンク：「設定 → アカウント管理」のアカウントリストでサイト名をクリックします。
  • 過盾アシスタント：Cloudflare 過盾アシスタント を参照してください。

3.9.0

  • 新機能:
    • アカウント管理：アカウントに「手動残高（USD）」フィールドを追加 — サイトが自動で残高/クォータを取得できない場合に、表示および統計のために手動で入力できます。
    • アカウント管理：アカウントに「総残高から除外」スイッチを追加 — 特定のアカウントを「総残高」の統計から除外するために使用されます（更新/チェックインなどの機能には影響しません）。
    • 設定：「更新後に自動で更新履歴を開く」スイッチを追加（更新後にこのページが自動で開く動作をオフにできます）。
    • 設定：ログ設定を追加し、コンソールログの出力有無と最小ログレベルを制御できるようにし、問題の調査を容易にしました。
    • 自動チェックイン：実行完了後、関連データを更新し、インターフェースも同期的に更新されます。

位置のヒント：

  • アカウントの追加/編集：「設定 → アカウント管理」でアカウントの追加/編集を開きます。
  • 更新履歴スイッチ、ログ設定：「設定 → 一般」で設定します。

3.8.0

  • 新機能:
    • 用量分析：「利用状況分析」ページを追加し、複数のサイト、複数のアカウントの利用傾向をグラフ化して、「どこが多く使われているか / 多く支払っているか / 遅くなっているか」を一目で把握できるようにし、コスト管理、照合、問題調査を容易にします。
    • 看板内容：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、使用時間ヒートマップ、遅延傾向/ヒストグラムなどを確認できます。
    • 用量历史同步：「利用履歴の同期」機能を追加しました。これは「集計された利用状況データ」を取得・保存するためのものです（元のログは保存されません）。保持日数、自動同期方法、最小同期間隔を設定でき、「同期ステータス」で各アカウントの同期結果とエラーメッセージを確認できます。
  • 怎么用:
    • まず「設定 → アカウント利用状況」で「利用履歴の同期」を有効にし、必要に応じて設定した後、「今すぐ同期」をクリックします。
    • 次に、左側のメニュー「用量分析」でグラフを確認します。データを保持したり、照合したりする必要がある場合は、「エクスポート」をクリックできます。

3.7.0

  • 新機能:
    • ソート：アカウントリストで「収入」によるソートをサポートしました。ソートの優先順位に「無効アカウントの最下位表示」が追加され、停止/無効なアカウントが日常的な使用を妨げなくなりました。
    • 自動チェックイン：新規「インターフェースを開いたときに今日のチェックインを事前にトリガーする」機能を追加 — 時間枠内でポップアップ/サイドバー/設定ページを開いたときに、自動的に今日のチェックインを一度試行し、予定された時間を待つ必要がなくなります。
  • 問題修复:
    • 自動チェックイン：同一アカウントは1日1回のみ実行されます。リトライは失敗したアカウントに対してのみ行われ、無意味なリクエストや重複した通知を減らします。
  • 位置提示:
    • ソート優先順位：「設定 → アカウント管理」で調整します。
    • 自動チェックインの事前トリガー/リトライ：左側のメニュー「自動チェックイン」で設定します。

3.6.0

  • 新機能:
    • アカウント管理：アカウントの有効化/無効化をワンクリックでサポート。無効化すると、すべて機能がそのアカウントをスキップするため、アカウントが無効になった後もデータを保持しやすくなります。
    • タグ：グローバルタグ管理を追加し、関連するインターフェースと操作を同期的に最適化し、アカウントの分類管理を容易にしました。
    • ポップアップ：タイトルバーに現在のバージョン番号を表示し、この更新履歴への直リンクを提供しました。
    • クイックエクスポート：CCSwitch エクスポートでアップストリームモデルの選択をサポートし、エクスポート設定が実際の使用シナリオにより近くなりました。

  • 問題修复:

3.3.0

    • 外部チェックイン：外部チェックインの一括トリガーをサポートし、個別の操作ステップを削減しました。
    • 自動更新：最小更新間隔の最大値制限を解除し、更新頻度を制御するために、より大きな最小間隔を設定できるようにしました。
  • 問題修复:
    • クリップボードの読み取り：トリガー条件を厳格化し、非コピーシナリオでの誤トリガーを減らしました。
    • 兌換助手：ポップアップアラートの前にすべての交換コードを検証し、無効な交換アラートを減らしました。
    • ストレージ：書き込み操作に書き込みロックを追加し、並行書き込み時のデータ一貫性を向上させました。
    • インターフェース：「モデル名のコピー」に関連するローカライズされた文言を調整しました。

3.2.0

  • 新機能:
    • 「モデルリスト」ページに「インターフェース可用性検出」（ベータ版）を追加しました。これにより、現在の秘密鍵が指定されたモデルで利用可能かどうかを迅速に確認できます（例：テキスト生成、ツール/関数呼び出し、構造化出力（JSON構造での返却）、インターネット検索（Grounding）などの検出項目）。
    • 「モデルリスト」ページに「CLIツール互換性検出」（ベータ版）を追加しました。Claude Code / Codex CLI / Gemini CLI のツール呼び出しプロセスをシミュレートし、これらのツールにおけるインターフェースの互換性を評価します。
    • 「概要」ページに「評価とダウンロード」を追加：現在のストアソース（Chrome / Edge / Firefox）を自動的に識別し、ワンクリック評価と他のストアからのダウンロードエントリを提供します。
  • 問題修复:
    • サイトの更新時にHTTPエラーが発生した場合、健全性ステータスにステータスコードとエラー原因が表示され、問題の特定が容易になります。
    • サイドバーモードでは、「サイドバーで開く」ボタンが表示されなくなり、重複して開くのを防ぎます。
