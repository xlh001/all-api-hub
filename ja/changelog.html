更新履歴

3.3.0

  • 新機能:
    • 自動チェックイン: アカウント識別に「ユーザー名」情報を追加し、複数アカウントのシナリオでのアカウント区別を容易にしました。
    • 外部チェックイン: 外部チェックインの一括トリガーをサポートし、個別の操作ステップを削減しました。
    • 自動更新: 最小更新間隔の最大値制限を撤廃し、更新頻度を制御するために、より大きな最小間隔を設定できるようになりました。
  • バグ修正:
    • クリップボード読み取り: トリガー条件を厳格化し、コピー以外のシナリオでの誤作動を減らしました。
    • 引き換えアシスタント: ポップアップ表示前にすべての引き換えコードを検証し、無効な引き換え通知を減らしました。
    • ストレージ: 書き込み操作に書き込みロックを追加し、並行書き込み時のデータ一貫性を向上させました。
    • UI: 「モデル名をコピー」に関連するローカライズされたテキストを調整しました。

3.2.0

  • 新機能:
    • 「モデルリスト」ページに「API可用性検出」（ベータ版）を追加しました。これは、現在のキーが指定されたモデルで利用可能かどうかを迅速に確認するために使用されます（例：テキスト生成、ツール/関数呼び出し、構造化出力（JSON形式で返す）、ネットワーク検索（Grounding）などの検出項目）。
    • 「モデルリスト」ページに「CLIツール互換性検出」（ベータ版）を追加しました。これは、Claude Code / Codex CLI / Gemini CLIのツール呼び出しプロセスをシミュレートし、これらのツールにおけるAPIの互換性を評価します。
    • 「概要」ページに「評価とダウンロード」を追加しました。これは、現在のストアソース（Chrome / Edge / Firefox）を自動的に識別し、ワンクリック評価と他のストアのダウンロードエントリを提供します。
  • バグ修正:
    • サイト更新時にHTTPエラーが発生した場合、ヘルスステータスにステータスコードとエラー原因が表示され、問題の特定が容易になりました。
    • サイドバーモードでは、「サイドバーで開く」ボタンが表示されなくなり、重複して開くのを防ぎます。

3.1.1

  • バグ修正:
    • サイトアクセス権限を拡大し（host_permissions: <all_urls>）、権限不足によるCookie取得失敗を減らしました。
