ブックマーク管理

サイトコンソール、ドキュメント、管理ページ、交換ページなどのクイックリンクを保存するために使用します。「一時的に完全なアカウントを作成する必要はないが、よく使う入り口をプラグインにまとめておきたい」というシナリオに適しています。

どのようなシナリオに適していますか

  • アカウント情報を入力せずに、サイトコンソールの入り口だけを保存したい場合。
  • 特定のサイトが自動認識しにくいが、管理ページ、ドキュメントページ、またはアクティビティページの入り口を保存しておきたい場合。
  • 複数のバックエンド、ドキュメント、交換ページを頻繁に切り替える必要があり、一元管理したい場合。
  • これらのリンクをアカウント情報と一緒にバックアップおよび同期に含めたい場合（ブラウザのブックマークに散らばらせるのではなく）。

機能概要

  • ブックマークの追加 / 編集 / 削除：名前、リンク、タグ、メモを保存します。
  • クイックオープンとリンクコピー：外部バックエンドへの迅速なジャンプや、他のユーザーへの共有に適しています。
  • ピン留め：重要な入り口を先頭に固定できます。
  • 検索とタグフィルタリング：名前、リンク、メモ、タグで素早く検索できます。
  • ドラッグ＆ドロップでの並べ替え：検索状態でない場合に直接順序を調整できます。
  • ポップアップでのクイック切り替え：拡張機能のポップアップ上部で「アカウント / ブックマーク」を切り替えることができます。

アクセス方法

  1. プラグインの設定ページを開きます。
  2. 左側メニューから ブックマーク を選択します。
  3. ブックマークを追加 をクリックします。

一時的なジャンプであれば、まず入り口をブックマークに保存してから、後で完全なアカウントとして登録するかどうかを決定することもできます。

保存できるもの

ブックマークは任意の URL にすることができます。一般的な例としては以下のようなものがあります。

  • サイトコンソールホームページ
  • 個人プロフィールページ
  • チャージページ / 交換ページ
  • 管理バックエンド
  • 公式ドキュメント
  • 一時的なアクティビティページ

使用方法

1. ブックマークの追加

以下のフィールドを入力してください。

フィールド説明
名前例：「サイトコンソール」「交換入り口」「公式ドキュメント」
リンク完全な URL。例：https://example.com/console
タグオプション；アカウント、API 認証情報庫とグローバルタグを共有します
メモオプション；この入り口の用途や注意事項を記録します

2. 検索とフィルタリング

ブックマークページでは以下の操作が可能です。

  • キーワードでの検索
  • タグでのフィルタリング
  • 検索 + タグの組み合わせで範囲を絞り込む

検索はブックマーク名だけでなく、リンク、メモ、タグ名にもマッチするため、「ドメイン名や用途は覚えているが、ブックマーク名は覚えていない」という状況に適しています。

3. ピン留めと並べ替え

  • ピン留め をクリックすると、よく使う入り口を先頭に固定できます。
  • 検索状態ではなく、タグフィルタリングも無効な場合は、直接ドラッグ＆ドロップで並べ替えられます。
  • 検索またはタグフィルタリング状態に入ると、ドラッグ＆ドロップは一時的に無効になります。これは、フィルタリングされたビューを全体の並べ替えと誤解するのを防ぐためです。

4. ポップアップでの使用

拡張機能のポップアップ上部では、アカウント と ブックマーク を切り替えることができます。

  • アカウント：残高の更新、モデルの確認、サインインなど、アカウントレベルの操作に適しています。
  • ブックマーク：コンソール、ドキュメント、管理ページなどのリンクを素早く開くのに適しています。

これは、「管理入り口が必要なサイトもあるが、常にアカウントデータを更新する必要はない」というユーザーにとって特に便利です。

使用上の推奨事項

  • タグで分類する：例：「ドキュメント」「コンソール」「チャージ」「アクティビティ」「本番環境」。
  • サイトの説明をメモに記載する：例：「メインサイトにログインしてからアクセス」「管理者のみ表示可能」「月末アクティビティ交換入り口」。
  • よく使う入り口をピン留めする：例：チャージページ、交換ページ、メインコンソール。
  • まずブックマークに保存し、後でアカウントを補完する：一時的に互換性がない、または検証が必要なサイトに遭遇した場合、これが最も手間がかかりません。

ブラウザのブックマークとの違い

ブックマーク管理 は、通常のブラウザのブックマークと比較して、プラグインのワークフローに重点を置いています。

  • アカウントや API 認証情報庫とタグシステムを共有します。
  • 拡張機能のポップアップに直接表示されます。
  • プラグインのデータインポート/エクスポートに含まれます。
  • WebDAV と連携してマルチデバイス同期が可能です。

よくある質問

質問説明
ブックマークとアカウントの違いは何ですか？アカウントには認証情報とデータ更新機能が含まれます。ブックマークは単なるショートカットリンクであり、残高、モデル、またはサインイン状態は更新されません。
ブックマークはバックアップに含まれますか？はい。ブックマークは共有データの一部であり、インポート/エクスポートおよび WebDAV 同期と一緒に移行できます。
検索時にドラッグ＆ドロップで並べ替えられないのはなぜですか？検索 / タグフィルタリングは結果のサブセットのみを表示します。直接ドラッグ＆ドロップすると、グローバルな順序を誤って変更する可能性があるため、この状態ではドラッグ＆ドロップが無効になります。
アカウントがなくてもサイトの入り口を保存できますか？はい、これはブックマーク管理が最も適した用途の 1 つです。
