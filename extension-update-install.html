安装渠道与更新说明

All API Hub 有两类常见安装方式：浏览器商店版 和 GitHub 手动安装版。如果你不确定该选哪个，优先使用浏览器商店版。

简单结论

Chrome、Edge、Firefox 用户优先安装对应商店版。商店版会由浏览器自动更新，最省心。只有商店还没更新、你的浏览器装不了商店版，或需要临时验证问题时，再考虑从 GitHub 手动安装。

我该安装哪个版本？

你的情况建议选择更新方式
使用 ChromeChrome Web Store 版本浏览器自动更新，也可以在插件内手动检查
使用 EdgeEdge Add-ons 版本浏览器自动更新，也可以在插件内手动检查
使用 FirefoxFirefox Add-ons 版本Firefox 自动更新
使用 Brave / Vivaldi / Opera优先尝试 Chrome Web Store；不可用时再用 GitHub Chrome 安装包商店版自动更新；手动版需重新下载
使用 QQ / 360 / 猎豹等浏览器浏览器内置扩展中心若可用则优先使用；否则用 GitHub Chrome 安装包取决于安装来源；手动版需重新下载
商店版还没收到某个紧急修复临时使用 GitHub Stable需要以后手动下载新版
想提前体验开发中的功能Nightly / 开发版可能不稳定，只建议测试使用

如果你使用 QQ、360、猎豹、Brave、Vivaldi、Opera 等浏览器，请查看 其他浏览器安装指南。其中 Brave、Vivaldi、Opera 通常可以先尝试 Chrome Web Store；需要手动加载时再下载 GitHub Chrome 安装包。

为什么 GitHub 有新版，商店还没更新？

这是正常现象。项目通常会先把新版本发布到 GitHub，然后再提交到各浏览器商店。商店还需要审核和推送，所以可能会晚一些。

常见原因包括：

  • 商店还在审核新版本。
  • 商店已经通过审核，但还没有推送到你的浏览器。
  • 你的浏览器还没来得及检查扩展更新。
  • 公司电脑、浏览器策略或网络环境限制了扩展更新。

如果你用的是商店版，通常等浏览器自动更新即可。急需某个修复时，可以先看下方的手动检查方法。

在插件里检查更新

打开 All API Hub 设置页，在版本与更新区域点击 “立即检查”。

插件会帮你看两件事：

  1. 项目是否已经在 GitHub 发布了新的正式版。
  2. 你的浏览器商店版是否已经可以更新。

你可能会看到这些结果：

  • 当前已是最新正式版：不用处理。
  • 发现可升级的新版本：GitHub 已经有新版；如果你是商店版，可能还要等商店审核或推送。
  • 商店更新已可用：浏览器已经拿到新版，点击 “重新加载完成更新” 即可。
  • 检查失败：可能是网络、GitHub 访问或浏览器更新检查暂时不可用，稍后再试即可。

注意

插件内检查不能跳过浏览器商店审核。也就是说，如果 GitHub 已经发布，但商店还没上架，插件也无法直接把商店版更新到那个版本。

在浏览器里手动检查商店更新

如果你使用 Chrome 或 Edge 商店版，也可以让浏览器主动检查一次扩展更新。

Chrome

  1. 在地址栏打开 chrome://extensions/。
  2. 打开右上角的 开发者模式。
  3. 点击页面上的 更新。
  4. 等待浏览器检查扩展更新。

Edge

  1. 在地址栏打开 edge://extensions/。
  2. 打开 开发人员模式。
  3. 点击页面上的 更新。
  4. 等待浏览器检查扩展更新。

这个方法只能更新到商店已经提供的版本。如果商店还在审核，新版本仍然不会出现。

Firefox

Firefox 通常会自动更新扩展。你也可以打开 about:addons，进入扩展管理页面，检查是否有可用更新。不同 Firefox 版本的按钮位置可能略有差异。

什么时候适合从 GitHub 手动安装？

GitHub 手动安装适合临时使用：

  • 商店版还没收到你急需的修复。
  • 当前浏览器没有可用的商店版，也无法使用 Chrome Web Store 等兼容商店入口。
  • 你需要帮忙确认某个新版本是否解决问题。

手动安装前请注意：

  • 手动安装版通常不会自动更新。
  • 后续升级需要重新下载新版安装包；你可以在 GitHub 仓库点 Star / Watch，这样更容易看到新版本发布通知。
  • 从商店版切到手动安装版前，建议先导出备份数据。
  • 不建议长期同时安装多个来源的 All API Hub，除非你明确是在测试。

手动安装和更新步骤请查看 其他浏览器安装指南。

常见问题

我需要一直点“立即检查”吗？

不需要。商店版会由浏览器自动更新。只有你看到 GitHub 已经发布新版，或者想确认商店版是否已经可更新时，再手动检查即可。

插件内“立即检查”会自动下载 GitHub 安装包吗？

不会。它只会显示版本状态，并让浏览器检查商店版是否能更新。GitHub 安装包需要你自己从 Releases 页面下载。

点击“重新加载完成更新”会丢数据吗？

不会。它只是让已经下载好的商店更新生效。账号、Key、设置等数据仍然保存在浏览器扩展存储里。

GitHub 已发布新版，但插件仍然提示商店没有更新，怎么办？

如果你使用商店版，先等待商店审核和推送。急需修复时，可以临时从 GitHub Stable 手动安装；使用前建议先导出备份数据。

手动安装版怎么更新？

重新下载新版安装包并加载新版扩展目录。具体步骤请看 其他浏览器安装指南。
