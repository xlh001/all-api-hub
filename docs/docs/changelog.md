# 更新日志

本页记录面向普通用户的主要更新点（功能变化 / 体验优化 / 问题修复）。如需查看完整历史版本与更详细的技术变更，请访问 [GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases)。

::: tip 新用户先读
- **怎么确认你当前的版本**：打开扩展弹窗，标题栏会显示版本号；也可以在设置页查看。
- **更新后自动打开本页**：可在「设置 → 通用 → 更新日志」控制是否“更新后自动打开更新日志”。
- **排查异常**：可在「设置 → 通用 → 日志」开启控制台日志，并附上复现步骤到 [Issues](https://github.com/qixing-jk/all-api-hub/issues)。
:::

## 3.13.0
- **新功能:**
  - 账号管理：新增“签到状态过期”提示——当「今日已签到/未签到」状态不是今天检测时，会显示橙色警告图标；点击可一键刷新该账号数据，避免被旧状态误导。
  - 界面：多选控件升级为更紧凑的选择器（更省空间，支持搜索与更清晰的已选展示）。
- **问题修复:**
  - Veloera：修复账号数据刷新与签到逻辑，提升可用性。
  - Cookie 认证：移除 Cookie 缓存机制，减少 Cookie 更新后仍读取旧值导致的异常。

**位置提示：**
- 签到状态过期提示：在「设置 → 账号管理」的账号列表中，站点信息右侧的签到图标处。

## 3.12.0
- **新功能:**
  - 密钥管理：新增「导出到 Kilo Code」——生成 Kilo Code / Roo Code 的 providerProfiles 配置，支持复制 apiConfigs 片段或下载 settings JSON 供导入（导入为增量添加，不会清空你已有的 provider）。
- **问题修复:**
  - 账号管理：修复站点名称等文本过长导致布局溢出的问题，改为自动截断显示。
  - 下拉选择器：优化空状态提示，并修复选项文本过长时溢出的问题。

**位置提示：**
- 导出到 Kilo Code：在「设置 → 密钥管理」的密钥列表里，点击某个密钥右上角的 Kilo Code 图标。

## 3.11.0
- **新功能:**
  - New API 渠道管理：新增「重复渠道」提醒——当检测到已存在相同/相似渠道时，会弹出警告对话框，可选择继续创建或取消（不再用错误 Toast 阻止创建）。
- **问题修复:**
  - 账号管理：修复站点名称等文本过长导致布局溢出的问题，改为自动截断显示。

## 3.10.0
- **新功能:**
  - 账号管理：在无痕模式下点击站点链接，会在当前无痕窗口内打开（便于保持无痕登录态）。
  - 账号管理：已禁用的账号也支持点击站点链接，便于作为书签使用。
  - 用量分析：当只有单账号时，图表与列表优先展示站点名称（而不是用户名），信息更直观。
  - 过盾助手：临时过盾窗口新增对 CAP（cap.js）Pow验证的支持，提升通过率。
- **问题修复:**
  - 兑换助手：优先从剪贴板读取兑换码，提升触发准确率。
  - 提示浮层：修复 Toaster 层级问题，提示不再被网页遮挡。

**位置提示：**
- 站点链接：在「设置 → 账号管理」的账号列表里点击站点名称。
- 过盾助手：可参考 [Cloudflare 过盾助手](./cloudflare-helper.md)。

## 3.9.0
- **新功能:**
  - 账号管理：账号新增「手动余额（USD）」字段——当站点无法自动获取余额/额度时，可手动填写用于展示与统计。
  - 账号管理：账号新增「从总余额中排除」开关——用于把特定账号从“总余额”统计里移除（不影响刷新/签到等功能）。
  - 设置：新增「更新后自动打开更新日志」开关（可关闭更新后自动打开本页的行为）。
  - 设置：新增日志设置，可控制是否输出控制台日志及最小日志级别，便于排查问题。
  - 自动签到：执行完成后会刷新相关数据并同步刷新界面。

**位置提示：**
- 账号新增/编辑：在「设置 → 账号管理」里打开新增/编辑账号。
- 更新日志开关、日志设置：在「设置 → 通用」里配置。

## 3.8.0
- **新功能:**
  - 用量分析：新增「用量分析」页面，帮你把多站点、多账号的用量趋势做成图表，一眼看清“哪里用得多 / 花得多 / 变慢了”，便于控成本、做对账和排查问题。
  - 看板内容：支持查看每日总览（请求数、Tokens、额度）、模型分布/花费分布、账号对比、使用时间热点，以及延迟趋势/直方图等。
  - 用量历史同步：新增「用量历史同步」能力，用于拉取并保存“汇总后的用量数据”（不会保存原始日志）；支持设置保留天数、自动同步方式与最小同步间隔，并在「同步状态」里查看每个账号的同步结果与错误提示。
- **怎么用:**
  - 先到「设置 → 账号用量」开启「用量历史同步」，按需设置后点击「立即同步」。
  - 再到左侧菜单「用量分析」查看图表；需要留存或做对账时可点击「导出」。

  - 自动签到预触发/重试：在左侧菜单「自动签到」里配置。
