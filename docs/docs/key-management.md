# 密钥管理

> 密钥管理（Key Management）功能允许你集中查看和管理已添加账号下的所有 API 密钥（Tokens）。如果你使用的是基于 New API 系列的站点，插件还可以自动同步你在站点后台创建的密钥，并支持一键修复、验证和导出。

## 适合什么场景

- 你在多个中转站创建了大量 Token，想在一个地方搜索或批量导出，而不必逐个登录后台。
- 某些站点的 Token 只有在创建时可见，之后无法再次查看；插件可以帮你记录并尝试“修复”这些不可见的 Key。
- 你需要验证某个账号下的特定 Token 是否仍然有效，或是否兼容 CLI 工具。
- 你想把账号下的 Token 快速保存到 [API 凭据库](./api-credential-profiles.md) 中，以便后续进行更细致的分类管理。

## 功能概览

- **Token 集中列表**：按账号分组或查看全部 Token，支持按名称、Key 内容搜索。
- **自动同步与刷新**：对于支持的站点（如 New API），可自动拉取 Token 列表。
- **Key 修复（Repair）**：针对某些不显示完整 Key 的站点，通过管理权限尝试重新获取或生成。
- **健康状态验证**：实时查看 Token 的余额、用量以及是否过期。
- **导出集成**：支持将单个或多个 Token 导出到常用的 AI 工具。

## 进入方式

1. 打开插件设置页。
2. 在左侧菜单进入 **`密钥管理`**。

## 核心操作

### 1. 自动刷新 Token 列表
在密钥管理页顶部，你可以点击 **`刷新`** 按钮。插件会尝试连接当前选中账号的后台 API，拉取最新的 Token 列表及其状态（如余额、用量、过期时间）。

### 2. 修复缺失的 Key（Repair）
有些站点为了安全，在 Token 列表页只显示 `sk-***...`。如果你在插件中配置了该站点的管理员权限或对应的管理账号：
1. 点击顶部的 **`修复缺失的 Key`**。
2. 插件会自动遍历当前列表，尝试通过管理接口补全那些不可见的完整 Key。
3. 补全后的 Key 会安全地存储在本地，方便你后续复制或导出。

### 3. 验证与健康检查
每个 Token 卡片上都会显示其当前的“健康状态”：
- **余额/用量**：实时从站点获取的配额信息。
- **验证接口**：手动触发一次 API 调用测试，确认 Key 是否可用。
- **验证 CLI 兼容性**：测试该 Token 是否能在特定的 CLI 环境下正常工作。

### 4. 导出到其他工具
你可以点击 Token 旁边的 **`导出`** 按钮，将其一键同步到：
- CherryStudio
- CC Switch
- Kilo Code
- CLIProxyAPI
- Claude Code Router
- 你配置的自建托管站点

## 管理联动

如果你觉得某个 Token 非常常用，或者想给它加上更详细的备注和标签，可以点击 **`保存到 API 凭据库`**。这会将该 `URL + Key` 组合复制到 [API 凭据库](./api-credential-profiles.md) 中。

## 常见问题

| 问题 | 说明 |
|------|------|
| 为什么我看不到完整的密钥？ | 站点后台可能禁用了密钥回显；请尝试使用“修复”功能（需对应权限）。 |
| 支持哪些类型的站点？ | 目前对 New API 系列及其变体（DoneHub、Veloera 等）支持最完整。 |
| 删除这里的 Token 会同步到站点后台吗？ | 不会。这里的删除只是从插件本地缓存中移除，不会操作你的线上账号。 |
| Key 的安全性如何？ | 所有 Key 均加密存储在浏览器本地存储中，除非你手动导出或配置 WebDAV 同步，否则不会离开你的设备。 |
