# セルフホスト型サイト管理 (New API / AxonHub / Claude Code Hub など)

> 🧪 この機能は、最も一般的なチャネル運用アクション（サイト構築、パラメータ調整、同期）を拡張機能に凝縮することを目的としており、異なるシステムのバックエンド間を頻繁に行き来する必要がなくなります。

## サポートされているシステムタイプ

All API Hub は、以下のオープンソース/セルフホスト型 AI 分散システムに深く適応しています。

| システムタイプ | コア管理オブジェクト | 特徴 |
|----------|------------|------|
| **New API / DoneHub / Veloera** | チャネル (Channel) | 55以上のアップストリームタイプをサポートするクラシックなチャネル管理。グループ、優先度、ウェイトシステムを備えています。 |
| **AxonHub** | チャネル (Channel) | 15以上のチャネルタイプをサポートする高性能 AI ゲートウェイ。シンプルなインターフェースと効率的な設定が特徴です。 |
| **Claude Code Hub** | プロバイダー (Provider) | マルチベンダーアクセスと弾力的なスケジューリングに焦点を当てており、明確なプロバイダー管理ロジックと複数のレスポンスプロトコルへの適応を提供します。 |
| **Octopus** | チャネル (Channel) | 個人向けの軽量な集約サービス。6つの主要なチャネルタイプをサポートしています。 |

## 機能の概要

- 📋 **統一リストビュー**: すべてのチャネル/プロバイダーの名前、タイプ、モデルリスト、優先度、ステータスを一目で確認できます。
- ✏️ **クロスプラットフォームの CRUD**: 選択したシステムタイプに応じてフォームフィールドを自動的に適応させ、パラメータを手動で変換する必要がありません。
- 🔄 **チャネルの移行と同期**: 既存のアカウント/キーをセルフホスト型サイトのチャネルとして直接インポートすることをサポートし、単一チャネルまたは一括モデル同期をサポートします。
- 🗑️ **安全な一括操作**: 一括有効化、無効化、削除をサポートし、操作前に確認ポップアップが表示されます。
- 🔌 **ダウンストリームエコシステム連携**: チャネルを作成した後、「キー管理」と連携して CherryStudio、CC Switch などにワンクリックでエクスポートできます。

## 設定ガイド

管理機能を使用する前に、拡張機能で対応するバックエンドの接続設定を完了する必要があります。

### 1. 設定ページにアクセスする
拡張機能の設定ページを開き、左メニューの **「基本設定」** に移動して、 **「セルフホスト型サイト管理」** を見つけます。

### 2. 接続情報を入力する

| 設定項目 | 説明 |
|------|----------|
| **Base URL** | セルフホスト型システムのバックエンドアドレス（通常はウェブアクセスアドレス）。 |
| **認証資格情報** | **New API シリーズ**: 管理者トークン (Admin Token) とユーザー ID が必要です。<br>**AxonHub**: 管理者メールアドレスとパスワード。<br>**Claude Code Hub**: 管理者メールアドレスとパスワード。<br>**Octopus**: ユーザー名とパスワード。 |

### 3. 接続を検証する
 **「設定を検証」** をクリックします。検証に成功すると、対応するシステムの管理エントランスが自動的にロック解除されます。

## チャネル管理操作ガイド

### 1. リスト操作
設定ページ上部の **「セルフホスト型サイト管理」** を選択します（または基本設定の **「チャネルを管理」** ボタンをクリックします）。

- **検索とフィルター**: 名前、タイプ、ステータスによるリアルタイム検索をサポートします。
- **システムの素早い切り替え**: 複数のセルフホスト型システムを設定している場合、上部のスイッチャーを使用して素早く切り替えることができます。
- **カスタム列**: 「優先度」や「ウェイト」をサポートしていないシステム（Octopus など）の場合、対応する列は自動的に非表示になります。

### 2. チャネルの作成または編集
1. 右上の **「チャネルを追加」** をクリックします。
2. フォームは現在のシステムタイプに応じて自動的に調整されます。
   - **New API**: 豊富なチャネルタイプとグループ設定を提供します。
   - **Claude Code Hub**: プロバイダータイプ（OpenAI Compatible、Claude、Gemini など）を選択する必要があります。
   - **AxonHub**: モデルリストの迅速な設定をサポートします。
3. 保存すると、システムはバックエンド API を直接呼び出して同期を完了します。

### 3. チャネルの移行 (Beta)
これは、 **「アカウント管理」** にある既存のアカウントをセルフホスト型サイトのチャネルに素早く変換できる高度な機能です。

1. チャネルリストページで **「チャネルの移行」** をクリックします。
2. ソース（既存のアカウント/ブックマーク）とターゲット（現在のセルフホスト型サイト）を選択します。
3. 拡張機能は自動的に以下を行います。
   - Base URL と API キーを抽出します。
   - 利用可能なアップストリームモデルリストを自動的に識別しようとします。
   - プリセットルールに従ってチャネル名を生成します。

## よくある質問

| 質問 | 解決策 |
|------|----------|
| 設定の検証に失敗しました | Base URL が正しく入力されているか（`https://` を含む）、および管理者権限が有効か確認してください。一部のシステムでは、2段階認証を無効にするか、特定の API トークンを使用する必要があります。 |
| リストの読み込みが遅い | チャネル数が多い場合（>100個）、バックエンド API のパフォーマンス制限により、読み込みに数秒かかることがあります。しばらくお待ちください。 |
| モデルを同期できません | セルフホスト型サイトのバックエンドネットワークがアップストリームアドレス（OpenAI / Claude 公式サイトなど）に正常にアクセスできるか確認してください。 |
| 一部のフィールドが Unknown と表示される | これは通常、セルフホスト型サイトのバージョンが新しすぎるか古すぎるため、拡張機能がまだ適応していないタイプ ID が返されていることが原因です。 |
