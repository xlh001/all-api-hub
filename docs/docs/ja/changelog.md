# 更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / バグ修正）を記録しています。完全な履歴バージョンとより詳細な技術的変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 新規ユーザー向け
- **現在のバージョン確認方法**: 拡張機能ポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページを自動で開かないようにする方法**: 「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動で開く」の制御が可能です。
- **異常のトラブルシューティング**: 「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.21.0
- **新機能:**
  - API認証情報: 「API認証情報」ページを追加。これは「アカウントはなく、`Base URL` + `API Key` のみを持つ」ユースケースに適しています。タグ/メモによる一元管理、可用性検証、およびクイックエクスポート（例: Cherry Studio / CC Switch / Kilo Code / CLIProxyAPI / Claude Code Router）をサポートし、コピー＆ペーストの手間を削減します。
  - 残高履歴: マルチアカウントビュー（概要 / アカウント分布 / トレンド）を追加し、統一された「アカウントサマリー」テーブルを提供。迅速な比較と集計統計に役立ちます。
  - カスタムサイト管理: ホストされているサイトに `Done Hub` を追加。管理者認証情報を設定し、「チャネル管理」「モデル同期」などの機能に使用できます。
- **体験最適化:**
  - 右クリックメニュー: 「交換アシスタント」と「AI API テスト」のエントリを個別にオン/オフ可能になり、切り替え後すぐに更新が反映されます。
  - キーのコピー: アカウントにキーがない場合、ポップアップに「デフォルトキーをクイック作成 / カスタムキーを作成」のエントリが表示され、行き来する手間が省けます。

**場所のヒント:**
- API認証情報: 「設定 → API認証情報」にあります。
- 残高履歴: 「設定 → 残高履歴」にあります。
- Done Hub 設定: 「設定 → 基本設定 → カスタムサイト管理」で `Done Hub` を選択し、「Done Hub 統合設定」に入力します。
- 右クリックメニューのエントリのオン/オフ: 「設定 → 基本設定 → チェックインと交換 / AI API テスト」内の「ブラウザの右クリックメニューに表示」でそれぞれ設定します。
- キーコピーのポップアップ: 「アカウント管理」などのページで「キーをコピー」をクリックして開きます。

## 3.19.0
- **新機能:**
  - カスタムサイト管理: `Octopus` ホストサイトのサポートを追加。Octopus バックエンドに接続し、「チャネル管理」でアカウントAPIキーをチャネルとしてインポートでき、利用可能なモデルリストの取得もサポートします。

- **体験最適化:**
  - アカウント管理: サイト/アカウント認識ロジックを強化し、同一サイトでの複数アカウントシナリオの安定性を改善しました。
  - 使用量/ログ取得: ログ系インターフェースにレート制限保護を追加し、頻繁な更新によるエラーやサイトのレート制限トリガーを削減します。
  - チャネル管理: チャネル作成時に、より正確な重複チェックを行い、確認プロンプトを表示することで、誤った重複作成を防ぎます。

  - アカウント停止: キー管理などの関連ドロップダウン/リストで、停止済みアカウントが自動的にフィルタリングされ、無効な操作を防ぎます。
  - 言語: 拡張機能の言語設定がウェブページ自体の言語値に影響を与える問題を修正しました。

**場所のヒント:**

- チャネル管理のエントリ: 「設定 → チャネル管理」にあります。
- デフォルトキー自動作成のオン/オフ: 「設定 → 基本設定 → アカウント管理 → APIキー」にあります。
- デフォルトキーの自動補完: 「設定 → キー管理」の右上にある「少なくとも1つのキーを保証」をクリックします。
- AI API テストのエントリ: ウェブページの右クリックメニューの「AI API の機能可用性をクイックテスト」からアクセスできます。

## 3.16.0
- **新機能:**
  - Sub2API（JWTサイト）: Sub2API サイトタイプを追加し、残高/クォータ照会をサポート。「自動認識」によるコンソールログイン状態の読み取りをサポートし、「プラグインホストセッション（マルチアカウント、推奨）」モードをサポート。各アカウントの認証を個別に更新でき、同一サイトでのマルチアカウント体験を改善します。
  - 表示設定: 「今日の収支を表示」スイッチ（デフォルトオン）を追加。これをオフにすると、「今日の消費/収入」などの統計表示が非表示になり、更新時のログ取得リクエストを削減できます。
- **注意:**
  - Sub2API は現在、サイトチェックイン/今日の使用量/収入などの機能はサポートしていません。基本的な残高/クォータ照会のみを提供します。関連機能はサイトの能力に応じて今後段階的に改善されます。
  - 「プラグインホストセッション（マルチアカウント）」では、`refresh_token` がアカウントのプライベート認証情報として保存され、エクスポート/WebDAV バックアップにも含まれます。バックアップファイルと WebDAV 認証情報を大切に保管してください。

**場所のヒント:**
- Sub2API の追加/モードの説明: 「設定 → アカウント管理」でアカウントを追加/編集する際に、サイトタイプとして Sub2API を選択します。詳細は [FAQ](./faq.md)（"Sub2API" で検索）を参照してください。
- 今日の収支のオン/オフスイッチ: 「設定 → 基本設定 → 表示設定」にあります。

## 3.15.0
- **新機能:**
  - ブックマーク管理: サイトコンソール/ドキュメント/管理ページなどのショートカットリンクを保存するための「ブックマーク管理」を追加。完全なアカウントを作成する必要がありません。追加/編集/削除、ピン留め、タグ付け、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポート。ポップアップに「アカウント / ブックマーク」切り替えを追加。ブックマークデータはバックアップ/復元および WebDAV 自動同期に含まれます。
- **バグ修正:**
  - アカウント更新: 重複する「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部サイトでは更新インターフェースで `today_income` が返されます）。
  - 自動更新: 更新間隔の最小値を 60 秒に、最小更新間隔保護を 30 秒に設定。更新後、古い設定は自動的に合法な範囲に調整され、関連するプロンプトテキストとドキュメントが改善されました。

::: warning 重要：自動更新設定は強制的に変更されます
**短すぎる自動更新間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける**というフィードバックに基づき、

v3.15.0 では**自動更新設定が強制的に変更**されました:
- 自動更新機能とプラグイン起動時の更新機能はオフになりました。引き続き使用したい場合は、手動で再度オンにする必要があります。
- `更新間隔` の最小値は 60 秒、`最小更新間隔保護` の最小値は 30 秒です。アップグレード前に設定した値が上記閾値を下回っていた場合、アップグレード後に自動的に最小値に引き上げられます。以前の設定値が新しい合法範囲内にある場合は影響ありません。
:::

**場所のヒント:**
- ブックマーク管理: 「設定 → ブックマーク管理」にあります。ポップアップの上部で「アカウント / ブックマーク」を切り替えます。
- 自動更新: 「設定 → 基本設定 → 自動更新」にあります。

## 3.14.0
- **新機能:**
  - ウェブページ AI API 機能可用性テスト（ベータ）: 右クリックメニューに「AI API の機能可用性をクイックテスト」を追加し、現在のページで直接テストパネルを開くことができます。`Base URL` と `API Key` の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Google などのインターフェースに対して基本的な機能プローブ（OpenAI互換ではモデルリストの一括取得も可能）を実行します。
  - （オプション）自動検出: 「設定 → AI API テスト」で有効にし、URLホワイトリストを設定できます。ホワイトリスト内のページで利用可能な `Base URL` と `API Key` が検出された場合、確認プロンプトが表示され、確認後にテストパネルが開きます（デフォルトオフ、Keyは保存されません）。
  - 自動チェックイン: 実行結果リストに、より多くのトラブルシューティングヒントを追加。これには、「一時的な盾回避タブが手動で閉じられた」「Access Token が無効」など、一般的な異常に対する対処法の提案とドキュメントへの入り口が含まれます。
- **バグ修正:**
  - WebDAV: 自動同期をタイマーからブラウザの Alarms API に移行し、バックグラウンドのスリープ/省電力ポリシーによる同期漏れの可能性を低減しました。

**場所のヒント:**
- AI API テストパネル: 任意のウェブページの右クリックメニューで「AI API の機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI API テスト」にあります。
- 自動チェックインのヒント: 「設定 → 自動チェックイン」の実行結果リストで確認できます。
