# 更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の改善 / 不具合の修正）を記録しています。完全な履歴バージョンとより詳細な技術的な変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 新規ユーザーの方へ
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページを自動で開かないようにする方法**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動で開く」を制御できます。
- **エラーのトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.19.0
- **新機能:**
  - セルフホスト型サイト管理：新たに `Octopus` ホスト型サイトのサポートを追加しました。Octopusの管理画面に接続し、「チャネル管理」でアカウントのAPIキーをチャネルとしてインポートできます。また、利用可能なモデルリストの取得もサポートしています。
  - キー管理：新たに「アカウント追加後にデフォルトキーを自動作成」（デフォルトで有効）を追加し、キーが不足しているアカウントに対してデフォルトキーを自動で補完する「少なくとも一つのキーを確保」機能を提供します。
  - AI API テスト：インターフェース検証の「モデルリスト検出」が、OpenAI/OpenAI互換、Anthropic、Google/Geminiなどのインターフェースタイプをサポートし、利用可能なモデルIDの提案を行うことで、手動でのモデル推測を減らします。
- **体験の改善:**
  - アカウント管理：サイト/アカウントの識別ロジックを強化し、同一サイトでのマルチアカウントシナリオにおける安定性を向上させました。
  - 利用量/ログ取得：ログ関連のインターフェースに対してレート制限保護を追加し、頻繁な更新によるエラーやサイトのレート制限トリガーを減らしました。
  - チャネル管理：チャネル作成時により正確な重複検出を行い、確認のプロンプトを表示することで、誤って重複したルートを作成するのを防ぎます。
- **不具合の修正:**
  - アカウントの無効化：キー管理などの関連ドロップダウン/リストから無効化されたアカウントを自動的にフィルタリングし、無効な操作を回避します。
  - 言語：拡張機能の言語設定がウェブページ自体の言語値に影響を与える可能性があった問題を修正しました。

**場所のヒント：**
- Octopus 設定：「設定 → 基本設定 → セルフホスト型サイト管理」で `Octopus` を選択し、`ベースURL` / ユーザー名 / パスワードを入力します。
- チャネル管理の入り口：「設定 → チャネル管理」。
- デフォルトキー自動作成のスイッチ：「設定 → 基本設定 → アカウント管理 → APIキー」。
- デフォルトキーの一括補完：「設定 → キー管理」の右上にある「少なくとも一つのキーを確保」。
- AI API テストの入り口：ウェブページの右クリックメニュー「AI API の機能可用性をクイックテスト」。

## 3.16.0
- **新機能:**
  - Sub2API（JWT サイト）：Sub2API サイトタイプを新たに追加し、残高/クォータの照会をサポートします。「自動識別」によるコンソールログイン状態の読み取りをサポートします。また、「プラグインホスト型セッション（マルチアカウント、推奨）」モードをサポートし、アカウントごとに認証を個別に更新でき、同一サイトでのマルチアカウント体験を改善します。
  - 表示設定：「今日の収支を表示」スイッチ（デフォルトで有効）を新たに追加しました。これをオフにすると、「今日の消費/収入」などの統計情報の取得を非表示にし、停止することで、更新時のログ取得リクエストを減らします。
- **注意:**
  - Sub2API は、サイトのチェックイン/今日の利用量/収入などの関連機能を一時的にサポートしていません。基本的な残高/クォータの照会のみを提供します。関連機能は、サイトの能力に応じて順次改善される予定です。
  - 「プラグインホスト型セッション（マルチアカウント）」では、`refresh_token` がアカウントのプライベートな認証情報として保存され、エクスポート/WebDAVバックアップに含まれます。バックアップファイルとWebDAV認証情報を適切に保管してください。

**場所のヒント：**
- Sub2API の追加/モードの説明：「設定 → アカウント管理」でアカウントを新規追加/編集し、サイトタイプで Sub2API を選択します。より詳細な手順は [FAQ](./faq.md)（「Sub2API」で検索）を参照してください。
- 今日の収支スイッチ：「設定 → 基本設定 → 表示設定」。

## 3.15.0
- **新機能:**
  - ブックマーク管理：サイトのコンソール/ドキュメント/管理ページなどのショートカットリンクを保存するための「ブックマーク管理」を新たに追加しました。完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元およびWebDAV自動同期に含まれます。
- **不具合の修正:**
  - アカウント更新：重複していた「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを減らしました（一部のサイトでは更新インターフェースで既に `today_income` が返されています）。
  - 自動更新：更新間隔の最低値を60秒、最小更新間隔保護の最低値を30秒に設定しました。更新後、古い設定は自動的に有効な範囲に修正され、関連するヒントの文言とドキュメントを改善しました。

::: warning 重要：自動更新設定が強制的に調整されます
**短すぎる自動更新間隔はサイトのレート制限をトリガーしやすく、サイトに過度な負荷をかける**というフィードバックに基づき、

v3.15.0 では**自動更新設定に強制的な変更**を加えました：
- 自動更新とプラグインを開いた時の更新機能がオフになりました。引き続き有効にする必要がある場合は、手動で再度有効にする必要があります。
- `更新間隔` の最低値は60秒、`最小更新間隔保護` の最低値は30秒です。アップグレード前にこれらのしきい値より低く設定されていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい有効な範囲内であれば、影響はありません。
:::

**場所のヒント：**
- ブックマーク管理：「設定 → ブックマーク管理」。ポップアップの上部で「アカウント / ブックマーク」を切り替えられます。
- 自動更新：「設定 → 基本設定 → 自動更新」。

## 3.14.0

**場所のヒント：**
- AI API テストパネル：任意のWebページで右クリックメニューから「AI API の機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI API テスト」にあります。
- 自動チェックインのヒント：「設定 → 自動チェックイン」の実行結果リストで確認できます。

## 3.8.0
- **新機能:**
  - 利用量分析：新たに「利用量分析」ページを追加しました。複数のサイト、複数のアカウントの利用傾向をグラフ化し、「どこで多く使っているか / 多く費やしているか / 遅くなっているか」を一目で確認でき、コスト管理、照合、問題のトラブルシューティングに役立ちます。
