# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験改善 / 問題修正）を記録しています。完全な履歴バージョンと詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにする方法**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.22.0
- **新機能:**
  - モデルリスト：新しい「モデル対応キー」ツール（鍵アイコン）を追加しました。これにより、現在のモデルに利用可能なキーが存在するかどうかを確認できます。利用可能なキーがない場合は、モデルの利用可能グループに基づいてデフォルトキーをワンクリックで作成するか、カスタム作成プロセスに進むことができます。キーのワンクリックコピーもサポートしています。
  - スナップショットの共有：ワンクリックで「概要スナップショット / アカウントスナップショット」を共有できるようになりました。優先的にクリップボードに画像をコピーし、サポートされていない場合は自動的にPNGをダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトル文案のワンクリックコピーも可能です。
- **体験改善:**
  - アカウントの無効化：「残高履歴 / 使用量分析 / 使用量同期」などの更新および定期タスクでは、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーが減少します。
- **問題修正:**
  - ボタン：「読み込み中」かつ左側アイコンが表示されている場合に、スピナーが表示されない問題を修正しました。

**場所のヒント：**
- モデル対応キー：[設定 → モデルリスト] で、モデル名の右側にある鍵アイコン（`モデル対応キー`）をクリックします。
- 概要スナップショットの共有：拡張機能ポップアップの概要ページのタイトルバーの右側にあるボタン（`概要スナップショットの共有`）をクリックします。
- アカウントスナップショットの共有：[設定 → アカウント管理] の個々のアカウントの操作メニュー（`アカウントスナップショットの共有`）から行います。

## 3.21.0
- **新機能:**
  - API認証情報：「API認証情報」ページを追加しました。「アカウントがなく、`Base URL` + `API Key` のみがある」という使用シナリオに適しています。タグ/メモによる一元管理をサポートし、直接可用性検証と高速エクスポート（例：Cherry Studio / CC Switch / Kilo Code / CLIProxyAPI / Claude Code Router）が可能になり、コピー＆ペーストの手間を削減します。
  - 残高履歴：複数アカウントビュー（概要 / アカウント分布 / トレンド）を追加し、統一された「アカウント集計」テーブルを提供することで、迅速な比較と集計統計を容易にします。
  - カスタムサイト管理：ホストサイトに `Done Hub` を追加しました。管理者認証情報の構成をサポートし、「チャネル管理」「モデル同期」などの機能に使用できます。
- **体験改善:**
  - 右クリックメニュー：「交換アシスタント」と「AI API検出」の入口を個別にオン/オフできるようになりました。切り替え後すぐにリフレッシュして有効になります。
  - キーのコピー：アカウントにキーがない場合、ポップアップに「デフォルトキーのクイック作成 / カスタムキーの作成」の入口が表示され、移動の手間を省きます。

**場所のヒント：**
- API認証情報：[設定 → API認証情報]
- 残高履歴：[設定 → 残高履歴]
- Done Hub 設定：[設定 → 基本設定 → カスタムサイト管理] で `Done Hub` を選択し、「Done Hub 統合設定」を入力します。
- 右クリックメニュー入口のオン/オフ：[設定 → 基本設定 → サインインと交換 / AI API テスト] のそれぞれの「ブラウザの右クリックメニューに表示」で設定します。
- キーコピーポップアップ：[アカウント管理] などのページで「キーをコピー」をクリックして開きます。

## 3.19.0
- **新機能:**
  - カスタムサイト管理：`Octopus` ホストサイトのサポートを追加しました。Octopus バックエンドに接続し、「チャネル管理」でアカウントAPIキーをチャネルとしてインポートできます。利用可能なモデルリストの取得もサポートしています。
  - キー管理：「アカウント追加後にデフォルトキーを自動作成」（デフォルトで有効）を追加しました。「少なくとも1つのキーを確保」機能により、キーが不足しているアカウントにデフォルトキーを自動補完します。
  - AI API テスト：インターフェース検証の「モデルリスト検出」で、OpenAI/OpenAI互換、Anthropic、Google/Geminiなどのインターフェースタイプをサポートし、利用可能なモデルIDの提案を提供することで、手動でのモデル推測を減らします。
- **体験改善:**
  - アカウント管理：サイト/アカウント識別ロジックを強化し、同一サイトでの複数アカウントシナリオの安定性を向上させました。
  - 使用量/ログ取得：ログインターフェースにレート制限保護を追加し、頻繁なリフレッシュによるエラーやサイトのレート制限のトリガーを減らしました。
  - チャネル管理：チャネル作成時に、より正確な重複検出を行い、確認プロンプトを表示することで、誤った重複ルート作成を防ぎます。
- **問題修正:**
  - アカウントの無効化：キー管理などの関連ドロップダウン/リストで無効化されたアカウントを自動的にフィルタリングし、無効な操作を防ぎます。
  - 言語：拡張機能の言語設定が、ウェブページ自体の言語値に影響を与える問題を修正しました。

**場所のヒント：**
- Octopus 設定：[設定 → 基本設定 → カスタムサイト管理] で `Octopus` を選択し、`基本URL` / ユーザー名 / パスワードを入力します。
- チャネル管理入口：[設定 → チャネル管理]
- デフォルトキー自動作成スイッチ：[設定 → 基本設定 → アカウント管理 → APIキー]
- デフォルトキーのワンクリック補完：「設定 → キー管理」の右上にある「少なくとも1つのキーを確保」をクリックします。
- AI API テスト入口：ウェブページの右クリックメニューから「AI APIの機能可用性を迅速にテスト」を選択します。

## 3.16.0
- **新機能:**
  - Sub2API（JWTサイト）：Sub2APIサイトタイプを追加し、残高/クォータクエリをサポートしました。「自動識別」によるコンソールログイン状態の読み取りをサポートし、「プラグインホストセッション（複数アカウント、推奨）」モードもサポートしています。これにより、各アカウントの認証を個別に更新でき、同一サイトでの複数アカウント体験を向上させます。
  - 表示設定：「今日の収支を表示」スイッチを追加しました（デフォルトでオン）。これにより、「今日の消費/収入」などの統計情報を非表示にし、取得を停止することで、リフレッシュ時のログ取得リクエストを減らします。
- **注意:**
  - Sub2API は現在、サイトへのサインイン/今日の使用量/収入などの機能はサポートしていません。基本的な残高/クォータクエリのみを提供します。後続でサイトの能力に応じて徐々に機能が追加されます。
  - 「プラグインホストセッション（複数アカウント）」は、`refresh_token` をアカウントのプライベート認証情報として保存し、エクスポート/WebDAVバックアップにも含まれます。バックアップファイルとWebDAV認証情報は慎重に保管してください。

**場所のヒント：**
- Sub2API の追加/モード説明：[設定 → アカウント管理] でアカウントを追加/編集し、サイトタイプに Sub2API を選択します。詳細な手順は [FAQ](./faq.md)（「Sub2API」で検索）を参照してください。
- 今日の収支スイッチ：[設定 → 基本設定 → 表示設定]

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」を追加しました。サイトコンソール/ドキュメント/管理ページなどのショートカットリンクを保存でき、完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ付け、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元およびWebDAV自動同期に含まれます。
- **問題修正:**
  - アカウントリフレッシュ：重複する「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部サイトではリフレッシュインターフェースが `today_income` を返します）。
  - 自動リフレッシュ：リフレッシュ間隔の最小値を60秒、最小リフレッシュ間隔保護を30秒に設定しました。更新後、古い設定は自動的に合法範囲に校正され、関連するヒント文言とドキュメントが整備されました。

::: warning 重要：自動リフレッシュ設定は強制的に調整されます
フィードバックによると、**短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける**可能性があります。

v3.15.0 では**自動リフレッシュ設定を強制的に変更**しました：
- 自動リフレッシュとプラグインを開いたときの自動リフレッシュ機能をオフにしました。引き続き使用したい場合は、手動で再度有効にする必要があります。
- `リフレッシュ間隔` の最小値を60秒、`最小リフレッシュ間隔保護` の最小値を30秒としました。アップグレード前にこれらの閾値を下回る設定をしていた場合、アップグレード後に自動的に最小値に引き上げられます。以前の設定値が新しい合法範囲内にある場合は影響ありません。
:::

**場所のヒント：**
- ブックマーク管理：[設定 → ブックマーク管理]；ポップアップの上部で「アカウント / ブックマーク」を切り替えます。
- 自動リフレッシュ：[設定 → 基本設定 → 自動リフレッシュ]

## 3.14.0
- **新機能:**
  - ウェブAI API機能可用性テスト（Beta）：右クリックメニューに「AI APIの機能可用性を迅速にテスト」を追加しました。これにより、現在のウェブページで直接テストパネルを開くことができます。`Base URL` と `API Key` の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Googleなどのインターフェースの基本的な機能検出を行います（OpenAI互換ではモデルリストのワンクリック取得も可能です）。

  - WebDAV：自動同期をタイマーからブラウザのAlarms APIに移行し、バックグラウンドのスリープ/省電力ポリシーによる同期漏れの可能性を低減しました。

**場所のヒント：**
- AI API テストパネル：任意のウェブページの右クリックメニューから「AI APIの機能可用性を迅速にテスト」を選択します。自動検出関連の設定は [設定 → AI API テスト] にあります。
- 自動サインインヒント：[設定 → 自動サインイン] の実行結果リストで確認します。
