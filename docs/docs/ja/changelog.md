# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録します。完全な履歴バージョンと詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにする方法**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.24.0
- **新機能:**
  - 更新履歴：プラグイン更新後にブラウザの新しいタブで自動的に開くことはなくなりました。代わりに、プラグインインターフェースを初めて開いた際に、プラグイン内でポップアップ表示され、今回の更新内容が表示されます。完全な更新履歴へのワンクリックオープンもサポートしています。
  - LDOH：アカウントリストに `LDOHで確認`（LDOHアイコン）のショートカットエントリが追加され、直接LDOHにジャンプして対応するサイトを自動的にフィルタリングできます。アカウント追加時にも `LDOHサイトリストを開く` エントリが提供され、サイトを見つけやすくなっています。
- **体験最適化:**
  - ドキュメントリンク：プラグインからドキュメント/更新履歴を開く際、現在のプラグイン言語に応じて対応する言語のドキュメントページに自動的にジャンプします。

**場所のヒント:**
- 更新履歴のスイッチ：「設定 → 一般 → 更新履歴」の `更新後に更新内容を自動表示`。
- 更新履歴ポップアップ：プラグイン更新後、初めて「拡張機能ポップアップ / 設定ページ / サイドバー」を開いた際に自動的にポップアップします（各バージョンにつき1回のみ）。
- LDOHショートカットエントリ：「アカウント管理」のアカウントリストのサイト名右側（LDOHアイコン、`LDOHで確認` と表示）；アカウント追加ダイアログでも `LDOHサイトリストを開く` をクリックできます。

## 3.23.0
- **新機能:**
  - 自動サインイン：アカウント操作メニューに `クイックサインイン` が追加され、個別のアカウントに対して即座にサインインを実行し、完了後にステータスをリフレッシュできます。
  - キー管理：`すべてのアカウント` ビューが追加され、アカウントごとにキーをグループ化して集約表示し、アカウントを跨いだ検索とコピーを容易にします。
  - モデルリダイレクト：`モデルリダイレクトマッピングをクリア` の一括操作が追加され、チャネルごとに選択し、二次確認後に `model_mapping` を素早くリセットできます（取り消し不可）。
- **体験最適化:**
  - New API チャネル管理：チャネルリストのURLがクリック可能になり、検索体験が最適化されました。
- **問題修正:**
  - チャネル管理：チャネルダイアログの `優先度` のヒント文言が不正確だった問題を修正しました。
  - モデルリダイレクト：自動生成マッピングに「バージョンガード」が追加され、バージョンを跨いだ誤ったマッチングを防ぎます。
  - サイドバー：サイドバーがサポートされていない実行環境では、自動的にポップアップ/設定ページを開くようにフォールバックし、クリックしても無反応になるのを防ぎます。

## 3.22.0
- **新機能:**
  - モデルリスト：`モデル対応キー` ツール（鍵アイコン）が追加され、現在のモデルに利用可能なキーがあるか確認できます。利用可能なキーがない場合、モデルの利用可能なグループごとにデフォルトキーをワンクリックで作成するか、カスタム作成プロセスに進むことができます。キーのコピーもワンクリックで可能です。
  - スナップショット共有：`概要スナップショット / アカウントスナップショット` のワンクリック共有をサポートし、優先的に画像をクリップボードにコピーします。コピーできない場合は自動的にPNGをダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトル文案のコピーもワンクリックで可能です。
- **体験最適化:**
  - アカウント無効化：`残高履歴 / 使用量分析 / 使用量同期` などのリフレッシュと定期タスクでは、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーを削減します。
- **問題修正:**
  - ボタン：ボタンが「ロード中」で、かつ左側のアイコンも表示されている場合にスピナーが見えなかった問題を修正しました。

**場所のヒント:**
- モデル対応キー：「設定 → モデルリスト」で、モデル名の右側にある鍵アイコン（`モデル対応キー`）をクリックします。
- 概要スナップショット共有：拡張機能ポップアップの概要ページタイトルバーの右側にあるボタン（`概要スナップショットを共有`）。
- アカウントスナップショット共有：「設定 → アカウント管理」の個別アカウントの操作メニュー（`アカウントスナップショットを共有`）。

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」が追加され、サイトコンソール/ドキュメント/管理ページなどのショートカットリンクを保存でき、完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ付け、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えが追加されました。ブックマークデータはバックアップ/復元とWebDAV自動同期に含まれます。
- **問題修正:**
  - アカウントリフレッシュ：「今日の収入」の重複取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部サイトではリフレッシュインターフェースで `today_income` が返されます）。
  - 自動リフレッシュ：リフレッシュ間隔の最小値が60秒、最小リフレッシュ間隔保護の最小値が30秒になりました。更新後、古い設定は自動的に合法範囲に補正され、関連するヒント文言とドキュメントが充実しました。

::: warning 重要：自動リフレッシュ設定は強制調整されます
フィードバックによると、**短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける可能性があります**。

v3.15.0 では**自動リフレッシュ設定を強制的に変更しました**：
- 自動リフレッシュとプラグインを開いた時のリフレッシュ機能をオフにしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
- `リフレッシュ間隔` の最小値は60秒、`最小リフレッシュ間隔保護` の最小値は30秒です。アップグレード前にこれらの閾値を下回る設定をしていた場合、アップグレード後に自動的に最小値に引き上げられます。以前設定した値が新しい合法範囲内であれば、影響はありません。
:::

**場所のヒント:**
- ブックマーク管理：「設定 → ブックマーク管理」；ポップアップ上部で「アカウント / ブックマーク」を切り替えます。
- 自動リフレッシュ：「設定 → 基本設定 → 自動リフレッシュ」。
