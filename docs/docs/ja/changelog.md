# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験改善 / 不具合修正）を記録します。完全な履歴バージョンとより詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 初めての方へ
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにするには**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.25.0
- **新機能:**
  - 自動サインイン：Cloudflare Turnstile（盾突破/人間認証）シナリオをサポート。サイトがTurnstile認証を要求する場合、一時的なページで認証を完了してからサインインを試みます。必要に応じて、手動で開けるサインインリンクとヒントも提供します。
  - CC Switch：`Codex` にエクスポートする際、ベースアドレスのデフォルト値に `/v1` を自動補完します（インターフェースアドレスを手動で変更していない場合）。これにより、直接インポートした後にインターフェースが利用できなくなる問題を軽減します。
  - モデルリダイレクト：オプションのスイッチ `同期後に無効なリダイレクト先をクリーンアップ` を追加。モデル同期リフレッシュ後に、`model_mapping` 内で存在しないモデルを指しているマッピングを自動的に削除します（危険な操作、デフォルトでは無効）。
- **体験改善:**
  - 一時ウィンドウ：チャレンジ/ログインページとしてより正確に識別し、誤判定や不要な中断を減らします。
- **不具合修正:**
  - Cookie認証：現在の実際の動作と能力に合致するように説明文を修正し、誤解を減らします。
  - サイドバー：小ウィンドウでサイドバーの最下部メニュー項目が見えなくなる問題を修正しました。

**場所のヒント：**
- Turnstile認証：`設定 → 自動サインイン` の実行結果で新しいヒントを確認してください。
- CC Switch エクスポート：`設定 → キー管理` で、キーを選択してから `CC Switch にエクスポート` をクリックし、ターゲットアプリケーションとして `Codex` を選択してください。
- モデルリダイレクトクリーンアップ：`設定 → 基本設定 → モデルリダイレクト` で `同期後に無効なリダイレクト先をクリーンアップ` を有効にしてください。

## 3.24.0
- **新機能:**
  - 更新履歴：プラグイン更新後にブラウザの新しいタブが自動的に開かなくなりました。代わりに、プラグインインターフェースを初めて開いたときに、プラグイン内でポップアップ表示され、ワンクリックで完全な更新履歴を開けるようになります。
  - LDOH：アカウントリストに `LDOH で表示`（LDOH アイコン）のクイックエントリを追加。LDOH に直接ジャンプし、対応するサイトを自動的にフィルタリングします。アカウント追加時にも `LDOH サイトリストを開く` エントリを提供し、サイトを見つけやすくします。
- **体験改善:**
  - ドキュメントリンク：プラグインからドキュメント/更新履歴を開く際、現在のプラグイン言語に応じて対応する言語のドキュメントページに自動的にジャンプします。

**場所のヒント：**
- 更新履歴スイッチ：`設定 → 一般 → 更新履歴` の `更新後に更新内容を自動表示`。
- 更新履歴ポップアップ：プラグイン更新後、最初に「拡張機能ポップアップ / 設定ページ / サイドバー」を開いたときに自動的にポップアップします（各バージョンにつき一度だけ）。
- LDOH クイックエントリ：`アカウント管理` のアカウントリストのサイト名の右側（LDOH アイコン、`LDOH で表示` と表示）。アカウント追加ダイアログでも `LDOH サイトリストを開く` をクリックできます。

## 3.22.0
- **新機能:**
  - モデルリスト：`モデル対応キー` ツール（鍵アイコン）を追加。現在のモデルに利用可能なキーが存在するかどうかを確認できます。利用可能なキーがない場合は、モデルの利用可能なグループに基づいてデフォルトキーをワンクリックで作成するか、カスタム作成プロセスに進みます。キーのコピーもワンクリックで可能です。
  - スナップショット共有：`概要スナップショット / アカウントスナップショット` のワンクリック共有をサポート。優先的に画像をクリップボードにコピーし、サポートされない場合はPNGを自動ダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトル文言のコピーもワンクリックで可能です。
- **体験改善:**
  - アカウント無効化：`残高履歴 / 使用量分析 / 使用量同期` などのリフレッシュと定期タスクでは、無効化されたアカウントを自動的にスキップし、無効なリクエストとエラーを減らします。
- **不具合修正:**
  - ボタン：「ロード中」で左側アイコンも表示されている場合にスピナーが見えなくなる問題を修正しました。

**場所のヒント：**
- モデル対応キー：`設定 → モデルリスト` で、モデル名の右側にある鍵アイコン（`モデル対応キー`）をクリックしてください。
- 概要スナップショット共有：拡張機能ポップアップの概要ページタイトルバー右側のボタン（`概要スナップショット共有`）をクリックしてください。
- アカウントスナップショット共有：`設定 → アカウント管理` の個別アカウントの操作メニュー（`アカウントスナップショット共有`）をクリックしてください。

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」を追加。サイトコンソール/ドキュメント/管理ページなどのショートカットリンクを保存するために使用し、完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ付け、検索フィルタリング、ドラッグ＆ドロップでの並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元とWebDAV自動同期に含まれます。
- **不具合修正:**
  - アカウントリフレッシュ：「今日の収入」の重複取得リクエストを削除し、不要なネットワーク呼び出しを減らしました（一部サイトではリフレッシュインターフェースで `today_income` が返されます）。
  - 自動リフレッシュ：リフレッシュ間隔の最小値を60秒、最小リフレッシュ間隔保護の最小値を30秒に設定しました。更新後、古い設定は自動的に合法な範囲に補正され、関連するヒント文言とドキュメントが充実しました。

::: warning 重要：自動リフレッシュ設定は強制調整されます
フィードバックによると、**短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかけやすい**ため、

v3.15.0 では**自動リフレッシュ設定を強制的に変更**しました：
- 自動リフレッシュとプラグインを開いたときの更新機能を無効にしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
- `リフレッシュ間隔` の最小値を60秒、`最小リフレッシュ間隔保護` の最小値を30秒としました。アップグレード前にこれらの閾値を下回る設定をしていた場合、アップグレード後に自動的に最小値に引き上げられます。以前の設定値が新しい合法な範囲内であれば、影響はありません。
:::

**場所のヒント：**
- ブックマーク管理：`設定 → ブックマーク管理`。ポップアップ上部で「アカウント / ブックマーク」を切り替えられます。
- 自動リフレッシュ：`設定 → 基本設定 → 自動リフレッシュ`。
