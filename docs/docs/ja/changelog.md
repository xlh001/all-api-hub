# 更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の最適化 / 問題の修正）を記録しています。完全な履歴バージョンとより詳細な技術的な変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 新しいユーザーの方へ
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページを自動で開かないようにする方法**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動で開く」を制御できます。
- **異常の調査**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.17.0
- **新機能:**
  - 残高履歴：「残高履歴」機能を追加しました（デフォルトではオフ）。日ごとの残高と収支のスナップショットを記録し、グラフで傾向を確認できます。タグ/アカウントによるフィルタリング、期間による絞り込みをサポートし、「即時更新 / 即時クリア」の便利な操作を提供します。
  - 残高履歴：有効化の有無、保持日数、および「日次終了時の取得」を制御するための設定項目を追加しました。ヒント：「今日の収支を表示」をオフにしており、「日次終了時の取得」を有効にしていない場合、「日次収支」グラフにはデータが表示されません。
- **体験の最適化:**
  - チャネル管理：小画面/狭いウィンドウでの「チャネル管理」ツールバーのレスポンシブレイアウトと使いやすさを最適化しました。
- **問題の修正:**
  - インポート/エクスポート：一部の画面サイズにおけるエクスポート領域のレスポンシブ表示の問題を修正しました。
  - ポップアップ：ポップアップのスクロールバーの位置が正しくないレイアウトの異常を修正しました。

**位置のヒント：**
- 残高履歴のスイッチ/保持日数/日次終了時の取得：「設定 → 基本設定 → 残高履歴」にあります。
- 残高履歴グラフの入り口：「設定 → 残高履歴」にあります。

## 3.16.0
- **新機能:**
  - Sub2API（JWTサイト）：Sub2APIサイトタイプを追加し、残高/クォータの照会をサポートしました。「自動識別」によるコンソールログイン状態の読み取りをサポートします。また、「プラグイン管理セッション（マルチアカウント、推奨）」モードをサポートし、アカウントごとに認証を個別に更新でき、同一サイトでのマルチアカウント体験を改善します。
  - 表示設定：「今日の収支を表示」スイッチ（デフォルトでオン）を追加しました。これにより、「今日の消費/収入」などの統計を非表示にし、更新時のログ取得リクエストを停止できます。
- **注意:**
  - Sub2APIは現在、サイトのチェックイン/今日の利用量/収入などの関連機能をサポートしておらず、基本的な残高/クォータの照会のみを提供します。関連機能は、サイトの能力に応じて順次改善される予定です。
  - 「プラグイン管理セッション（マルチアカウント）」では、`refresh_token` がアカウントのプライベートな認証情報として保存され、エクスポート/WebDAVバックアップに含まれます。バックアップファイルとWebDAV認証情報を適切に保管してください。

**位置のヒント：**
- Sub2APIの追加/モードの説明：「設定 → アカウント管理」でアカウントを新規作成/編集し、サイトタイプでSub2APIを選択します。詳細な手順は [FAQ](./faq.md)（「Sub2API」で検索）を参照してください。
- 今日の収支スイッチ：「設定 → 基本設定 → 表示設定」にあります。

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」を追加し、サイトのコンソール/ドキュメント/管理ページなどのショートカットリンクを保存できるようにしました。完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元およびWebDAV自動同期に含まれます。
- **問題の修正:**
  - アカウント更新：重複する「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部のサイトでは更新インターフェースで既に `today_income` を返しています）。
  - 自動更新：更新間隔の最低値を60秒、最小更新間隔保護の最低値を30秒に設定しました。更新後、古い設定は自動的に有効な範囲に修正され、関連するヒントの文言とドキュメントを改善しました。

::: warning 重要：自動更新設定が強制的に調整されます
フィードバックによると、**短すぎる自動更新間隔はサイトのレート制限をトリガーしやすく、サイトに過度な負荷をかける傾向がある**ためです。

v3.15.0では、**自動更新設定が強制的に変更されました**：
- 自動更新機能とプラグインを開いたときの更新機能がオフになりました。引き続き有効にする必要がある場合は、手動で再度有効にする必要があります。
- `更新間隔` は最低60秒、`最小更新間隔保護` は最低30秒です。アップグレード前にこれらのしきい値を下回る設定をしていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい有効な範囲内にある場合は影響ありません。
:::

**位置のヒント：**
- ブックマーク管理：「設定 → ブックマーク管理」にあります。ポップアップ上部で「アカウント / ブックマーク」を切り替えられます。
- 自動更新：「設定 → 基本設定 → 自動更新」にあります。

## 3.14.0
- **新機能:**
  - ウェブページ AI API 機能の可用性テスト（ベータ）：右クリックメニューに「AI API の機能可用性をクイックテスト」を追加しました。現在のウェブページで直接テストパネルを開くことができます。`Base URL` と `API Key` の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Googleなどのインターフェースの基本機能検出を行います（OpenAI互換ではモデルリストの一括取得も可能です）。

  - 自動チェックイン：実行結果リストに、より多くのトラブルシューティングのヒントを追加しました。これには、「一時的な保護回避タブが手動で閉じられた」「Access Tokenが無効」などの一般的な異常に対する処理の提案とドキュメントへの入り口が含まれます。
- **問題の修正:**
  - WebDAV：自動同期をタイマーからブラウザのAlarms APIに移行し、バックグラウンドの休止/省電力ポリシーによる同期漏れの確率を低減しました。

**位置のヒント：**
- AI API テストパネル：任意のウェブページの右クリックメニューで「AI API の機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI API テスト」にあります。
- 自動チェックインのヒント：「設定 → 自動チェックイン」の実行結果リストで確認できます。

## 3.8.0
