# 更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / エクスペリエンスの最適化 / 不具合修正）を記録しています。完全な履歴バージョンとより詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 初めてのユーザーへ
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **更新後に自動で本ページを開く**：「設定 → 一般 → 更新履歴」で「更新後に自動で更新履歴を開く」を制御できます。
- **異常の調査**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.9.0
- **新機能:**
  - アカウント管理：アカウントに「手動残高（USD）」フィールドを追加 — サイトが残高/クォータを自動取得できない場合、表示と統計のために手動で入力できます。
  - アカウント管理：アカウントに「合計残高から除外」スイッチを追加 — 特定のアカウントを「合計残高」の統計から除外するために使用します（更新/チェックインなどの機能には影響しません）。
  - 設定：「更新後に自動で更新履歴を開く」スイッチを追加（更新後に本ページが自動で開く動作をオフにできます）。
  - 設定：ログ設定を追加し、コンソールログの出力有無と最小ログレベルを制御できるようにしました。問題の調査に役立ちます。
  - 自動チェックイン：実行完了後、関連データをリフレッシュし、インターフェースも同期して更新します。

**場所のヒント：**
- アカウントの追加/編集：「設定 → アカウント管理」でアカウントの追加/編集を開きます。
- 更新履歴スイッチ、ログ設定：「設定 → 一般」で設定します。

## 3.8.0
- **新機能:**
  - 利用状況分析：「利用状況分析」ページを追加しました。複数のサイト、複数のアカウントにわたる利用傾向をグラフ化し、「どこで多く使っているか / 費用がかかっているか / 遅くなっているか」を一目で把握でき、コスト管理、照合、問題調査に役立ちます。
  - ダッシュボード内容：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、使用時間ヒートマップ、遅延傾向/ヒストグラムなどを確認できます。
  - 利用履歴の同期：「利用履歴の同期」機能を追加しました。これは「集計された利用データ」（元のログは保存しません）を取得・保存するために使用されます。保持日数、自動同期方法、最小同期間隔を設定でき、「同期ステータス」で各アカウントの同期結果とエラーメッセージを確認できます。
- **使用方法:**
  - まず「設定 → アカウント利用状況」で「利用履歴の同期」を有効にし、必要に応じて設定後、「今すぐ同期」をクリックします。
  - 次に、左側メニューの「利用状況分析」でグラフを確認します。データを保持したい場合や照合を行う場合は「エクスポート」をクリックできます。

## 3.7.0
- **新機能:**
  - 並べ替え：アカウントリストが「収入」による並べ替えをサポートしました。並べ替えの優先順位に「無効なアカウントを最下部に」を追加し、停止中/無効なアカウントが日常の使用を妨げないようにしました。
  - 自動チェックイン：「インターフェースを開いたときに今日のチェックインを事前にトリガー」を追加 — 時間枠内にポップアップ/サイドバー/設定ページを開くと、計画された時間を待たずに、今日のチェックインを事前に実行しようと自動で試みます。
- **不具合修正:**
  - 自動チェックイン：同一アカウントは1日1回のみ実行されます。リトライは失敗したアカウントのみを対象とし、無意味なリクエストと重複した通知を減らします。
- **場所のヒント:**
  - 並べ替えの優先順位：「設定 → アカウント管理」で調整します。
  - 自動チェックインの事前トリガー/リトライ：左側メニューの「自動チェックイン」で設定します。
