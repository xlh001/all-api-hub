# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録します。完全な履歴バージョンと詳細な技術的変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにする方法**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) までお送りください。
:::

## 3.29.0
- **新機能:**
  - 自動サインイン：`一括手動サインインページを開く` を追加しました。失敗したアカウントの手動サインインページを一度に開き、進捗、完了、一部失敗のプロンプトを表示します。`Shift` キーを押しながらクリックすると、新しいウィンドウで開くことができます。
  - フィードバックとサポート：拡張機能ポップアップの `フィードバック` メニューと「バージョン情報」ページに `コミュニティ交流グループ` のエントリを追加しました。WeChat グループ QR コード、Telegram グループなどの交流チャネルを確認できるコミュニティ集計ページに直接ジャンプできます。
- **体験最適化:**
  - ブラウザ言語：繁体字中国語などのブラウザ言語環境との互換性を追加し、デフォルト言語のフォールバックロジックを調整しました。初回起動時の言語認識がより安定しました。
  - ドキュメントリンク：多言語ドキュメントのジャンプルールを最適化しました。サポートされていない言語環境は自動的に英語ドキュメントにフォールバックし、誤った言語ページにジャンプする状況を減らしました。
  - 多言語：一部の翻訳値の取得方法を統一し、一部の不要なテキストをクリーンアップして、多言語体験を最適化しました。
- **問題修正:**
  - サイドバー：Chrome/Edge (MV3) でツールバーをクリックした後にサイドバーが偶発的に開かなくなる問題を修正しました。
  - キー管理：マスクされたキーを表示する際に、新しいロードステータスを追加し、完全なキーの解析表示をサポートしました。再度展開しても、繰り返しリクエストされません。
  - ブラウザバックグラウンド：拡張機能が一時停止した場合、一時ページをより迅速にクリーンアップし、一時ページが残る問題を減らしました。

**場所のヒント:**
- 一括手動サインイン：`設定 → 自動サインイン` ページの失敗したアカウント関連操作にあります。
- コミュニティエントリ：拡張機能ポップアップの右上にある `フィードバック` メニュー、および `設定 → バージョン情報` の `フィードバックとサポート` セクションにあります。
- キー表示：`設定 → キー管理` のキーリストで表示/非表示ボタンをクリックします。

## 3.28.0
- **新機能:**
  - API クレデンシャル：`CLI 互換性検証` 操作を追加しました。検証時にモデル ID を自動取得または手動入力でき、現在一時的な `API タイプ` オーバーライドを使用しているかどうかを明確に示し、一度限りのテスト結果を保存済み設定と誤認しないようにします。
  - API クレデンシャル / モデルリスト：`API クレデンシャル` から対応する `モデルリスト` データソースにワンクリックでジャンプできるようになりました。`モデルリスト` も API クレデンシャルをデータソースとして直接使用して、モデルディレクトリと検証結果を表示でき、サイトアカウントを事前に作成する必要はありません。
  - キー管理：`キー管理` にホストサイトのチャネルステータス、マッチングシグナル、ジャンプエントリが表示されるようになりました。また、キーを `API クレデンシャル` に保存する際に、より明確な名前が生成され、後での検索と再利用が容易になります。
  - New API ホストサイト：ホストサイト設定にログイン補助情報（ユーザー名、パスワード、オプションの TOTP）とセッション検証を追加しました。ステータスを検証したり、実際のチャネルキーを読み取ったりする必要がある場合は、拡張機能内で直接検証を完了できます。
  - ホストサイトマッチング：チャネル識別を `URL`、キー、モデルなどのシグナルを総合してソートするように変更しました。バックエンドがマスクされたトークンのみを返すシナリオでも、チャネルステータスの判断、コピー、統合操作を継続して完了できます。
  - 初回使用：ウェルカム / 権限ガイドポップアップに言語セレクターを追加しました。初回起動時にインターフェース言語を切り替え、後続の使用設定を記憶できます。
- **体験最適化:**
  - Veloera：現在、`Base URL` に基づくチャネルの特定とステータス検出をサポートしていないシナリオに対して、関連エントリは自動的に非表示または無効になり、説明が表示されるため、誤ってクリックしても結果が得られないことによる混乱を減らしました。
- **問題修正:**
  - 言語：起動時にブラウザ検出言語に安定して従わない問題を修正し、インターフェーステキストと日付/時刻のローカライズを同期して修正しました。
  - 権限ガイド：権限説明ポップアップのボタンレイアウトを最適化しました。小ウィンドウやボタンの折り返し時により整然とし、クリックしやすくなりました。

**場所のヒント:**
- API クレデンシャル：`設定 → API クレデンシャル` で、`CLI 互換性検証`、`モデル管理で開く` などの操作を使用できます。
- モデルリストデータソース：`設定 → モデルリスト` の上部にあるデータソース選択エリアで、`API クレデンシャル` に切り替えることができます。
- ホストサイトチャネルステータス：`設定 → キー管理` で各キーのホストサイトステータスとマッチングプロンプトを確認します。
- New API ホストサイトログイン補助：`設定 → 自建サイト管理` の `New API 集成設定` セクションにあります。
- 初回言語選択：拡張機能を初回起動した際に表示されるウェルカム / 権限ガイドポップアップにあります。

## 3.27.0

**場所のヒント:**

- フィードバックエントリ：拡張機能ポップアップのタイトルバーにある `フィードバック` ボタン、および `設定 → バージョン情報` の `フィードバックとサポート` セクションにあります。

## 3.26.0
- **新機能:**
  - アカウント管理：`対応チャネルを特定` クイック操作を追加しました。ホストサイトアカウントからチャネル管理リストにフィルタリングしてワンクリックでジャンプできます。また、「重複アカウント追加前のリマインダー」を有効にすることで、誤って重複アカウントを追加するのを減らしました。
  - 重複アカウントクリーンアップ：`重複アカウントクリーンアップ` ツールを追加しました。URLソースサイト + ユーザー ID でスキャンして重複項目を削除でき、重複アカウントの一括整理がより便利になりました。
  - アカウント管理：無効化されたアカウントの操作メニューに直接削除エントリを追加し、無効なアカウントのクリーンアップ手順を減らしました。
  - API クレデンシャル：`API クレデンシャル` ページは、設定ナビゲーションと拡張機能ポップアップから直接アクセスできます。設定のエクスポート時にもトークンメモが保持され、複数のツール間での移行が容易になります。
  - WebDAV：同期データ選択を追加しました。必要に応じて `アカウント`、`ブックマーク`、`API クレデンシャル` などの共有データのみを同期でき、複数デバイス間での不要な上書きを減らしました。
  - Sub2API：`Sub2API` アカウントのキー管理をサポートしました。キーの表示、作成、編集、削除を直接行えます。
  - CLIProxyAPI：インポート時にプロバイダータイプ選択を追加し、一般的なエンドポイントアドレスを自動的に正規化して、手動での URL 変更を減らしました。
- **体験最適化:**
  - 交換アシスタント：交換成功後、アカウント残高が自動的に更新され、手動更新して結果を確認する手順を減らしました。
- **問題修正:**
  - 自動サインイン：固定時間サインインに、より安定した補完実行メカニズムを追加し、拡張機能の更新などによる実行ウィンドウの欠落によるサインイン漏れを減らしました。
  - 自動認識：アカウント再認識後にカスタムサインイン設定が失われる可能性のある問題を修正し、設定の意図しない損失を防ぎました。
  - 自動サインイン：Turnstile アシストまたは手動サインインプロンプトが一部のアカウントで誤って `外部サインインアドレス` を使用する問題を修正しました。代わりに常にサイトのデフォルトサインインページを開くようにし、誤ったページへのジャンプやサインイン完了不可を減らしました。
  - ホストサイト：ホストサイトへのデータインポートまたは同期時に、ターゲットサイトのデフォルトグループをより優先的に使用し、グループの不一致による異常を減らしました。

::: warning 注意
- WebDAV の `同期データ選択` とアカウント自動更新などのローカルデバイス設定は、WebDAV を介してデバイス間で上書きされなくなりました。
:::

**場所のヒント:**
- 重複アカウントリマインダー：`設定 → 基本設定 → アカウント管理` の `重複アカウント追加前のリマインダー` にあります。
- 重複アカウントクリーンアップ：`設定 → アカウント管理` のページツールバーにあります。
- チャネル特定：`設定 → アカウント管理` の個々のアカウント操作メニューにあります。
- API クレデンシャル：`設定 → API クレデンシャル`。拡張機能ポップアップでも `API クレデンシャル` ビューに切り替えられます。
- WebDAV 同期データ選択：`設定 → インポート/エクスポート` の `WebDAV 設定` にあります。

## 3.25.0
- **新機能:**
  - 自動サインイン：Cloudflare Turnstile（過盾/人機認証）シナリオをサポートしました。サイトが Turnstile 認証を要求する場合、一時ページで認証を試行してからサインインを続行し、必要に応じて手動で開くことができるサインインリンクとプロンプトを提供します。
  - CC Switch：`Codex` にエクスポートする際に、基本アドレスのデフォルト値に `/v1` インターフェースアドレスを自動補完します（インターフェースアドレスを手動で変更していない場合）。これにより、直接インポート後にインターフェースが使用できなくなる問題を減らしました。
  - モデルリダイレクト：オプションのスイッチ `同期後に無効なリダイレクトターゲットをクリーンアップ` を追加しました。モデル同期リフレッシュ後に、存在しないモデルを指す `model_mapping` のマッピングを自動的に削除します（危険な操作、デフォルトで無効）。
- **体験最適化:**
  - 一時ウィンドウ：チャレンジ/ログインページとしてより正確に識別し、誤判定や不要な干渉を減らしました。
- **問題修正:**
  - Cookie 認証：現在の実際の動作と能力に一致するようにテキスト説明を修正し、誤解を減らしました。
  - サイドバー：小ウィンドウでサイドバーの最下部メニュー項目が見えなくなる問題を修正しました。

**場所のヒント:**
- Turnstile 認証：`設定 → 自動サインイン` の実行結果で新しいプロンプトを確認します。
- CC Switch エクスポート：`設定 → キー管理` でキーを選択し、`CC Switch にエクスポート` をクリックし、ターゲットアプリケーションを `Codex` に選択します。
- モデルリダイレクトクリーンアップ：`設定 → 基本設定 → モデルリダイレクト` で `同期後に無効なリダイレクトターゲットをクリーンアップ` を有効にします。

## 3.24.0
- **新機能:**
  - 更新履歴：プラグイン更新後、ブラウザの新しいタブページが自動的に開かなくなりました。代わりに、プラグインインターフェースを初めて開いた際に、プラグイン内でポップアップ表示され、完全な更新履歴をワンクリックで開くことができます。
  - LDOH：アカウントリストに `LDOH で表示`（LDOH アイコン）クイックエントリを追加しました。LDOH に直接ジャンプし、対応するサイトに自動的にフィルタリングします。アカウント追加時にも `LDOH サイトリストを開く` エントリを提供し、サイトを見つけやすくしました。
- **体験最適化:**
  - ドキュメントリンク：プラグインからドキュメント/更新履歴を開く際に、現在のプラグイン言語に応じて対応する言語のドキュメントページにジャンプします。

**場所のヒント:**
- 更新履歴スイッチ：`設定 → 一般 → 更新履歴` の `更新後に更新内容を自動表示` にあります。
- 更新履歴ポップアップ：プラグイン更新後、`拡張機能ポップアップ / 設定ページ / サイドバー` を初めて開いた際に自動的にポップアップします（各バージョンにつき 1 回）。
- LDOH クイックエントリ：`アカウント管理` のアカウントリストのサイト名の右側（LDOH アイコン、`LDOH で表示` と表示）。アカウント追加ダイアログでも `LDOH サイトリストを開く` をクリックできます。

## 3.22.0
- **新機能:**
  - モデルリスト：`モデル対応キー` ツール（鍵アイコン）を追加しました。現在のモデルに利用可能なキーが存在するかどうかを確認できます。利用可能なキーがない場合、モデルの利用可能なグループ別にデフォルトキーをワンクリックで作成するか、カスタム作成プロセスに入り、キーのワンクリックコピーもサポートします。
  - スナップショット共有：`概要スナップショット / アカウントスナップショット` のワンクリック共有をサポートしました。優先的に画像をクリップボードにコピーし、サポートしない場合は PNG を自動ダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトルテキストのワンクリックコピーも可能です。
- **体験最適化:**
  - 無効化されたアカウント：`残高履歴 / 利用量分析 / 利用量同期` などのリフレッシュと定期タスクで、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーを減らしました。
- **問題修正:**
  - ボタン：ボタンが「ロード中」で、かつ左アイコンが表示されている場合にスピナーが見えなくなる問題を修正しました。

**場所のヒント:**
- モデル対応キー：`設定 → モデルリスト` で、モデル名の右側にある鍵アイコン（`モデル対応キー`）をクリックします。
- 概要スナップショット共有：拡張機能ポップアップの概要ページタイトルバーの右側にあるボタン（`概要スナップショット共有`）です。
- アカウントスナップショット共有：`設定 → アカウント管理` の個別アカウント操作メニュー（`アカウントスナップショット共有`）です。

## 3.21.0
- **新機能:**

- **体験最適化:**

  - キーのコピー：アカウントにまだキーがない場合、ポップアップに「デフォルトキーのクイック作成 / カスタムキー作成」エントリが表示され、行き来する手間を減らしました。

**場所のヒント:**

- `リフレッシュ間隔` の最小値を 60 秒、`最小リフレッシュ間隔保護` の最小値を 30 秒にしました。アップグレード前に上記閾値を下回る設定をしていた場合、アップグレード後に自動的に最小値に引き上げられます。以前の設定値が新しい合法範囲内であれば、影響はありません。
:::
