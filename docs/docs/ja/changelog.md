# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / エクスペリエンス改善 / 問題修正）を記録しています。完全な履歴バージョンとより詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 初めての方へ
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **更新後にこのページを自動的に開く**：「設定 → 一般 → 更新履歴」で「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.14.0
- **新機能:**
  - ウェブAI API機能の可用性テスト（ベータ版）：右クリックメニューに「AI APIの機能可用性をクイックテスト」を追加しました。現在のウェブページで直接テストパネルを開くことができます。`Base URL` と `API Key` の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Google などのインターフェースの基本機能検出を行います（OpenAI互換ではモデルリストの一括取得も可能です）。
  - （オプション）自動検出：「設定 → AI APIテスト」で有効にし、URLホワイトリストを設定できます。ホワイトリストに登録されたページで利用可能な `Base URL` と `API Key` が検出された場合、まず確認プロンプトが表示され、確認後にテストパネルが開きます（デフォルトではオフで、Keyは保存されません）。
  - 自動チェックイン：実行結果リストに、より多くのトラブルシューティングヒントを追加しました。「一時的なシールド通過タブが手動で閉じられた」「Access Tokenが無効」などの一般的な異常に対する処理の提案とドキュメントへの入り口が含まれます。
- **問題修正:**
  - WebDAV：自動同期をタイマーからブラウザのAlarms APIに移行し、バックグラウンドでの休止/省電力ポリシーによる同期漏れの確率を低減しました。

**場所のヒント：**
- AI APIテストパネル：任意のウェブページの右クリックメニューで「AI APIの機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI APIテスト」にあります。
- 自動チェックインのヒント：「設定 → 自動チェックイン」の実行結果リストで確認できます。

## 3.13.0
- **新機能:**

  - Cookie認証：Cookieキャッシュメカニズムを削除し、Cookie更新後に古い値を読み込むことによる異常を減らしました。

**場所のヒント：**
- チェックインステータス期限切れのヒント：「設定 → アカウント管理」のアカウントリストで、サイト情報の右側にあるチェックインアイコンの場所にあります。

## 3.8.0
- **新機能:**
  - 使用量分析：「使用量分析」ページを追加しました。複数のサイト、複数のアカウントの使用量トレンドをグラフ化し、「どこで多く使われているか / 費用がかかっているか / 遅くなっているか」を一目で把握でき、コスト管理、照合、問題のトラブルシューティングに役立ちます。
  - ダッシュボードコンテンツ：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、使用時間ホットスポット、遅延トレンド/ヒストグラムなどを表示できます。
  - 使用量履歴同期：新しい「使用量履歴同期」機能を追加しました。これは「集計された使用量データ」を取得して保存するために使用されます（元のログは保存されません）。保持日数、自動同期方法、最小同期間隔を設定でき、「同期ステータス」で各アカウントの同期結果とエラーヒントを確認できます。
- **使い方:**
  - まず「設定 → アカウント使用量」で「使用量履歴同期」を有効にし、必要に応じて設定した後、「今すぐ同期」をクリックします。
  - 次に、左側のメニュー「使用量分析」でグラフを確認します。保持または照合が必要な場合は「エクスポート」をクリックできます。
