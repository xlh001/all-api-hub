# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の最適化 / 不具合修正）を記録しています。完全な履歴バージョンとより詳細な技術的な変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 初めての方へ
- **現在のバージョンの確認方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **更新後に自動で本ページを開く**：「設定 → 一般 → 更新履歴」で「更新後に自動で更新履歴を開く」を制御できます。
- **異常の調査**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.11.0
- **新機能:**
  - New API 渠道管理：新規「重複チャネル」アラートを追加 — 既存の同じ/類似のチャネルが検出された場合、警告ダイアログがポップアップ表示され、作成を続行するかキャンセルするかを選択できます（エラーToastで作成を阻止することはなくなりました）。
- **問題修复:**
  - アカウント管理：サイト名などのテキストが長すぎるためにレイアウトがオーバーフローする問題を修正し、自動的に切り詰めて表示するように変更しました。

## 3.10.0
- **新機能:**
  - アカウント管理：シークレットモードでサイトリンクをクリックすると、現在のシークレットウィンドウ内で開きます（シークレットログイン状態を維持しやすくなります）。
  - アカウント管理：無効化されたアカウントもサイトリンクのクリックをサポートし、ブックマークとして使用しやすくなります。
  - 用量分析：単一アカウントのみの場合、グラフとリストでサイト名（ユーザー名ではなく）を優先的に表示し、情報がより直感的になります。
  - 過盾アシスタント：一時的な過盾ウィンドウに CAP（cap.js）Pow認証のサポートが追加され、通過率が向上しました。
- **問題修复:**
  - 兌換助手：クリップボードから交換コードを優先的に読み取り、トリガーの精度を向上させました。
  - 提示浮层：Toasterの階層問題を修正し、アラートがウェブページに隠れなくなりました。

**位置のヒント：**
- サイトリンク：「設定 → アカウント管理」のアカウントリストでサイト名をクリックします。
- 過盾アシスタント：[Cloudflare 過盾アシスタント](./cloudflare-helper.md) を参照してください。

## 3.9.0
- **新機能:**
  - アカウント管理：アカウントに「手動残高（USD）」フィールドを追加 — サイトが自動で残高/クォータを取得できない場合に、表示および統計のために手動で入力できます。
  - アカウント管理：アカウントに「総残高から除外」スイッチを追加 — 特定のアカウントを「総残高」の統計から除外するために使用されます（更新/チェックインなどの機能には影響しません）。
  - 設定：「更新後に自動で更新履歴を開く」スイッチを追加（更新後にこのページが自動で開く動作をオフにできます）。
  - 設定：ログ設定を追加し、コンソールログの出力有無と最小ログレベルを制御できるようにし、問題の調査を容易にしました。
  - 自動チェックイン：実行完了後、関連データを更新し、インターフェースも同期的に更新されます。

**位置のヒント：**
- アカウントの追加/編集：「設定 → アカウント管理」でアカウントの追加/編集を開きます。
- 更新履歴スイッチ、ログ設定：「設定 → 一般」で設定します。

## 3.8.0
- **新機能:**
  - 用量分析：「利用状況分析」ページを追加し、複数のサイト、複数のアカウントの利用傾向をグラフ化して、「どこが多く使われているか / 多く支払っているか / 遅くなっているか」を一目で把握できるようにし、コスト管理、照合、問題調査を容易にします。
  - 看板内容：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、使用時間ヒートマップ、遅延傾向/ヒストグラムなどを確認できます。
  - 用量历史同步：「利用履歴の同期」機能を追加しました。これは「集計された利用状況データ」を取得・保存するためのものです（元のログは保存されません）。保持日数、自動同期方法、最小同期間隔を設定でき、「同期ステータス」で各アカウントの同期結果とエラーメッセージを確認できます。
- **怎么用:**
  - まず「設定 → アカウント利用状況」で「利用履歴の同期」を有効にし、必要に応じて設定した後、「今すぐ同期」をクリックします。
  - 次に、左側のメニュー「用量分析」でグラフを確認します。データを保持したり、照合したりする必要がある場合は、「エクスポート」をクリックできます。

## 3.7.0
- **新機能:**
  - ソート：アカウントリストで「収入」によるソートをサポートしました。ソートの優先順位に「無効アカウントの最下位表示」が追加され、停止/無効なアカウントが日常的な使用を妨げなくなりました。
  - 自動チェックイン：新規「インターフェースを開いたときに今日のチェックインを事前にトリガーする」機能を追加 — 時間枠内でポップアップ/サイドバー/設定ページを開いたときに、自動的に今日のチェックインを一度試行し、予定された時間を待つ必要がなくなります。
- **問題修复:**
  - 自動チェックイン：同一アカウントは1日1回のみ実行されます。リトライは失敗したアカウントに対してのみ行われ、無意味なリクエストや重複した通知を減らします。
- **位置提示:**
  - ソート優先順位：「設定 → アカウント管理」で調整します。
  - 自動チェックインの事前トリガー/リトライ：左側のメニュー「自動チェックイン」で設定します。
