# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録しています。完全な履歴バージョンと詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにするには**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.27.0
- **新機能:**
  - アカウント管理：アカウントリストに有効状態でのフィルタリングが追加され、`有効` / `無効` アカウントを素早く切り替えて表示できるようになりました。無効なアカウントを一括整理するのがより便利になりました。
  - フィードバックとサポート：拡張機能のポップアップタイトルバーに `フィードバック` のクイックエントリーが追加されました。「バージョン情報」ページにも `フィードバックとサポート` セクションが追加され、GitHub の問題報告、機能提案、ディスカッションエリアに直接アクセスできます。
- **体験最適化:**
  - アカウント表示：複数のアカウントのサイト名が同じ場合、自動的にユーザー名が補完され `サイト名 · ユーザー名` と表示されるようになり、リスト、検索、セレクター、統計ビューでの区別が容易になりました。
- **問題修正:**
  - サイドバー：サイドバーのサポート検出がさらに最適化されました。ブラウザやモバイル環境でサイドバーがサポートされていない場合、無効なエントリーは自動的に非表示になるか、設定ページにフォールバックし、応答のないクリックを減らします。

**場所のヒント：**
- アカウント状態フィルタリング：`設定 → アカウント管理` のリスト上部にあるフィルタリングエリア。
- フィードバックエントリー：拡張機能ポップアップのタイトルバーにある `フィードバック` ボタン、および `設定 → バージョン情報` の `フィードバックとサポート` セクション。

## 3.26.0
- **新機能:**
  - アカウント管理：`対応するチャネルを特定` のクイック操作が追加され、ホストされたサイトのアカウントから対応する「チャネル管理」リストにフィルタリング付きで一括ジャンプできるようになりました。「重複アカウント追加前のリマインダー」を有効にするオプションも追加され、重複アカウントの誤追加を減らします。
  - 重複アカウントクリーンアップ：`重複アカウントクリーンアップ` ツールが追加され、URLのソースサイト + ユーザーIDで重複項目をスキャンして削除できるようになり、重複アカウントの一括整理がより便利になりました。
  - アカウント管理：無効化されたアカウントの操作メニューに直接削除のエントリーが追加され、無効なアカウントの整理手順が省かれました。
  - API認証情報：`API認証情報` ページが設定ナビゲーションと拡張機能ポップアップから直接アクセス可能になりました。設定のエクスポート時にトークンメモも保持されるようになり、複数のツール間での移行が容易になりました。
  - WebDAV：同期データの選択が追加され、必要に応じて `アカウント`、`ブックマーク`、`API認証情報` などの共有データのみを同期できるようになり、マルチデバイス間での不要な上書きが減少しました。
  - Sub2API：`Sub2API` アカウントのキー管理サポートが追加され、キーの表示、作成、編集、削除が直接行えるようになりました。
  - CLIProxyAPI：インポート時にプロバイダータイプの選択が追加され、一般的なエンドポイントアドレスが自動的に正規化されるため、手動でのURL変更が減少しました。
- **体験最適化:**
  - 交換アシスタント：交換成功後にアカウント残高が自動的に更新され、手動での更新確認手順が減少しました。
- **問題修正:**
  - 自動サインイン：固定時間サインインに、より安定した補完実行メカニズムが追加され、拡張機能の更新などによる実行ウィンドウの欠落によるサインイン漏れが減少しました。
  - 自動認識：アカウントの再認識後にカスタムサインイン設定が失われる可能性があった問題を修正し、設定の意図しない消失を防ぎました。
  - 自動サインイン：Turnstile サポートまたは手動サインインで、一部のアカウントで `外部サインインURL` が誤って使用されていた問題を修正しました。代わりに常にサイトのデフォルトサインインページを開くようになり、誤ったページへのジャンプやサインイン完了不可が減少しました。
  - ホストされたサイト：ホストされたサイトへのデータインポートまたは同期時に、ターゲットサイトのデフォルトグループがより優先されるようになり、グループの不一致による異常が減少しました。

::: warning 注意
- WebDAV の `同期データ選択` とアカウント自動更新などのデバイスローカル設定は、WebDAV を介してデバイス間で相互に上書きされなくなりました。
:::

**場所のヒント：**
- 重複アカウントリマインダー：`設定 → 基本設定 → アカウント管理` の `重複アカウント追加前のリマインダー`。
- 重複アカウントクリーンアップ：`設定 → アカウント管理` のページツールバー。
- チャネルの特定：`設定 → アカウント管理` の個々のアカウントの操作メニュー。
- API認証情報：`設定 → API認証情報`。拡張機能ポップアップでも `API認証情報` ビューに切り替え可能です。
- WebDAV同期データ選択：`設定 → インポート/エクスポート` の `WebDAV設定`。

## 3.25.0
- **新機能:**
  - 自動サインイン：Cloudflare Turnstile（アンチボット/人間認証）シナリオをサポートしました。サイトが Turnstile 認証を要求する場合、一時的なページで認証を完了してからサインインを続行しようとします。必要に応じて、手動で開くことができるサインインリンクとヒントも提供します。
  - CC Switch：`Codex` へのエクスポート時に、基本アドレスのデフォルト値に `/v1` が自動的に補完されます（インターフェースアドレスを手動で変更していない場合）。これにより、直接インポート後のインターフェースが使用不可になる問題が減少します。
  - モデルリダイレクト：オプションのスイッチ `同期後に無効なリダイレクトターゲットをクリーンアップ` が追加され、モデル同期更新後に `model_mapping` 内の存在しないモデルへのマッピングが自動的に削除されます（危険な操作、デフォルトで無効）。
- **体験最適化:**
  - 一時ウィンドウ：チャレンジ/ログインページとしてより正確に認識し、誤判定や不要な中断を減らします。
- **問題修正:**
  - Cookie認証：現在の実際の動作と能力に合致するように説明文を修正し、誤解を減らします。
  - サイドバー：小ウィンドウでサイドバーの最下部メニュー項目が見えなくなる問題を修正しました。

**場所のヒント：**
- Turnstile 認証：`設定 → 自動サインイン` の実行結果で新しいヒントを確認します。
- CC Switch エクスポート：`設定 → キー管理` でキーを選択し、`CC Switch にエクスポート` をクリックし、ターゲットアプリケーションとして `Codex` を選択します。
- モデルリダイレクトクリーンアップ：`設定 → 基本設定 → モデルリダイレクト` で `同期後に無効なリダイレクトターゲットをクリーンアップ` を有効にします。

## 3.24.0
- **新機能:**
  - 更新履歴：プラグイン更新後にブラウザの新しいタブで自動的に開くことはなくなりました。代わりに、プラグインインターフェースを初めて開いた際に、プラグイン内でポップアップ表示され、更新内容と完全な更新履歴を開くためのワンクリックボタンを提供します。
  - LDOH：アカウントリストに `LDOH で表示`（LDOH アイコン）のクイックエントリーが追加され、LDOH に直接ジャンプして対応するサイトに自動的にフィルタリングされます。アカウント追加時にも `LDOH サイトリストを開く` エントリーを提供し、サイトの検索を容易にします。
- **体験最適化:**
  - ドキュメントリンク：プラグインからドキュメント/更新履歴を開く際に、現在のプラグイン言語に応じて対応する言語のドキュメントページに自動的にジャンプします。

**場所のヒント：**
- 更新履歴スイッチ：`設定 → 一般 → 更新履歴` の `更新後に更新内容を自動表示`。
- 更新履歴ポップアップ：プラグイン更新後、初めて「拡張機能ポップアップ / 設定ページ / サイドバー」を開いた際に自動的にポップアップします（各バージョンにつき一度のみ）。
- LDOH クイックエントリー：`アカウント管理` のアカウントリストのサイト名の右側（LDOH アイコン、`LDOH で表示` と表示）。アカウント追加ダイアログでも `LDOH サイトリストを開く` をクリックできます。

## 3.22.0
- **新機能:**
  - モデルリスト：「モデル対応キー」ツール（鍵アイコン）が追加され、現在のモデルに利用可能なキーが存在するかどうかを確認できます。利用可能なキーがない場合、モデルの利用可能なグループごとにデフォルトキーを一括作成するか、カスタム作成プロセスに進むことができます。キーの一括コピーもサポートします。
  - スナップショット共有：「概要スナップショット / アカウントスナップショット」の一括共有をサポートしました。優先的にクリップボードに画像をコピーし、サポートされていない場合は PNG を自動ダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトル文案の一括コピーも可能です。
- **体験最適化:**
  - アカウント無効化：`残高履歴 / 利用量分析 / 利用量同期` などの更新と定期タスクで、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーが減少しました。
- **問題修正:**
  - ボタン：ボタンが「ロード中」で、かつ左アイコンが表示されている場合にスピナーが見えなくなる問題を修正しました。

**場所のヒント：**
- モデル対応キー：`設定 → モデルリスト` で、モデル名の右側にある鍵アイコン（`モデル対応キー`）をクリックします。
- 概要スナップショット共有：拡張機能ポップアップの概要ページタイトルバーの右側にあるボタン（`概要スナップショットを共有`）。
- アカウントスナップショット共有：`設定 → アカウント管理` の個々のアカウントの操作メニュー（`アカウントスナップショットを共有`）。

## 3.21.0
- **新機能:**
  - API認証情報：「API認証情報」ページが追加され、「アカウントがなく、`Base URL` + `API Key` のみ」という使用シナリオに適しています。タグ/メモの一元管理をサポートし、可用性検証とクイックエクスポート（例：Cherry Studio / CC Switch / Kilo Code / CLIProxyAPI / Claude Code Router）を直接行えるため、コピー＆ペーストの手間が減少しました。

  - 自社サイト管理：ホストされたサイトに `Done Hub` が追加され、管理者認証情報を設定して「チャネル管理」「モデル同期」などの機能に使用できるようになりました。
- **体験最適化:**
  - 右クリックメニュー：「交換アシスタント」と「AI API テスト」のエントリーが個別に有効/無効化できるようになり、切り替え後に即座に反映されます。
  - キーコピー：アカウントにまだキーがない場合、ポップアップに「デフォルトキーをクイック作成 / カスタムキーを作成」のエントリーが表示され、行き来する手間が減少しました。

**場所のヒント：**
- API認証情報：`設定 → API認証情報`。
- 残高履歴：`設定 → 残高履歴`。
- Done Hub 設定：`設定 → 基本設定 → 自社サイト管理` で `Done Hub` を選択し、「Done Hub 統合設定」を入力します。
- 右クリックメニューエントリーのスイッチ：`設定 → 基本設定 → サインインと交換 / AI API テスト` の各「ブラウザの右クリックメニューに表示」。
- キーコピーポップアップ：`アカウント管理` などのページで「キーをコピー」をクリックして開きます。

::: warning 重要：自動更新設定は強制的に調整されます
フィードバックによると、**短すぎる自動更新間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける可能性があります**。

v3.15.0 では**自動更新設定を強制的に変更しました**：
- 自動更新とプラグインを開いたときの更新機能を無効にしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
- `更新間隔` の最小値は 60 秒、`最小更新間隔保護` の最小値は 30 秒です。アップグレード前に上記閾値を下回る設定をしていた場合、アップグレード後に自動的に最小値に引き上げられます。以前の設定値が新しい合法な範囲内であれば、影響はありません。
:::
