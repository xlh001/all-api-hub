# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / エクスペリエンス最適化 / 問題修正）を記録しています。完全な履歴バージョンとより詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 新規ユーザーの方へ
- **現在のバージョンの確認方法**: 拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **更新後にこのページを自動的に開く**: 「設定 → 一般 → 更新履歴」で「更新後に更新履歴を自動的に開く」を制御できます。
- **異常のトラブルシューティング**: 「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を [Issues](https://github.com/qixing-jk/all-api-hub/issues) に添付してください。
:::

## 3.15.1
- **問題修正:**
  - Chrome/Edge（MV3）：バックグラウンドのService Workerの安定性を向上させ、非同期の定期タスク（WebDAV自動同期 / 使用量同期 / モデル同期 / 自動チェックインなど）がバックグラウンドの早期終了によって実行漏れする問題を軽減しました。また、ブラウザ再起動後にこれらの定期タスクを自動的に再開します。
  - インストール/更新/起動プロセス：初期化段階でのエラー処理とログ機能を強化し、予期せぬクラッシュを回避します。

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」を新設し、サイトのコンソール/ドキュメント/管理ページなどのショートカットリンクを保存できるようになりました。完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルター、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元およびWebDAV自動同期に含まれます。
- **問題修正:**
  - アカウント更新：重複する「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部のサイトでは更新インターフェースで `today_income` が返されるようになりました）。
  - 自動更新：更新間隔の最低値を60秒、最小更新間隔保護の最低値を30秒に設定しました。更新後、古い設定は自動的に有効な範囲に修正され、関連するヒントとドキュメントが改善されました。

::: warning 重要：自動更新設定が強制的に調整されます
フィードバックによると、**短すぎる自動更新間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける傾向がある**ため、

v3.15.0 では**自動更新設定が強制的に変更されました**：
- 自動更新とプラグイン起動時の更新機能がオフになりました。引き続き有効にする必要がある場合は、手動で再度有効にする必要があります。
- `更新間隔` は最低60秒、`最小更新間隔保護` は最低30秒です。アップグレード前にこれらのしきい値を下回る設定をしていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい有効な範囲内であった場合は影響ありません。
:::

**設定場所:**
- ブックマーク管理：「設定 → ブックマーク管理」にあります。ポップアップ上部で「アカウント / ブックマーク」を切り替えられます。
- 自動更新：「設定 → 基本設定 → 自動更新」にあります。

## 3.14.0
- **新機能:**
  - ウェブAI API機能可用性テスト（ベータ版）：右クリックメニューに「AI APIの機能可用性をクイックテスト」を追加しました。現在のウェブページで直接テストパネルを開くことができます。`Base URL` と `API Key` の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Google などのインターフェースの基本機能検出を行います（OpenAI互換ではモデルリストの一括取得も可能です）。
  - （オプション）自動検出：「設定 → AI APIテスト」で有効にし、URLホワイトリストを設定できます。ホワイトリスト内のページで利用可能な `Base URL` と `API Key` が検出された場合、まず確認プロンプトが表示され、確認後にテストパネルが開きます（デフォルトではオフで、Keyは保存されません）。
  - 自動チェックイン：実行結果リストに、より多くのトラブルシューティングヒントを追加しました。「一時的なシールド回避タブが手動で閉じられた」「Access Tokenが無効」など、一般的な異常の処理アドバイスとドキュメントへの入り口が含まれます。
- **問題修正:**
  - WebDAV：自動同期をタイマーからブラウザのAlarms APIに移行し、バックグラウンドの休止/省電力ポリシーによる同期漏れの確率を低減しました。

**設定場所:**
- AI APIテストパネル：任意のウェブページで右クリックメニューから「AI APIの機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI APIテスト」にあります。
- 自動チェックインヒント：「設定 → 自動チェックイン」の実行結果リストで確認できます。

## 3.8.0
