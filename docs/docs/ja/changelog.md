# 更新ログ

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録します。完全な履歴バージョンと詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開き、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにする方法**：「設定 → 一般 → 更新ログ」で、「更新後に更新ログを自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) までお送りください。
:::

## 3.23.0
- **新機能:**
  - 自動サインイン：アカウント操作メニューに `クイックサインイン` を追加しました。これにより、単一のアカウントに対して即座にサインインを実行し、完了後にステータスを更新できます。
  - キー管理：`すべてのアカウント` ビューを追加しました。アカウントごとにキーを集約表示し、アカウントを横断した検索とコピーを容易にします。
  - モデルリダイレクト：`モデルリダイレクトマッピングをクリア` の一括操作を追加しました。チャネルを選択し、再確認後に `model_mapping` を迅速にリセットできます（取り消し不可）。
- **体験最適化:**
  - New API チャネル管理：チャネルリストの URL がクリック可能になり、検索体験を最適化しました。
- **問題修正:**
  - チャネル管理：チャネルダイアログの `優先度` のヒント文言の不正確さを修正しました。
  - モデルリダイレクト：自動生成マッピングに「バージョンガード」を追加し、クロスバージョンでの誤ったマッチングを防ぎます。
  - サイドバー：実行環境がサイドバーをサポートしていない場合、ポップアップ/設定ページを開くように自動フォールバックし、クリックしても応答がない問題を回避します。

## 3.22.0
- **新機能:**
  - モデルリスト：`モデル対応キー` ツール（鍵アイコン）を追加しました。現在利用可能なキーがモデルに存在するかどうかを確認できます。利用可能なキーがない場合、モデルの利用可能なグループに基づいてデフォルトキーをワンクリックで作成するか、カスタム作成フローに進むことができます。キーのワンクリックコピーもサポートしています。
  - スナップショット共有：`概要スナップショット / アカウントスナップショット` をワンクリックで共有できるようになりました。優先的に画像をクリップボードにコピーし、サポートされていない場合は PNG を自動ダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトル文言のワンクリックコピーも可能です。
- **体験最適化:**
  - アカウント無効化：`残高履歴 / 使用量分析 / 使用量同期` などの更新および定期タスクで、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーを削減します。
- **問題修正:**
  - ボタン：ボタンが「読み込み中」で左側アイコンも表示されている場合にスピナーが見えなくなる問題を修正しました。

**場所のヒント:**
- モデル対応キー：`設定 → モデルリスト` で、モデル名の右側にある鍵アイコン（`モデル対応キー`）をクリックします。
- 概要スナップショット共有：拡張機能ポップアップの概要ページタイトルバーの右側にあるボタン（`概要スナップショット共有`）。
- アカウントスナップショット共有：`設定 → アカウント管理` の個々のアカウントの操作メニュー（`アカウントスナップショット共有`）。

## 3.16.0
- **新機能:**
  - Sub2API（JWT サイト）：Sub2API サイトタイプを追加し、残高/クォータクエリをサポートします。コンソールログイン状態を読み取るための「自動識別」をサポートし、「プラグインホストセッション（複数アカウント、推奨）」モードもサポートしています。これにより、各アカウントの認証を個別に更新でき、同一サイトでの複数アカウント体験を向上させます。
  - 表示設定：「今日の収支を表示」スイッチを追加しました（デフォルトでオン）。これにより、統計情報（今日の消費/収入など）を非表示にし、取得を停止できるため、更新時のログ取得リクエストを削減します。
- **注意:**
  - Sub2API は、サイトサインイン/今日の使用量/収入などの関連機能は現時点ではサポートしていません。基本的な残高/クォータクエリのみを提供します。後続でサイト能力に応じて関連機能を段階的に改善していきます。
  - 「プラグインホストセッション（複数アカウント）」は `refresh_token` をアカウントのプライベート認証情報として保存し、エクスポート/WebDAV バックアップにも含まれます。バックアップファイルと WebDAV 認証情報は適切に保管してください。

**場所のヒント:**
- Sub2API 追加/モード説明：「設定 → アカウント管理」でアカウントを追加/編集し、サイトタイプに Sub2API を選択します。詳細な手順は [FAQ](./faq.md)（“Sub2API”で検索）を参照してください。
- 今日の収支オン/オフ：「設定 → 基本設定 → 表示設定」。

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」を追加しました。サイトコンソール/ドキュメント/管理ページなどのショートカットリンクを保存でき、完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元および WebDAV 自動同期に含まれます。
- **問題修正:**
  - アカウントリフレッシュ：重複する「今日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部サイトではリフレッシュインターフェースで `today_income` が返されます）。
  - 自動リフレッシュ：リフレッシュ間隔の最小値を 60 秒、最小リフレッシュ間隔保護を 30 秒に設定しました。更新後、古い設定は自動的に合法な範囲に補正され、関連するヒント文言とドキュメントが改善されました。

::: warning 重要：自動リフレッシュ設定は強制的に調整されます
フィードバックによると、**短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける**ため、

v3.15.0 では**自動リフレッシュ設定を強制的に変更**しました：
- 自動リフレッシュおよびプラグインを開いたときの自動リフレッシュ機能をオフにしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
- `リフレッシュ間隔` は最低 60 秒、`最小リフレッシュ間隔保護` は最低 30 秒です。アップグレード前に設定した値が上記閾値を下回っていた場合、アップグレード後に自動的に最低値に引き上げられます。以前設定した値が新しい合法範囲内であれば、影響はありません。
:::

**場所のヒント:**
- ブックマーク管理：「設定 → ブックマーク管理」；ポップアップ上部で「アカウント / ブックマーク」を切り替えます。
- 自動リフレッシュ：「設定 → 基本設定 → 自動リフレッシュ」。
