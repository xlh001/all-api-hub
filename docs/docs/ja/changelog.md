# 更新ログ

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験最適化 / 問題修正）を記録します。完全な履歴バージョンと詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開き、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにする方法**：「設定 → 一般 → 更新ログ」で、「更新後に更新ログを自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.28.0
- **新機能:**
  - API認証情報：`CLI互換性検証` 操作を追加しました。検証時にモデルIDの自動取得または手動入力をサポートし、一時的な`APIタイプ`の上書き使用中であることを明確に通知することで、使い捨てテスト結果を保存済み設定と誤認するのを防ぎます。
  - API認証情報 / モデルリスト：`API認証情報` から対応する `モデルリスト` データソースへワンクリックでジャンプできるようになりました。`モデルリスト` もAPI認証情報をデータソースとして直接モデルディレクトリと検証結果を表示でき、サイトアカウントを事前に作成する必要はありません。
  - 鍵管理：`鍵管理` にホストサイトのチャネル状態、マッチングシグナル、ジャンプ可能なエントリが表示されるようになりました。鍵を `API認証情報` に保存する際には、より明確な名称が生成され、後からの検索と再利用が容易になります。
  - New APIホストサイト：ホストサイト設定にログイン補助情報（ユーザー名、パスワード、オプションのTOTP）とセッション検証を追加しました。状態の検証や実際のチャネル鍵の読み取りが必要な場合、拡張機能内で直接検証を完了できます。
  - ホストサイトマッチング：チャネル識別を `URL`、鍵、モデルなどのシグナルを総合してソートするように変更しました。バックエンドがマスクされたトークンのみを返す場合でも、チャネル状態の判断、コピー、統合操作を継続して完了できます。
  - 初回使用：ウェルカム / 権限ガイドポップアップに言語セレクターを追加しました。初回起動時にインターフェース言語を切り替え、以降の使用設定を記憶します。
- **体験最適化:**
  - Veloera：現在 `Base URL` ベースのチャネル特定と状態検出をサポートしていないシナリオでは、関連エントリが自動的に非表示または無効化され、説明が表示されます。これにより、誤ってクリックして結果が得られない場合の困惑を減らします。
- **問題修正:**
  - 言語：起動時にブラウザの検出言語に安定して従わない問題を修正し、インターフェースの文言と日付・時刻のローカライズを同期・校正しました。
  - 権限ガイド：権限説明ポップアップのボタンレイアウトを最適化し、小ウィンドウやボタンの折り返し時により整然とし、クリックしやすくしました。

**場所のヒント：**
- API認証情報：「設定 → API認証情報」で、`CLI互換性検証`、`モデル管理で開く` などの操作を使用できます。
- モデルリストデータソース：「設定 → モデルリスト」の上部データソース選択エリアで、`API認証情報` に切り替えることができます。
- ホストサイトチャネル状態：「設定 → 鍵管理」で、各鍵のホストサイト状態とマッチングヒントを確認できます。
- New APIホストサイトログイン補助：「設定 → 自社サイト管理」の `New API統合設定` エリア。
- 初回言語選択：拡張機能初回起動時に表示されるウェルカム / 権限ガイドポップアップ内。

## 3.27.0
- **新機能:**
  - アカウント管理：アカウントリストに有効状態でのフィルタリングを追加し、`有効` / `無効` アカウントを素早く切り替えて表示できるようになりました。無効なアカウントの一括整理がより便利になります。
  - フィードバックとサポート：拡張機能ポップアップのタイトルバーに `フィードバック` のクイックエントリを追加しました。「バージョン情報」ページにも `フィードバックとサポート` セクションを追加し、GitHub の問題フィードバック、機能提案、ディスカッションエリアを直接開くことができます。
- **体験最適化:**
  - アカウント表示：複数のアカウントのサイト名が同じ場合、自動的にユーザー名を追加して `サイト名 · ユーザー名` と表示し、リスト、検索、セレクター、統計ビューで区別しやすくしました。
- **問題修正:**
  - サイドバー：サイドバーのサポート検出をさらに最適化しました。ブラウザまたはモバイル環境がサイドバーをサポートしていない場合、無効なエントリは自動的に非表示になるか、設定ページにフォールバックし、クリックしても応答がない状況を減らします。

**場所のヒント：**
- アカウント状態フィルタリング：「設定 → アカウント管理」リスト上部のフィルタリングエリア。
- フィードバックエントリ：拡張機能ポップアップタイトルバーの `フィードバック` ボタン、および「設定 → バージョン情報」の `フィードバックとサポート` セクション。

## 3.26.0
- **新機能:**
  - アカウント管理：`対応チャネルの特定` クイック操作を追加しました。ホストサイトアカウントから対応する「チャネル管理」リストへフィルタリング付きでワンクリックジャンプできます。また、「重複アカウント追加前のリマインダー」を有効にすることで、誤って重複アカウントを追加するのを減らせます。
  - 重複アカウントクリーンアップ：`重複アカウントクリーンアップ` ツールを追加しました。URLソースサイト + ユーザーIDでスキャンし、重複項目を削除できます。重複アカウントの一括整理がより便利になります。
  - アカウント管理：無効化されたアカウントの操作メニューに直接削除エントリを追加し、無効なアカウントの整理手順を省きました。
  - API認証情報：`API認証情報` ページは、設定ナビゲーションと拡張機能ポップアップから直接アクセスできます。設定のエクスポート時にもトークンメモを保持し、複数のツール間での移行を容易にします。
  - WebDAV：同期データ選択を追加しました。必要に応じて `アカウント`、`ブックマーク`、`API認証情報` などの共有データのみを同期でき、複数デバイス間での不要な上書きを減らします。
  - Sub2API：`Sub2API` アカウントの鍵管理をサポートし、鍵の表示、作成、編集、削除を直接行えます。
  - CLIProxyAPI：インポート時にProviderタイプ選択を追加し、一般的なendpointアドレスを自動的に正規化することで、手動でのURL変更を減らします。
- **体験最適化:**
  - 交換アシスタント：交換成功後、アカウント残高が自動的に更新され、手動リフレッシュで結果を確認する手順を減らします。
- **問題修正:**
  - 自動サインイン：固定時間サインインに、より安定した補完実行メカニズムを追加し、拡張機能の更新などによる実行ウィンドウの欠落によるサインイン漏れを減らします。
  - 自動認識：アカウント再認識後にカスタムサインイン設定が失われる問題を修正し、設定の意図しない消失を防ぎます。
  - 自動サインイン：Turnstileアシスタントまたは手動サインインプロンプトが一部のアカウントで誤って `外部サインインアドレス` を使用する問題を修正しました。代わりに常にサイトのデフォルトサインインページを開くようにし、誤ったページへのジャンプやサインイン完了不可を減らします。
  - ホストサイト：ホストサイトへのデータインポートまたは同期時に、ターゲットサイトのデフォルトグループを優先的に使用し、グループの不一致による異常を減らします。

::: warning 注意
- WebDAV の `同期データ選択` とアカウント自動リフレッシュなどのデバイスローカル設定は、WebDAV を介してデバイス間で上書きされなくなりました。
:::

**場所のヒント：**
- 重複アカウントリマインダー：「設定 → 基本設定 → アカウント管理」の `重複アカウント追加前のリマインダー`。
- 重複アカウントクリーンアップ：「設定 → アカウント管理」のページツールバー。
- チャネルの特定：「設定 → アカウント管理」の個々のアカウントの操作メニュー。
- API認証情報：「設定 → API認証情報」、拡張機能ポップアップでも `API認証情報` ビューに切り替え可能です。
- WebDAV同期データ選択：「設定 → インポート/エクスポート」の `WebDAV設定`。

## 3.25.0
- **新機能:**
  - 自動サインイン：Cloudflare Turnstile（ガード/人間認証）シナリオをサポートします。サイトがTurnstile認証を要求する場合、一時ページで認証を完了してからサインインを続行しようとします。必要に応じて、手動で開けるサインインリンクとヒントを提供します。
  - CC Switch：`Codex` へのエクスポート時に、基本アドレスのデフォルト値に `/v1` を自動補完します（インターフェースアドレスを手動で変更していない場合）。これにより、直接インポート後にインターフェースが利用できなくなる問題を減らします。
  - モデルリダイレクト：オプションのスイッチ `同期後に無効なリダイレクトターゲットをクリーンアップ` を追加しました。モデル同期リフレッシュ後に、`model_mapping` で存在しないモデルを指しているマッピングを自動的に削除します（危険な操作、デフォルトはオフ）。
- **体験最適化:**
  - 一時ウィンドウ：チャレンジ/ログインページとしてより正確に識別し、誤判定や不要な干渉を減らします。
- **問題修正:**
  - Cookie認証：現在の実際の動作と能力に合致するように文言説明を修正し、誤解を減らします。
  - サイドバー：小ウィンドウでサイドバーの最下部メニュー項目が見えなくなる問題を修正しました。

**場所のヒント：**
- Turnstile認証：「設定 → 自動サインイン」の実行結果で新しいヒントを確認できます。
- CC Switch エクスポート：「設定 → 鍵管理」で鍵を選択し、`Codex` へのエクスポートをクリックし、ターゲットアプリケーションを `Codex` に選択します。
- モデルリダイレクトクリーンアップ：「設定 → 基本設定 → モデルリダイレクト」で `同期後に無効なリダイレクトターゲットをクリーンアップ` を有効にします。

## 3.24.0

**場所のヒント：**
- 更新ログスイッチ：「設定 → 一般 → 更新ログ」の `更新後に更新内容を自動表示`。
- 更新ログポップアップ：プラグイン更新後、初めて「拡張機能ポップアップ / 設定ページ / サイドバー」を開いた際に自動的にポップアップします（各バージョンにつき1回）。
- LDOH クイックエントリ：「アカウント管理」アカウントリストのサイト名右側（LDOHアイコン、`LDOHで表示` と表示）；アカウント追加ダイアログでも `LDOHサイトリストを開く` をクリックできます。

## 3.22.0
- **新機能:**
  - モデルリスト：`モデル対応鍵` ツール（鍵アイコン）を追加しました。現在のモデルに利用可能な鍵が存在するかどうかを確認できます。利用可能な鍵がない場合、利用可能なモデルグループごとにデフォルト鍵をワンクリックで作成するか、カスタム作成プロセスに入り、鍵のワンクリックコピーもサポートします。
  - スナップショット共有：`概要スナップショット` / `アカウントスナップショット` のワンクリック共有をサポートします。優先的にクリップボードに画像をコピーし、サポートしない場合はPNGを自動ダウンロードします。スナップショットには共有可能な情報のみが含まれ（`API Key` などの機密フィールドは含まれません）、タイトルの文言をワンクリックでコピーできます。
- **体験最適化:**
  - アカウント無効化：`残高履歴` / `使用量分析` / `使用量同期` などのリフレッシュと定期タスクで、無効化されたアカウントは自動的にスキップされ、無効なリクエストとエラーを減らします。
- **問題修正:**
  - ボタン：ボタンが「ロード中」で左アイコンも同時に表示される場合にスピナーが見えない問題を修正しました。

**場所のヒント：**
- モデル対応鍵：「設定 → モデルリスト」で、モデル名の右側にある鍵アイコン（`モデル対応鍵`）をクリックします。
- 概要スナップショット共有：拡張機能ポップアップの概要ページタイトルバー右側のボタン（`概要スナップショット共有`）。
- アカウントスナップショット共有：「設定 → アカウント管理」の個々のアカウントの操作メニュー（`アカウントスナップショット共有`）。

## 3.21.0
- **新機能:**
  - API認証情報：「API認証情報」ページを追加しました。「アカウントがなく、`Base URL` + `API Key` のみ」という使用シナリオに適しています。タグ/メモの一元管理をサポートし、直接可用性検証とクイックエクスポート（例：Cherry Studio / CC Switch / Kilo Code / CLIProxyAPI / Claude Code Router）が可能で、コピー＆ペーストの手間を減らします。

  - 右クリックメニュー：「交換アシスタント」と「AI APIテスト」のエントリが個別にオン/オフできるようになりました。切り替え後すぐにリフレッシュして有効になります。

**場所のヒント：**
- API認証情報：「設定 → API認証情報」。
- 残高履歴：「設定 → 残高履歴」。
- Done Hub 設定：「設定 → 基本設定 → 自社サイト管理」で `Done Hub` を選択し、「Done Hub 統合設定」を入力します。
- 右クリックメニューエントリスイッチ：「設定 → 基本設定 → サインインと交換 / AI APIテスト」の各「ブラウザの右クリックメニューに表示」。
- 鍵コピーポップアップ：「アカウント管理」などのページで「鍵のコピー」をクリックして開きます。

## 3.19.0
- **新機能:**
  - 自社サイト管理：`Octopus` ホストサイトをサポートしました。Octopusバックエンドに接続し、「チャネル管理」でアカウントAPI鍵をチャネルとしてインポートできます。利用可能なモデルリストの取得もサポートします。
  - 鍵管理：「追加後にデフォルト鍵を自動作成」（デフォルトオン）を追加しました。「少なくとも1つの鍵を確保」機能により、鍵が不足しているアカウントにデフォルト鍵を自動補完できます。
  - AI APIテスト：インターフェース検証の「モデルリスト検出」がOpenAI/OpenAI互換、Anthropic、Google/Geminiなどのインターフェースタイプをサポートし、利用可能なモデルIDの提案を提供して、手動でのモデル推測を減らします。
- **体験最適化:**
  - アカウント管理：サイト/アカウント識別ロジックを強化し、同一サイトでの複数アカウントシナリオの安定性を改善しました。
  - 使用量/ログ取得：ログ系インターフェースにレート制限保護を追加し、頻繁なリフレッシュによるエラーやサイトのレート制限トリガーを減らします。
  - チャネル管理：チャネル作成時に、より正確な重複検出を行い、確認プロンプトを表示することで、誤った重複ルーティング作成を防ぎます。

::: warning 重要：自動リフレッシュ設定は強制的に調整されます
フィードバックによると、**短すぎる自動リフレッシュ間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける可能性があります**。

v3.15.0 では**自動リフレッシュ設定を強制的に変更しました**：
- 自動リフレッシュとプラグインを開いた時のリフレッシュ機能をオフにしました。引き続き有効にしたい場合は、手動で再度有効にしてください。
- `リフレッシュ間隔` の最小値を60秒、`最小リフレッシュ間隔保護` の最小値を30秒としました。アップグレード前に上記閾値を下回る設定をしていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい合法範囲内であれば、影響はありません。
:::
