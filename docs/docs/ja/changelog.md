# 更新履歴

本ページでは、一般ユーザー向けの主要な更新点（機能変更 / 体験最適化 / 問題修正）を記録しています。完全な履歴バージョンとより詳細な技術的変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) をご覧ください。

::: tip 新規ユーザーの方へ
- **現在のバージョンの確認方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。または設定ページでも確認できます。
- **このページを自動で開かないようにする方法**：「設定 → 一般 → 更新履歴」で、「更新後に更新履歴を自動で開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.18.0
- **新機能:**
  - 残高履歴：グラフに「通貨単位」切り替え（`USD` / `CNY`）を追加し、座標軸/ヒントに通貨記号を表示するようにしました。`CNY` を選択した場合、アカウントの「チャージ金額比率」に基づいて換算され、金額ベースで傾向の確認や照合が容易になります。
- **体験最適化:**
  - タグフィルタリング：タグ/アカウントのオプションが多すぎる場合、デフォルトで「展開表示」に変更し、閲覧と選択をより直感的にしました。
  - タブページ：「設定」グループタグと「モデルリスト」ベンダータグに左右スクロールボタンを追加し、狭いウィンドウでの切り替えを容易にしました。
- **問題修正:**
  - アカウント管理：サイトタイプ「自動識別」の精度が向上し、最近のバージョンで頻繁に発生していた未知のサイトタイプの問題を修正しました。

**場所のヒント：**
- 残高履歴の通貨単位：「設定 → 残高履歴」ページのフィルタリングエリア「通貨単位」。
- アカウントの換算レート（チャージ金額比率）：「設定 → アカウント管理」の新規/編集アカウントフォーム「チャージ金額比率」。

## 3.17.0

- **体験最適化:**
  - チャネル管理：「チャネル管理」ツールバーの、小画面/狭いウィンドウでのレスポンシブレイアウトと使いやすさを最適化しました。

  - インポート/エクスポート：一部の画面サイズでのエクスポートエリアのレスポンシブ表示の問題を修正しました。
  - ポップアップ：ポップアップのスクロールバーの位置が正しくないレイアウトの異常を修正しました。

**場所のヒント：**
- 残高履歴のスイッチ/保持日数/日次終了時の取得：「設定 → 基本設定 → 残高履歴」。

## 3.16.0
- **新機能:**
  - Sub2API（JWTサイト）：Sub2APIサイトタイプを追加し、残高/クォータの照会をサポートしました。「自動識別」によるコンソールログイン状態の読み取りをサポートしました。また、「プラグインホスト型セッション（マルチアカウント、推奨）」モードをサポートし、アカウントごとに認証を個別に更新でき、同一サイトでのマルチアカウント体験を改善しました。
  - 表示設定：「本日の収支を表示」スイッチ（デフォルトでオン）を追加しました。これにより、「本日の消費/収入」などの統計情報を非表示にし、リフレッシュ時のログ取得リクエストを減らすことができます。
- **注意:**
  - Sub2API は、現時点ではサイトのチェックイン/本日の利用量/収入などの関連機能をサポートしておらず、基本的な残高/クォータ照会のみを提供します。今後はサイトの機能に応じて関連機能を順次充実させていく予定です。
  - 「プラグインホスト型セッション（マルチアカウント）」では、`refresh_token` がアカウントのプライベート認証情報として保存され、エクスポート/WebDAVバックアップに含まれます。バックアップファイルとWebDAV認証情報は適切に保管してください。

**場所のヒント：**
- Sub2APIの追加/モードの説明：「設定 → アカウント管理」でアカウントを新規追加/編集し、サイトタイプで Sub2API を選択します。詳細な手順は [FAQ](./faq.md)（「Sub2API」で検索）を参照してください。
- 本日の収支スイッチ：「設定 → 基本設定 → 表示設定」。

## 3.15.0
- **新機能:**
  - ブックマーク管理：「ブックマーク管理」を追加しました。サイトのコンソール/ドキュメント/管理ページなどのショートカットリンクを保存でき、完全なアカウントを作成する必要はありません。追加/編集/削除、ピン留め、タグ、検索フィルタリング、ドラッグ＆ドロップによる並べ替えをサポートします。ポップアップに「アカウント / ブックマーク」切り替えを追加しました。ブックマークデータはバックアップ/復元およびWebDAV自動同期に含まれます。
- **問題修正:**
  - アカウントリフレッシュ：重複する「本日の収入」取得リクエストを削除し、不要なネットワーク呼び出しを削減しました（一部のサイトではリフレッシュインターフェースで既に `today_income` が返されています）。
  - 自動リフレッシュ：リフレッシュ間隔は最低60秒、最小リフレッシュ間隔保護は最低30秒としました。更新後、古い設定は自動的に有効な範囲に修正され、関連するヒントの文言とドキュメントを充実させました。

::: warning 重要：自動リフレッシュ設定が強制的に調整されます
**短すぎる自動リフレッシュ間隔は、サイトのレート制限をトリガーしやすく、サイトに過負荷を与える可能性がある**というフィードバックに基づき、

v3.15.0 では**自動リフレッシュ設定に強制的な変更**を加えました：
- 自動リフレッシュ機能とプラグインオープン時のリフレッシュ機能をオフにしました。引き続き有効にする必要がある場合は、手動で再度オンにする必要があります。
- `リフレッシュ間隔` は最低60秒、`最小リフレッシュ間隔保護` は最低30秒です。アップグレード前にこれらのしきい値を下回る設定をしていた場合、アップグレード後に自動的に最低値に引き上げられます。以前の設定値が新しい有効範囲内にある場合は影響ありません。
:::

**場所のヒント：**
- ブックマーク管理：「設定 → ブックマーク管理」にあります。ポップアップ上部で「アカウント / ブックマーク」を切り替えられます。
- 自動リフレッシュ：「設定 → 基本設定 → 自動リフレッシュ」。

## 3.14.0
- **新機能:**
  - Web AI API機能の可用性テスト（ベータ）：右クリックメニューに「AI APIの機能可用性をクイックテスト」を追加しました。現在のウェブページで直接テストパネルを開くことができます。`Base URL` と `API Key` の入力/貼り付けをサポートし、OpenAI互換 / OpenAI / Anthropic / Google などのインターフェースの基本機能検出を行います（OpenAI互換ではモデルリストの一括取得も可能です）。
  - （オプション）自動検出：「設定 → AI APIテスト」で有効にし、URLホワイトリストを設定できます。ホワイトリスト内のページで利用可能な `Base URL` と `API Key` が検出された場合、まず確認プロンプトが表示され、確認後にのみテストパネルが開かれます（デフォルトではオフで、Keyは保存されません）。
  - 自動チェックイン：実行結果リストに、より多くのトラブルシューティングのヒントを追加しました。これには、「一時的な回避タブが手動で閉じられた」「Access Tokenが無効」などの一般的な異常に対する処理の提案とドキュメントへの入り口が含まれます。
- **問題修正:**
  - WebDAV：自動同期をタイマーからブラウザのAlarms APIに移行し、バックグラウンドの休止/省電力ポリシーによる同期漏れの可能性を低減しました。

**場所のヒント：**
- AI APIテストパネル：任意のウェブページで右クリックメニューから「AI APIの機能可用性をクイックテスト」を選択します。自動検出関連の設定は「設定 → AI APIテスト」にあります。
- 自動チェックインのヒント：「設定 → 自動チェックイン」の実行結果リストで確認できます。

## 3.8.0
- **新機能:**
  - 利用状況分析：「利用状況分析」ページを追加しました。複数のサイト、複数のアカウントの利用傾向をグラフ化し、「どこで多く使っているか / 多く費用がかかっているか / 遅くなっているか」を一目で把握できるようにし、コスト管理、照合、問題のトラブルシューティングを容易にします。
