# 更新ログ

このページでは、一般ユーザー向けの主な更新点（機能変更 / 体験改善 / 問題修正）を記録しています。完全な履歴バージョンとより詳細な技術変更については、[GitHub Releases](https://github.com/qixing-jk/all-api-hub/releases) を参照してください。

::: tip 新規ユーザー向けガイド
- **現在のバージョンを確認する方法**：拡張機能のポップアップを開くと、タイトルバーにバージョン番号が表示されます。設定ページでも確認できます。
- **このページが自動的に開かないようにする方法**：「設定 → 一般 → 更新ログ」で、「更新後に更新ログを自動的に開く」を制御できます。
- **異常のトラブルシューティング**：「設定 → 一般 → ログ」でコンソールログを有効にし、再現手順を添えて [Issues](https://github.com/qixing-jk/all-api-hub/issues) に報告してください。
:::

## 3.34.0
- **新機能:**
  - モデルリスト：価格比較機能を追加しました。課金方式によるフィルタリング、価格による並べ替えが可能です。「全アカウント」ビューでは「同一モデルの最安優先」表示もサポートし、複数アカウントでの低価格モデル探しがより直接的になりました。
  - API 認証情報：設定カードに利用状況テレメトリ概要と手動更新を追加しました。ヘルス状態、残高、当日利用量、リクエスト数、利用可能モデルなどの情報を確認できます。また、サイトタイプによる自動識別、またはカスタムの読み取り専用クエリエンドポイントの使用もサポートします。
  - モデルリスト：「全アカウント」ビューにアカウント、アカウントグループなどのディメンションによるフィルタリングを追加しました。ソースが多い場合に範囲を絞り込みやすくなりました。
  - モデル検証：現在のフィルタリング結果に対して一括検証を実行できるようになりました。「全アカウント」ビューでも直接使用でき、モデルの可用性を一括でトラブルシューティングするのがより手軽になりました。
  - アカウント管理：一括選択、一括無効化、一括削除を追加し、大量のアカウント整理がより便利になりました。
  - アカウント管理：`作成時間` によるソートをサポートし、リストに作成時間を直接表示します。
  - 自動サインイン：失敗した結果に対して、直接 `アカウント無効化` または `アカウント削除` を実行できるようになりました。異常なアカウントをトラブルシューティングした後、結果ページでその場で処理でき、アカウント管理に戻る必要がなくなりました。
  - ホストサイト：現在のサイトの設定が完了していなくても、「チャネル管理」「モデル同期」などの入口を保持し、ページ内で不足している設定とジャンプ先の設定入口を直接通知します。
  - アカウント管理：アカウント追加時に、現在のサイト URL を自動入力するオプションを選択でき、操作手順を減らしました。
- **体験改善:**
  - アカウント管理：アカウント管理ページ上部にグローバル更新ボタンを追加し、ワンクリックで全てのアカウントを更新できます。
  - アカウント管理：ドラッグ＆ドロップによる並べ替えとフィルタリング後のアニメーション体験もよりスムーズになりました。
  - モデルリスト：「全アカウント」ビューで、「読み込み中」「読み込み失敗」「形式非互換」などの状態をより明確に区別します。展開項目は更新後も可能な限り保持され、繰り返し検索する手間を減らします。
  - キー管理：現在のアカウントまたは「全アカウント」のフィルタリングビューでキーを追加する際に、対応するアカウントが自動的に選択され、重複選択の手間を減らしました。
  - 更新体験：複数の設定ページと履歴記録ページで、データを更新する際に現在の内容を可能な限り保持し、頻繁にクリアしてから再読み込みするのをやめました。閲覧中に中断されにくくなりました。
- **パフォーマンス最適化:**
  - アカウント管理：大規模リストの読み込みパフォーマンスを最適化しました。アカウントが多い場合、より高速な読み込み速度を実現します。
- **問題修正:**
  - WebDAV：一部のサービスが `206 Partial Content` を返すバックアップダウンロード応答に対応し、ダウンロード可能なのに誤って失敗と判断される状況を減らしました。
  - WONG 公益サイト：マスキングされた `token` の実際のキー読み取り方法を修正し、関連アカウントまたはキー情報の正常な解析を回復しました。
  - 一時ウィンドウリマインダーの有効化が必要：アカウント更新時に一時ウィンドウのフォールバックが必要と確認されたが、現在の関連設定がオフまたは権限が不足している場合にのみ調整を促し、無関係なシナリオでの通知を減らしました。

**場所のヒント:**
- モデル価格比較、アカウントフィルタリング、一括検証：「設定 → モデルリスト」にあります。
- API 認証情報テレメトリ概要と設定：「設定 → API 認証情報」のカードと新規/編集ダイアログボックスにあります。
- アカウント追加の自動入力、一括管理、作成時間ソート：「設定 → アカウント管理」にあります。
- 自動サインイン失敗アカウント処理：「設定 → 自動サインイン」の結果リストにあります。
- キー管理での新規追加時のアカウント自動選択：「設定 → キー管理」にあります。
- ホストサイトの設定不足通知：「設定 → 自社サイト管理」の関連ホストサイトページにあります。
- WebDAV バックアップ同期：「設定 → インポート/エクスポート」の `WebDAV 設定` 関連プロセスにあります。

## 3.33.0
- **新機能:**
  - アカウント追加：ホストサイトの設定が不完全な場合、`保存して自動設定` の前に設定の補完を促し、関連する設定ページに直接ジャンプできるようになりました。これにより、自動設定をクリックしてから設定不足に気づく状況を減らします。
  - ホストサイト検証：`New API` の検証ポップアップを、より認証コード入力に適した `OTP` スタイルに変更しました。ワンタイム認証コードの入力がよりスムーズになります。
  - API 検証履歴：検証履歴の状態表示がより明確になりました。`API 認証情報` やモデルリストなどの場所で前回の検証結果を確認する際に、現在の状態を判断しやすくなりました。
  - ホストサイト設定：管理者ユーザー ID には数字のみが許可されるようになりました。不正な値は設定ページで直接ブロックされ、通知が表示されます。これにより、値の入力ミスによる後続の検証や読み取り異常を減らします。
  - モデルリスト：データソースがまだ選択されていない場合、より明確な空の状態説明と次の操作を提供し、ページに入ってから何をすべきか分からない状況を減らします。
  - キー管理：アカウントにキーがない場合やキーの読み込みに失敗した場合、より明確な空の状態通知と、再試行または関連ページへの移動操作を提供します。
  - モデル対応キーポップアップ：対応するアカウントの `キー管理` に直接ジャンプできるようになりました。アカウントのアイコンを使用してソースを素早くフィルタリングでき、特定アカウントのモデルとキーのトラブルシューティングがより便利になりました。
  - 更新通知：手動でパッケージをインストールしたユーザーは、現在のバージョンと最新の安定版との違いを確認できるようになりました。ダウンロードページや更新説明を直接開くことができ、長期間古いバージョンに留まっていることに気づかない状況を減らします。
- **体験改善:**
  - ページ内ジャンプ：設定ページ、アカウントページ、またはホストサイトページに現在のフローからジャンプする必要がある複数の入口について、ナビゲーション履歴をできるだけ保持するようにし、前のステップへの戻りがより自然になりました。
  - フロー離脱時の通知：関連ボタンの文言で、「現在のフローを離れる」と「次のステップに進む」をより明確に区別し、誤ってクリックしてページを離れてしまった状況を減らしました。
  - 警告通知：一部の「部分的に成功 / 後続処理が必要」なシナリオを、統一された警告通知スタイルに変更し、直接実行可能な後続操作を付加しました。
  - アカウントデータ読み込み：手動でアカウントデータを取得するのに約 `20` 秒以上かかった場合、長時間待機を停止し、保存可能な結果をできるだけ保持することで、待機中に固まってしまう状況を減らしました。
- **問題修正:**
  - WebDAV 同期：自動同期前に、ローカルで削除されたアカウントの変更をできる限りアップロードし、削除されたアカウントが他のデバイスや後続の同期で再度取り込まれる状況を減らしました。
  - アカウント管理：現在の環境で実際に制限されている場合にのみ、一時ウィンドウを使用したフローの補助を通知するようにし、誤報を減らしました。
  - モデルリスト：元の表示されていたアカウント名アイコンを復元しました。アカウントソースのモデルを確認する際に、現在の項目がどのアカウントからのものか区別しやすくなりました。
  - DoneHub：一部サイトの日次使用量クエリ方法に対応し、読み取り異常や結果の不安定さを減らしました。
- **パフォーマンス最適化:**
  - データ統計：日次使用量の高速読み取りパスでの追加リクエストを削減し、一部サイトの更新待機時間を短縮しました。

**場所のヒント:**
- ホストサイト設定補完：「設定 → アカウント管理」の新規作成または編集アカウントダイアログボックスにあります。
- `OTP` 検証とホストサイト設定：「設定 → 自社サイト管理」の `New API 統合設定` および関連する検証ポップアップにあります。
- ホストサイト管理者ユーザー ID 検証：「設定 → 自社サイト管理」の関連ホストサイト設定にあります。
- モデルリストの空の状態とソースフィルタリング：「設定 → モデルリスト」にあります。
- キー管理の空の状態とクイックジャンプ：「設定 → キー管理」およびモデル対応キーポップアップにあります。
- バージョン更新通知：拡張機能ポップアップまたは設定ページ上部のバージョン番号、および「設定 → バージョン情報」にあります。
- WebDAV 自動同期：「設定 → インポート/エクスポート」の `WebDAV 設定` および `WebDAV 自動同期` にあります。

## 3.32.0
- **新機能:**
  - 自動サインイン：結果テーブルの各行から、対応するサイトを直接開けるようになりました。現在のサイトタイプが自動サインイン自体に対応していない場合も、手動で対応するか、先にサイトのルールを確認するか判断しやすいよう、より明確な案内を表示します。
  - New API ホストサイト検証：検証時に `Base URL`、ユーザー名、またはパスワードが不足していると判定された場合、そのまま検証ダイアログ内で入力し、`保存して再試行` で続行できるようになりました。設定ページに戻る必要はありません。
- **体験改善:**
  - アカウント一覧のソート：デフォルトのソート優先度を見直し、一覧で選択したソート項目がこれまでより優先して反映されるようになりました。たとえば `残高`、`消費`、`収入` で並べ替えたときは、従来の手動順より先にその条件が効きます。既存の設定は自動移行されるため、再設定は不要です。
- **問題修正:**
  - アカウント管理：Cookie インポート時に権限不足が発生して `権限設定` へ遷移しても、アカウントダイアログが途中で閉じないようになりました。権限対応後、そのまま入力を続けられます。
  - 自動認識：現在のタブが拡張機能のインストールや更新より前から開かれていた場合、ページの再読み込みが必要なことを明確に案内し、ワンクリックの再読み込み導線も表示するようになりました。ランダムに失敗しているように見える状況を減らします。
  - 一時ページ / ウィンドウフロー：一時ページを使う一部フローでコンテキストが早期に解放される問題を修正し、途中失敗、ページが閉じられるケース、`No tab with id` のようなエラーを減らしました。

**場所のヒント:**
- 自動サインイン結果のクイック操作：「設定 → 自動サインイン」の結果テーブル各行にあります。
- New API の設定補完：「設定 → 自社サイト管理」および関連する `New API` 検証ダイアログにあります。
- Cookie インポート権限案内 / 自動認識の再読み込み案内：「設定 → アカウント管理」の新規作成または編集ダイアログにあります。
- ソート優先度設定：「設定 → アカウント管理 → ソート優先度設定」にあります。

## 3.31.0
- **新機能:**
  - Safari サポート：Safari 用のインストールドキュメントと配布パッケージを追加し、macOS 上での Xcode を使用したインストールと利用を容易にしました。
  - アカウント管理：独立した高度なフィルタリングバーを追加し、`サイトタイプ`、`サインイン状態`、`更新状態`、`アカウント状態` で組み合わせフィルタリングできるようになりました。各オプションは数量も同期表示し、フィルタリングがより直感的になりました。
  - モデルリスト：単一アカウントのモデルリストの読み込みに失敗した場合、そのアカウントの `API Key` を使用して読み込みを継続できるようになり、インターフェースの異常でモデルが全く見えなくなる状況を減らしました。
  - CLIProxyAPI：`CLIProxyAPI 設定` に `接続テスト` を追加しました。アドレスを保存したり、キーを管理したりした後、自動的にチェックできます。手動で再テストすることも可能です。
- **体験改善:**
  - ホストサイトチャネルインポート：インポート前に、現在のチャネルが重複する可能性があることをより明確に通知し、誤ってインポートしてから再度確認する手間を減らしました。
  - ホストサイトチャネルインポート：さらに確認が必要な状況に遭遇した場合、ダイアログボックスで検証を完了してから重複かどうかを判断できるようになり、操作が直接ブロックされることはなくなりました。
  - チャネルへのエクスポート：モデルを取得できなかった場合でも、チャネルダイアログボックスは正常に開き、手動入力が必要であることを明確に通知します。自動読み込みの失敗によって操作が直接中断されることはありません。
  - アカウント Cookie インポート：現在、「現在のサイトに読み取り可能な Cookie がない」「Cookie 権限が不足している」「読み取りに失敗した」などの異なる理由を区別するようになりました。権限不足の場合は、アカウントダイアログボックスに通知が表示され、権限設定ページを直接開くことができます。
  - チャネル管理：チャネルリストはデフォルトで最新の記録が優先表示されるようになり、新規または最近追加されたチャネルを見つけやすくなりました。
  - 自動サインイン：`スキップ済み` 状態のアカウントは後方に配置され、よく使うアカウントや処理待ちのアカウントを見つけやすくなりました。
  - 操作通知：一部の操作でサーバーから通知文案が返ってこない場合でも、ローカルでの成功通知を追加し、クリック後にフィードバックがない状況を回避しました。
  - 多言語対応：一部欠落していた翻訳を補完し、複数のインターフェース文言やエラーメッセージをさらに改善しました。
- **問題修正:**
  - モバイルブラウザ互換性：ブラウザが一時ウィンドウの作成を許可しない場合、自動的にタブでプロセスを継続するようになり、クリックしても反応がない状況を減らしました。
  - Sub2API：キー作成時にグループを明確に選択する必要があるように変更し、無効なグループを含む API キーが作成されるのを防ぎました。
  - ページレイアウト：設定ページのチャネルテーブル操作列とサイドバーのオーバーレイの重なり順の問題を修正しました。
  - 更新ログダイアログボックス：小画面で下部のボタンが押し出されたり、改行が乱雑になったりする問題を修正しました。

**場所のヒント:**
- アカウント高度フィルタリング：「設定 → アカウント管理」リスト上部のフィルタリングエリアにあります。
- アカウントキーによるモデルフォールバック読み込み：「設定 → モデルリスト」で単一アカウントをデータソースとして選択した後、読み込みに失敗した場合、エラー通知エリアで操作を継続できます。
- `CLIProxyAPI` 接続テスト：「設定 → CLIProxyAPI 設定」にあります。
- ホストサイトチャネルインポート通知 / 重複排除検証 / モデル事前読み込み失敗通知：「設定 → キー管理」または関連する `チャネルへのインポート` / `新規チャネル作成` ダイアログボックスにあります。

## 3.30.0
- **新機能:**
  - チャネル管理：`チャネル移行` を追加しました。移行プレビューを表示してから、現在のフィルタリング結果または選択したチャネルを他のホストサイトに移行できます。

  - 多言語対応：日本語と繁体字中国語（`zh-TW`）のインターフェース言語を追加しました。
- **体験改善:**
  - 初回利用とインターフェース：
    - ウェルカム / 権限ガイドダイアログボックスで直接言語を選択できるようになりました。
    - 設定ページ上部にテーマと言語のクイック切り替えを追加しました。
  - API 検証：保存済みの `API 認証情報` とモデル検証は、直近のプローブ結果と時間を保持します。
  - リソース使用量：一部の設定ページ、拡張機能ポップアップ、ビューでオンデマンド読み込みを採用し、不要な初期化とリソース使用量を削減しました。
- **問題修正:**
  - WebDAV：Nutstore が `409 AncestorsNotFound` を返すシナリオに対応しました。この場合、「リモートバックアップが存在しない」として処理され、初回同期または空ディレクトリでの誤った失敗報告を減らしました。

**場所のヒント:**
- ホストサイトタイプ切り替え：「設定 → 自社サイト管理」にあります。
- チャネル移行：「設定 → チャネル管理」ページ上部の `チャネル移行` にあります。
- CC Switch エクスポート：「設定 → キー管理」または「設定 → API 認証情報」のエクスポート入口にあります。
- テーマ / 言語クイックコントロール：設定ページ右上にあります。初回言語選択は、初回拡張機能を開いた際のウェルカム / 権限ガイドダイアログボックスに表示されます。
- 直近の API 検証結果：関連する検証ダイアログボックス、およびサポートされている `API 認証情報` / モデル検証インターフェースで確認できます。

::: warning 重要：自動更新設定は強制的に調整されます
フィードバックによると、**短すぎる自動更新間隔はサイトのレート制限をトリガーしやすく、サイトに過大な負荷をかける可能性があります**。

v3.15.0 では**自動更新設定を強制的に変更しました**：
- 自動更新とプラグインオープン時の更新機能をオフにしました。引き続き有効にしたい場合は、手動で再度有効にする必要があります。
- `更新間隔` は最低 60 秒、`最小更新間隔保護` は最低 30 秒です。アップグレード前に上記閾値を下回る設定をしていた場合、アップグレード後に最低値に自動的に引き上げられます。以前の設定値が新しい合法範囲内にある場合は影響ありません。
:::
