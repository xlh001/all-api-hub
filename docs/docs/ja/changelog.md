# 更新履歴

このページでは、一般ユーザー向けの主な更新点（機能変更 / エクスペリエンス最適化 / 問題修正）を記録しています。完全な履歴バージョンとより詳細な技術変更については、GitHub Releases をご覧ください。

## 3.6.0
- **新機能:**
  - アカウント管理：アカウントの一括有効化/無効化をサポート。無効化後、すべてのアカウント機能はスキップされ、アカウントが失効してもデータを保持できます。
  - タグ：グローバルタグ管理を新たに追加し、関連するインターフェースとインタラクションを最適化して、アカウントの分類管理を容易にしました。
  - ポップアップ：タイトルバーに現在のバージョン番号を表示し、この更新履歴への直接リンクを提供します。
  - 高速エクスポート：CCSwitch エクスポートがアップストリームモデルの選択をサポートし、エクスポート設定が実際の使用シナリオにより近くなりました。

## 3.5.2
- **問題修正:**
  - 金額表示：極めて小さい数値の表示戦略を最適化し、精度/丸めによる予期せぬ表示を防ぎます。

## 3.5.1
- **新機能:**
  - アカウント管理：「自動チェックイン」スイッチの位置を調整（カスタムチェックインURLの上に移動）し、より直感的な設定にしました。
  - UI：ダイアログのタイトルアイコンからグラデーションの背景色を削除し、視覚的にすっきり統一感のあるものにしました。
- **問題修正:**
  - キーリスト：キーの詳細を展開した際に、ダイアログを閉じると画面が真っ白になる問題を修正しました。

## 3.5.0
- **新機能:**
  - 自動認識：「検出が遅い」というヒントと関連ドキュメントリンクを追加し、ユーザーが問題をトラブルシューティングして解決するのに役立ちます。
  - 外部チェックインの一括オープン：すべてを新しいウィンドウで開くことをサポートし、一括クローズと干渉の軽減を容易にします。
- **問題修正:**
  - 外部チェックインの一括オープン：ポップアップシナリオで全てのサイトが正しく開かれるように、バックグラウンドサービスで実行されるようにフローを再構築しました。

## 3.4.0
- **新機能:**
  - CLIProxy：モデルマッピング設定を強化し、アップストリームモデルの直接選択をサポートすることで、より正確なモデルマッピングを容易にします。
- **問題修正:**
  - API：アクセスキーが常に `sk-` プレフィックスを持つことを保証し、フォーマットの不一致による認識/コピーの問題を防ぎます。

## 3.3.0
- **新機能:**
  - 自動チェックイン：アカウント認識に「ユーザー名」情報を追加し、複数アカウントのシナリオでアカウントを区別しやすくしました。
  - 外部チェックイン：外部チェックインの一括トリガーをサポートし、個別の操作ステップを削減します。
  - 自動更新：最小更新間隔の最大値制限をなくし、更新頻度を制御するために大きな最小間隔を設定できるようになりました。
- **問題修正:**
  - クリップボード読み取り：トリガー条件を厳しくし、非コピーシナリオでの誤作動を減らしました。
  - 兌換アシスタント：ポップアップヒントの前にすべての兌換コードを検証し、無効な兌換ヒントを減らしました。
  - ストレージ：書き込み操作に書き込みロックを追加し、同時書き込み時のデータ一貫性を向上させました。
  - UI：「モデル名をコピー」に関連するローカライズされたテキストを調整しました。

## 3.2.0
- **新機能:**
  - 「モデルリスト」ページに「インターフェース可用性検出」（ベータ版）を新たに追加しました。これは、指定されたモデルで現在のキーが利用可能かどうかを迅速に確認するために使用されます（例：テキスト生成、ツール/関数呼び出し、構造化出力（JSON構造で返す）、インターネット検索（Grounding）などの検出項目）。
  - 「モデルリスト」ページに「CLIツール互換性検出」（ベータ版）を新たに追加しました。これは、Claude Code / Codex CLI / Gemini CLI のツール呼び出しプロセスをシミュレートし、これらのツールにおけるインターフェースの互換性を評価します。
  - 「バージョン情報」ページに「評価とダウンロード」を新たに追加しました。これは、現在のストアソース（Chrome / Edge / Firefox）を自動的に認識し、ワンクリック評価と他のストアからのダウンロードエントリを提供します。
- **問題修正:**
  - サイト更新時にHTTPエラーが発生した場合、ヘルスステータスにステータスコードとエラー原因が表示され、問題の特定に役立ちます。
  - サイドバーモードでは、「サイドバーで開く」ボタンが表示されなくなり、重複して開くことを防ぎます。
