# 更新履歴

本ページでは、一般ユーザー向けの主な更新点（機能変更 / 体験の最適化 / 問題の修正）を記録しています。完全な履歴バージョンとより詳細な技術的な変更については、GitHub Releases をご覧ください。

## 3.8.0
- **新機能:**
  - 利用状況分析：新しい「利用状況分析」ページを追加しました。複数のサイト、複数のアカウントにわたる利用傾向をグラフ化し、「どこで多く使っているか / 多く費用をかけているか / 遅くなっているか」を一目で把握できるようにすることで、コスト管理、照合、問題の切り分けに役立ちます。
  - ダッシュボードの内容：日次概要（リクエスト数、トークン、クォータ）、モデル分布/費用分布、アカウント比較、利用時間ヒートマップ、遅延傾向/ヒストグラムなどを確認できます。
  - 利用履歴の同期：「利用履歴の同期」機能を追加しました。これは「集計された利用状況データ」（元のログは保存されません）を取得し保存するために使用されます。保持日数、自動同期方法、最小同期間隔を設定でき、「同期ステータス」で各アカウントの同期結果とエラーメッセージを確認できます。
- **使用方法:**
  - まず「設定 → アカウント利用状況」で「利用履歴の同期」を有効にし、必要に応じて設定した後、「今すぐ同期」をクリックします。
  - 次に、左側メニューの「利用状況分析」でグラフを確認します。データを保持したり照合したりする必要がある場合は、「エクスポート」をクリックできます。

## 3.7.0
- **新機能:**
  - ソート：アカウントリストが「収益」によるソートに対応しました。ソート優先度に「無効なアカウントを最下部に表示」を追加し、停止中/無効なアカウントが日常的な使用を妨げないようにしました。
  - 自動チェックイン：新しい「インターフェースを開いた際に本日のチェックインを事前にトリガー」機能を追加しました。時間枠内にポップアップ/サイドバー/設定ページを開くと、予定された時間を待たずに、本日のチェックインを事前に実行しようと自動的に試みます。
- **問題の修正:**
  - 自動チェックイン：同一アカウントは1日1回のみ実行されます。リトライは失敗したアカウントに対してのみ行われ、無意味なリクエストや重複した通知を減らします。
- **場所のヒント:**
  - ソート優先度：「設定 → アカウント管理」で調整します。
  - 自動チェックインの事前トリガー/リトライ：左側メニューの「自動チェックイン」で設定します。

## 3.6.0

  - アカウント管理ページを新規追加し、完全なCRUD機能（作成、読み取り、更新、削除）をサポートしました。
  - ポップアップ内のカスタムダイアログを直接関数呼び出しに置き換え、操作を簡素化しました。

## 1.1.1

## 1.1.0
- **新機能:**
  - アカウントの手動追加サポートを新規追加し、UIプロセスを最適化しました。

## 1.0.0
- **新機能:**
  - 現在のサイトの検出とハイライト表示を実装しました。

## 0.0.3
- **新機能:**
  - アカウント識別プロセスを最適化し、アクセスキーの自動作成をサポートするようになりました。
  - アカウントリストにソート可能なヘッダー、シークレットキーコピーダイアログ、ホバー操作ボタンを新規追加しました。
  - アカウント管理に備考フィールドを新規追加しました。
  - ウェブサイト名をクリックしてジャンプできるようになりました。
  - モデルリストがグループ選択をサポートしました。
  - ポップアップページに数字スクロールアニメーションとサイトステータスインジケーターを追加しました。
  - アカウントの追加/編集ダイアログを最適化し、チャージ比率設定とサイト名の自動抽出を含めました。
  - 設定ページシステムを全面的に実装し、ユーザー設定の永続化と自動更新をサポートしました。
  - 自動更新機能のフロントエンドインターフェースとバックグラウンドサービスを強化しました。
  - シークレットキーコピー時に自動的に `sk-` プレフィックスを追加するようにしました。
  - 業界標準のタブスクロール操作体験を導入しました。
  - アカウントのヘルスステータスの動的更新と削除機能を最適化しました。
  - より多くのAIモデルベンダー（例：OneHub、DoneHub、Super-API、VoAPIなど）をサポートしました。
  - ポップアップインターフェースをAPIマネージャー形式にリファクタリングし、本日の総消費金額表示を追加しました。
  - 全体的なスクロールレイアウトとツールチップアニメーション効果を最適化しました。
- **問題の修正:**
  - モデルデータ形式の非互換性、`localStorage` アクセス、APIリクエストクレデンシャルの問題を修正しました。
  - API認証方式を修正しました。
  - URL入力処理と自動更新設定を最適化しました。
  - ページネーションロジック処理ログデータを追加しました。
  - 複数のUIと互換性の問題を修正しました。
