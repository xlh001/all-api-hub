# Redemption Assist

> 引き換えコードをワンクリックで認識し、ワンクリックで引き換え、繰り返しコピー＆ペーストする手間を省きます。

## 機能概要

- **引き換えコードの自動認識**：ウェブページ上でテキストをクリックまたは選択すると、プラグインがそこから引き換えコードを識別しようとします。手動で一文字ずつ確認する必要はありません。
- **便利なポップアップ通知**：引き換えコードの疑いがあるものを検出すると、右下隅に小さな通知カードがポップアップ表示され、すぐに引き換えるかどうかを尋ねます。
- **アカウントのスマートマッチング**：現在のページアドレスに基づいて、All API Hub から最も一致するサイトアカウントを自動的に検索し、自分でリストを探す手間を省きます。
- **ワンクリックで引き換え完了**：「自動引き換え」をクリックすると、プラグインがバックグラウンドでリクエストを送信し、通知バブルの形式で引き換え結果を表示します。
- **独立したオン/オフスイッチ**：設定でRedemption Assistを個別に有効/無効にでき、他の機能モジュールには影響しません。

## 使用前提

1.  **All API Hub に対応するサイトアカウントが既にあること**：
    -   少なくとも1つのアカウントが正常に機能し、残高と権限に異常がないこと。
2.  **ページ上に実際に引き換えコードが存在すること**：
    -   一般的な形式には、「今日の引き換えコード」、「ギフトコード」、「チェックインコード」などがあり、通常は32文字の組み合わせです。

## 使用チュートリアル

### チュートリアル1：Redemption Assistをゼロから設定する

1.  **少なくとも1つのターゲットサイトアカウントを追加する**
    -   アカウント管理で引き換えコードを使用するサイトアカウントを追加し、正常にログインしてアクセスできることを確認します。
2.  **Redemption Assistのスイッチをオンにする（デフォルトでオン）**
    -   「基本設定」ページで「Redemption Assist」をオンにし、他のデフォルト設定を維持するだけで正常に使用できます。
3.  **アカウントにカスタムチェックイン/引き換えアドレスを追加する（推奨）**
    -   アカウント編集ページを開き、「カスタムチェックイン/引き換えアドレス」を見つけます。
    -   普段直接アクセスする引き換えページのアドレスを入力すると、システムが現在のページとアカウントをより正確にマッチングしやすくなります。

上記の手順を完了すると、グローバルな引き換え支援機能を体験できます。

### チュートリアル2：日常使用における典型的な引き換えプロセス

1.  **引き換えコードを含むページを開く**
    -   例：イベント告知ページ、チェックイン報酬ページ、チャンネルメッセージなど。
    -   ページには通常、引き換えコードとして32文字の文字列が表示されます。
2.  **検出アクションをトリガーする**
    -   引き換えコードに対して以下のいずれかの操作を実行します。
        -   マウスで引き換えコードをドラッグして選択する;
        -   引き換えコードがある領域をクリックし、現在の選択テキストにする。
3.  **右下隅のRedemption Assist通知を観察する**
    -   システムが引き換えコードの疑いがあるものを認識すると、右下隅に通知カードがポップアップ表示され、匿名化された引き換えコードのプレビューが表示されます。
4.  **確認して自動引き換えを開始する**
    -   カード内の「自動引き換え」をクリックします。
    -   プラグインは現在のページURLに基づいて、最適なアカウントを自動的に選択し、引き換えリクエストを開始します。
5.  **結果の通知を確認する**
    -   成功時：
        -   通常、「引き換え成功」と入金された**額度**などの情報が表示されます。
    -   失敗時：
        -   失敗理由（例：引き換えコードが無効、引き換えコードは既に使用済みなど）または一般的な失敗通知が表示されます。

### チュートリアル3：複数アカウントのシナリオで適切​​なアカウントを選択する方法

1.  **同じサイトに複数の利用可能なアカウントがある場合**
    -   Redemption Assistは、任意のアカウントを直接使用するのではなく、「アカウント選択」をポップアップ表示します。
2.  **ポップアップでアカウント情報を確認する**
    -   アカウント名、サイトアドレス、メモなどの主要な情報が表示されます。
    -   アカウントが多い場合は、検索フィルターを利用してターゲットアカウントを素早く見つけることができます。
3.  **アカウントを選択して引き換えを確認する**
    -   最適なアカウントを選択したら、「確認」をクリックすると、システムは選択されたアカウントに対して実際の引き換え操作を実行します。

### ワンクリック引き換えの成功率を高める方法

「自動引き換えを1回クリックするだけで完了」を目指す場合は、以下の2つの点を重点的に最適化できます。

-   **よく使うアカウントにカスタムチェックイン/引き換えアドレスを設定する**：
    -   アカウント編集ページで、「カスタムチェックイン/引き換えアドレス」を見つけ、実際に引き換えに使用するページアドレスを入力します。
    -   これにより、システムがURLに基づいてアカウントをマッチングする際の精度が向上し、複数アカウントの曖昧さが軽減されます。

上記の条件を満たせば、ほとんどの場合、「自動引き換え」をクリックするだけで直接成功し、手動でアカウントを選択する必要はほとんどなくなります。

## よくある質問

### Q1：ページに引き換えコードがあるのに、通知がポップアップ表示されないのはなぜですか？

考えられる原因：

-   設定で「Redemption Assist」が有効になっていない;
-   引き換えコードに対して選択などの操作を実行しておらず、プラグインがテキストを読み取る機会がない;
-   引き換えコードが標準の32文字の文字列ではない、または途中にスペースや特殊記号が含まれていて認識できない;
-   ブラウザまたは他のプラグインがスクリプトの実行またはクリップボードへのアクセスを制限している（例：極端なシークレットモード、一部のセキュリティプラグインによるブロック）。

推奨されるトラブルシューティング手順：

1.  All API Hub の設定で「Redemption Assist」がオンになっていることを確認する;
2.  アカウント管理で現在のサイトアカウントが追加されており、正常にアクセスできることを確認する;
3.  常用デスクトップブラウザ（Chrome / Edge / Firefox）で再試行する;
4.  それでも解決しない場合は、リポジトリのIssuesにサイトのスクリーンショットとデバッグ情報を添えてフィードバックしてください。

### Q2：自動引き換えはアカウントのセキュリティに影響しますか？

-   Redemption Assistのロジックはローカルブラウザ内でのみ実行され、アカウント情報や引き換えコードを第三者のサーバーにアップロードすることはありません;
-   すべてのAPIリクエストは、対応するアグリゲーションサイトに直接送信されます;
-   ブラウザ自身のパスワードとプライバシー保護ポリシー（ブラウザのパスワード管理、プライベートモードなど）と組み合わせて使用することをお勧めします。
