# インストール方法と更新について

All API Hub には、よく使われるインストール方法が 2 つあります。**ブラウザストア版** と **GitHub 手動インストール版** です。どちらを選べばよいか迷う場合は、まずブラウザストア版を使ってください。

::: tip かんたんな結論
Chrome、Edge、Firefox ユーザーは、対応するストア版を優先してインストールしてください。ストア版はブラウザが自動更新するため、最も手間がかかりません。ストアにまだ新しいバージョンが来ていない場合、ストア版をインストールできないブラウザを使っている場合、または一時的に修正を確認したい場合だけ、GitHub からの手動インストールを検討してください。
:::

## どのバージョンをインストールすべきですか？

| 状況 | おすすめ | 更新方法 |
|---|---|---|
| Chrome を使っている | Chrome ウェブストア版 | ブラウザが自動更新します。拡張機能内で手動チェックもできます |
| Edge を使っている | Edge アドオン版 | ブラウザが自動更新します。拡張機能内で手動チェックもできます |
| Firefox を使っている | Firefox Add-ons 版 | Firefox が自動更新します |
| QQ / 360 / Brave / Vivaldi / Opera などのブラウザを使っている | GitHub の Chrome パッケージ | 今後の更新は新しいバージョンを手動でダウンロードする必要があります |
| ストア版に緊急修正がまだ届いていない | 一時的に GitHub Stable を使う | 今後の更新は新しいバージョンを手動でダウンロードする必要があります |
| 開発中の機能を早めに試したい | Nightly / 開発版 | 不安定な可能性があるため、テスト用途にのみおすすめします |

QQ、360、Brave、Vivaldi、Opera などのブラウザで拡張機能を手動で読み込む場合は、[QQ / 360 等ブラウザへのインストールガイド](./other-browser-install.md) を確認してください。

## GitHub には新しいバージョンがあるのに、ストアがまだ更新されていないのはなぜですか？

これは正常な動作です。プロジェクトでは通常、先に GitHub へ新しいバージョンを公開し、その後で各ブラウザストアへ提出します。ストア側では審査と配信が必要なため、少し遅れることがあります。

よくある理由は次のとおりです。

- ストアが新しいバージョンをまだ審査中。
- ストアの審査は通過したが、まだあなたのブラウザへ配信されていない。
- ブラウザがまだ拡張機能の更新を確認していない。
- 会社の端末、ブラウザポリシー、またはネットワーク環境が拡張機能の更新を制限している。

ストア版を使っている場合、通常はブラウザの自動更新を待てば十分です。急いで修正を使いたい場合は、下の手動チェック方法を試してください。

## 拡張機能内で更新を確認する

All API Hub の設定ページを開き、バージョンと更新のエリアで **「今すぐ確認」** をクリックします。

拡張機能は次の 2 つを確認します。

1. GitHub に新しい正式版が公開されているか。
2. あなたのブラウザストア版がすでに更新可能か。

表示される結果には、次のようなものがあります。

- **現在は最新の正式版です**：対応は不要です。
- **更新可能な新しいバージョンがあります**：GitHub には新しいバージョンがあります。ストア版を使っている場合は、ストアの審査または配信を待つ必要があるかもしれません。
- **ストア更新を利用できます**：ブラウザが新しいバージョンを取得済みです。**「再読み込みして更新」** をクリックすると反映できます。
- **確認に失敗しました**：ネットワーク、GitHub へのアクセス、またはブラウザの更新確認が一時的に利用できない可能性があります。少し待ってから再試行してください。

::: warning 注意
拡張機能内の確認機能で、ブラウザストアの審査を飛ばすことはできません。つまり、GitHub に新しいバージョンが公開されていても、ストアにまだ掲載されていない場合、ストア版をそのバージョンへ直接更新することはできません。
:::

## ブラウザ側でストア更新を手動確認する

Chrome または Edge のストア版を使っている場合は、ブラウザに拡張機能の更新確認を実行させることもできます。

### Chrome

1. アドレスバーで `chrome://extensions/` を開きます。
2. 右上の **デベロッパーモード** をオンにします。
3. ページ上の **更新** をクリックします。
4. ブラウザが拡張機能の更新を確認するまで待ちます。

### Edge

1. アドレスバーで `edge://extensions/` を開きます。
2. **開発者モード** をオンにします。
3. ページ上の **更新** をクリックします。
4. ブラウザが拡張機能の更新を確認するまで待ちます。

この方法で更新できるのは、ストアがすでに提供しているバージョンまでです。ストアがまだ審査中の場合、新しいバージョンは表示されません。

### Firefox

Firefox は通常、拡張機能を自動更新します。`about:addons` を開いて拡張機能管理ページに入り、利用可能な更新があるか確認することもできます。Firefox のバージョンによって、ボタンの位置が少し異なる場合があります。

## GitHub から手動インストールするのはどんなときですか？

GitHub からの手動インストールは、一時的な利用に向いています。

- ストア版に、急いで必要な修正がまだ届いていない。
- 現在使っているブラウザに利用可能なストア版がない。
- 新しいバージョンで問題が解決したか確認したい。

手動インストールの前に、次の点に注意してください。

- 手動インストール版は通常、自動更新されません。
- 今後の更新では、新しいパッケージを再度ダウンロードする必要があります。GitHub リポジトリを Star / Watch しておくと、新しいリリースに気づきやすくなります。
- ストア版から手動インストール版へ切り替える前に、データをバックアップとしてエクスポートしておくことをおすすめします。
- 明確にテスト目的でない限り、複数の提供元から All API Hub を長期間同時にインストールすることはおすすめしません。

手動インストールと更新の手順は、[QQ / 360 等ブラウザへのインストールガイド](./other-browser-install.md) を確認してください。

## よくある質問

### 「今すぐ確認」を何度も押す必要がありますか？

いいえ。ストア版はブラウザが自動更新します。GitHub に新しいバージョンが公開されたことを確認したい場合、またはストア版がすでに更新可能か確かめたい場合だけ、手動で確認してください。

### 拡張機能内の「今すぐ確認」は GitHub のインストールパッケージを自動でダウンロードしますか？

いいえ。バージョン状態を表示し、ブラウザにストア版の更新確認を依頼するだけです。GitHub のインストールパッケージは、Releases ページから自分でダウンロードする必要があります。

### 「再読み込みして更新」をクリックするとデータは消えますか？

消えません。すでにダウンロード済みのストア更新を反映するだけです。アカウント、Key、設定などのデータは、ブラウザの拡張機能ストレージに残ります。

### GitHub には新しいバージョンがあるのに、拡張機能ではストア更新がないと表示されます。どうすればよいですか？

ストア版を使っている場合は、まずストアの審査と配信を待ってください。急いで修正が必要な場合は、一時的に GitHub Stable を手動インストールできます。利用前にバックアップをエクスポートしておくことをおすすめします。

### 手動インストール版はどうやって更新しますか？

新しいパッケージをダウンロードし、新しい拡張機能ディレクトリを読み込み直します。詳しい手順は、[QQ / 360 等ブラウザへのインストールガイド](./other-browser-install.md#update-extension) を確認してください。
