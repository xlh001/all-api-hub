# 余额历史

> 记录账号每日余额与收支快照，并用图表长期观察变化趋势。适合做对账、看充值消耗节奏，或排查“余额为什么忽高忽低”。 

## 功能概览

- **按日记录快照**：每个账号每天最多记录一条快照。
- **记录余额与收支**：可保存余额、今日收入、今日支出、记录时间和来源。
- **多账户汇总**：支持按标签、账户和时间范围汇总查看。
- **图表分析**：提供概览、趋势、账户分布和账户汇总表。
- **手动维护**：支持立即刷新和立即清理过期数据。

## 记录什么数据

每个账号每天最多保留一条聚合快照，常见字段包括：

- 余额 / 额度：`quota`
- 今日收入：`today_income`
- 今日支出：`today_quota_consumption`
- 记录时间：`capturedAt`
- 来源：`source`

其中 `source` 目前主要有两类：

- **刷新驱动**：账号刷新成功后更新当天快照
- **日终抓取**：接近当天结束时的后台补抓

## 如何启用

1. 打开 **`设置 → 基本设置 → 余额历史`**。
2. 开启 **`启用余额历史`**。
3. 视需要设置 **`保留天数`**。
4. 如需尽量补齐每日收入 / 支出，可开启 **`日终抓取`**。

启用后，再刷新任意账户，即可开始写入第一条快照。

## 两种记录方式

### 1. 刷新驱动

当账号刷新成功后，会更新当天快照。

特点：

- 不需要额外等待定时任务
- 更适合日常打开插件时自然积累数据
- 会遵循“显示今日收支”相关开关，不会为了余额历史额外强制拉日志

### 2. 日终抓取

启用后，插件会在每天接近 `23:55` 尝试补抓一次当天数据。

特点：

- 适合希望尽量补齐收入 / 支出历史的人
- 依赖浏览器对 `Alarms API` 的支持
- 属于 best-effort，不保证每天都一定成功

## 使用前需要注意

如果你想记录 **收入 / 支出历史**，建议满足以下任一条件：

- 开启“显示今日收支”，让刷新时顺带获取今日收支
- 开启“日终抓取”，让插件在每天结束前补抓一次

如果这两项都没有启用，那么余额历史里通常只能稳定看到余额快照，而收入 / 支出图表可能为空。

## 页面怎么看

进入 **`设置 → 余额历史`** 后，常用区域包括：

### 筛选区

支持按以下维度过滤：

- 标签
- 账户
- 时间范围
- 快捷区间（近 7 天、30 天、90 天、180 天、1 年）
- 货币单位（跟随全局货币设置）

### 概览

会显示当前筛选区间内的核心汇总指标，例如：

- 结束余额
- 区间净变化
- 区间收入
- 区间支出
- 覆盖账户数量

### 趋势图

可切换查看：

- 余额
- 收入
- 支出
- 净变化

也可切换视图范围：

- **按账户**：分别看每个账户的曲线
- **汇总**：把所选账户按天汇总求和

图表支持：

- 折线图 / 柱状图切换
- 多账号对比
- 覆盖不足提示

### 账户分布

用于查看某一天或某个指标在不同账户之间的分布情况。

支持：

- 饼图 / 直方图切换
- 选择参考日期
- 余额 / 收入 / 支出 / 净变化切换

如果当前指标存在负值，饼图会自动不适用，这是正常限制。

### 账户汇总表

表格会汇总每个账户在当前区间内的：

- 开始余额
- 结束余额
- 区间收入
- 区间支出
- 区间净变化
- 快照覆盖率
- 收支覆盖率

很适合做人工对账或快速找异常账户。

## 常用操作

### 立即刷新

点击 **`立即刷新`**，会对当前选择的账户主动抓取最新快照，适合：

- 刚启用余额历史后补第一批数据
- 想立刻更新今天的数据
- 对比某次充值 / 消耗前后的变化

### 立即清理

点击 **`立即清理`**，会按保留天数删除过旧记录，适合在你缩短保留天数后手动收口数据。

## 重要限制

- **best-effort**：浏览器休眠、网络异常、站点限制都可能导致某些日期没有快照。
- **不会回填历史**：不会为了补齐过去日期而反查整段历史日志，避免产生大量请求。
- **仅本地保存**：当前版本下，余额历史数据保存在本地，不会随普通数据导入导出或 WebDAV 一起迁移。
- **禁用账号会被跳过**：相关刷新与定时任务会自动跳过已禁用账号。

## 使用建议

- 想长期看趋势时，把保留天数调大一些。
- 想让收入 / 支出更完整时，启用日终抓取。
- 做对账时先筛选具体标签或账户，再看汇总表，效率更高。
- 若你更关注请求量、模型分布和延迟，而不是余额变化，请改看 [用量分析](./usage-analytics.md)。
