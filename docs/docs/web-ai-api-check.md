# 网页 API 嗅探与验证

> 嗅探网页上的 `Base URL` 与 `API Key` 并即时验证。支持右键菜单快速检查，以及针对特定站点的自动探测。

## 功能背景

在浏览 AI 社区（如 Linux.do）、Github Issue 或技术文档时，经常会遇到分享的测试 Key。传统的做法是复制 `Base URL`、复制 `Key`、打开插件、填入配置、点击验证。

**网页 API 嗅探** 功能将这一流程简化为：   
- 选中文字 → 右键「检查 AI API」 → 直接测试。
- 或者：在白名单站点内，插件自动识别页面上的配置并弹出提示。

## 核心能力

### 1. 右键菜单检查 (Context Menu)
在任何网页上选中包含 `Base URL` 或 `API Key` 的文本，点击右键并选择 **“检查 AI API”**。
插件会尝试从选中文本（或剪贴板）中提取有效的接口信息，并在当前页面内弹出一个测试浮层。

### 2. 自动嗅探 (Auto Detect)
在特定的白名单站点（如 `linux.do`），当你打开一个包含大量 Key 的帖子时，插件会自动扫描页面内容。
- 如果提取到疑似有效的 `Base URL + Key` 组合，页面右上角会弹出轻量提示。
- 点击提示即可打开测试对话框。

### 3. 页面内置测试对话框
无需离开当前网页，即可执行以下操作：
- **自动提取**：从源文本中自动分离 `Base URL`、`密钥` 和 `API 类型`。
- **即时测试**：点击“开始测试”，插件会验证 Key 的连通性及可用模型。
- **获取模型列表**：一键拉取上游站点支持的全部模型 ID。
- **保存到 API 凭据库**：测试通过后，可一键将其保存到插件的 [API 凭据库](./api-credential-profiles.md) 中，方便后续复制、验证或导出到 CherryStudio 等工具。

## 配置与自定义

进入 **`设置 → 基础设置 → 网页 API 检查`** 进行如下配置：

### 1. 基础开关
- **右键菜单**：是否在浏览器右键菜单中显示“检查 AI API”。
- **自动探测**：是否在匹配的站点中启用后台自动扫描。

### 2. 站点白名单 (URL Whitelist)
你可以自定义哪些站点允许插件进行自动嗅探。
- 支持 **正则表达式 (RegExp)** 匹配（忽略大小写）。
- **默认配置**：通常已内置了对常见 AI 论坛和交流社区的匹配规则。
- **示例**：输入 `linux\.do` 可匹配该论坛的所有页面。

## 隐私与安全

- **本地提取**：嗅探和提取过程完全在你的浏览器本地执行。
- **隐私保护**：插件不会自动向任何服务器上传探测到的文本，只有在你点击“开始测试”时，才会按照你填写的 `Base URL` 发送 API 请求。
- **静默扫描**：自动探测过程极其轻量，不会影响网页的滚动和加载性能。
